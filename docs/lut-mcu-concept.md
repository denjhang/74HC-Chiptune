# 查表 MCU 构想 (LUT-MCU)

## 核心思路

74HC 只做调度，ROM 当 ALU。所有算术逻辑运算通过查预计算表完成，不使用硬件运算单元。

## 与 WT 合成器的关系

WT 合成器验证了查表累加器架构的可行性。MCU 是同一思路的通用化扩展：
- WT：查表做 wave×level×vol
- MCU：查表做 ADD/SUB/AND/OR/XOR/MUL 等全部运算

所有模块统一挂在内存总线上，主机写寄存器就完事：

```
┌─────────────────────────────────────────┐
│              内存总线 (A0-18, D0-7)       │
│   GAL16V8 地址译码                       │
├──────────┬──────────┬──────────┬─────────┤
│ 主 CPU   │ VGA 显卡  │ WT 音源   │ PS/2   │
│ 24片     │ 7片      │ 9片      │ 1片    │
│ 地址0-3  │ 地址4-6  │ 地址7-9  │ 地址A  │
└──────────┴──────────┴──────────┴─────────┘
```

MCU 做主机，WT 音源和 VGA 显卡做从机。所有从机都是查表累加器架构：
- WT 音源：ROM 查 wave×level×vol，RAM 存通道参数
- VGA 显卡：ROM 查字模/图案，RAM 存 framebuffer
- 主机只需要往总线写数据，不占 CPU 时间

总计 32 片（含 PS/2），三层板叠起来就是完整的 8-bit 电脑。

## 8-bit 硬件 MCU

### 芯片清单（24 片）

| 模块 | 芯片 | 数量 | 说明 |
|------|------|------|------|
| 程序计数器 16-bit | 74161 | 4 | PC 增量/跳转 |
| 地址 mux | 74157 | 4 | 切换 PC/操作数地址 |
| 地址+数据锁存 | 74377 | 4 | MAR + MDR + 累加器 |
| 运算 ROM (ALU) | 39SF040A | 1 | 8 种运算 × 64K 查表 |
| 程序 ROM | 39SF040A | 1 | 指令 + 常量数据 |
| RAM | 62256 | 1 | 寄存器 + 栈 + 数据（32KB） |
| 控制解码 | 74138+74139+7404 | 3 | 指令译码 → 控制信号 |
| 逻辑门 | 7400/7408/7432 | 2 | 控制逻辑补充 |
| SD 卡接口 | 74245 | 1 | 总线隔离 |
| UART | MAX3232 + 74245 | 2 | 串口终端 |
| 地址译码 | GAL16V8 | 1 | 地址空间划分 + I/O 使能 |
| **合计** | | **24 片** |

### ALU 查表方案

8-bit × 8-bit = 64K 条目，每条目 8-bit。一片 39SF040A (512KB) 可存放 8 种运算：

| 区域 | 运算 | 大小 |
|------|------|------|
| 0x00000-0x0FFFF | ADD (A+B) | 64KB |
| 0x10000-0x1FFFF | SUB (A-B) | 64KB |
| 0x20000-0x2FFFF | AND | 64KB |
| 0x30000-0x3FFFF | OR | 64KB |
| 0x40000-0x4FFFF | XOR | 64KB |
| 0x50000-0x5FFFF | MUL (低8位) | 64KB |
| 0x60000-0x6FFFF | 保留 | 64KB |
| 0x70000-0x7FFFF | 保留 | 64KB |

标志位（Zero/Carry/Sign）可编码在额外输出位或单独查表。

### I/O 地址映射

| 地址 | 设备 | 接口 |
|------|------|------|
| 0x00-0x01 | SPI 数据/控制 | 软件_bit-bang (SD卡) |
| 0x02-0x03 | I2C 数据/控制 | 软件_bit-bang |
| 0x04-0x07 | GPIO 端口 | 74245 隔离 |
| 0x08-0x0B | UART 数据/状态 | 软件_bit-bang 115200bps |

SPI/I2C/GPIO 全部软件 bit-bang，不需要额外芯片。GAL16V8 做地址译码。

### ISP 在线烧录

39SF040A 支持软件写命令序列（不需要特殊电压）。方案：
- 外接一颗便宜单片机（STC8/CH551）做 ISP 控制器
- 烧录时 74HC 全部高阻，烧完释放总线
- ISP 单片机平时可兼做串口转并口桥
- 参考 Gigatron v1a：SD 卡直接挂内存总线，GAL16V8 + 74245

## 16-bit vCPU + lcc 移植（Gigatron 路线）

1. 8-bit 硬件 CPU 跑 ~12 MIPS
2. 软件 interpreter 模拟 16-bit vCPU（每条 16-bit 指令约 5-10 个 8-bit 拍）
3. vCPU 提供 16-bit 整数、指针、栈 → 满足 C 最小要求
4. lcc 后端生成 vCPU 汇编

预计 vCPU 性能：~1-2 MIPS（与 Apple II 的 6502 相当）

参考 Gigatron vIRQ：纯软件中断，利用时序空隙实现前后台并行（后台播音乐 + 前台跑程序，几乎零性能损失）。

## PCB 方案

- JLC 双层板，白嫖
- 全部自动布线
- 走线 < 5cm，12MHz 稳定运行
- 如需更高频率加 74HC245 总线驱动
