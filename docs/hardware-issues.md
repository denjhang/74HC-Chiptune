# 硬件落地问题盘点

仿真跑通 ≠ 硬件能做。下面是当前 RTL 落到 PCB 上所有「还没实例化」和「实现不了」的问题。

## A. reg 冒充芯片（核心问题）

仿真用 reg 数组代替真实芯片。每条 reg 都需要对应的物理 IC，但 RTL 没实例化。

### A1. phase 存储 (62256 / 189)

| 项 | 现状 | 问题 |
|---|---|---|
| `phase_mem[0:14]` | 15×4=60bit reg 数组 | **62256 单端口** → 读改写 I/O 冲突，需严格时序 + 可能要 244 双向缓冲 |
| `phase_v0/v1/v2` | 3×20=60bit reg | **与 phase_mem 重复**（仿真图省事，硬件两套存储无意义） |
| 替代方案 | — | 改用 **74LS189 (16×4 双端口 RAM)** → 读写引脚独立，无冲突。但需找货源（停产） |
| 地址映射 | `{c_voice[1:0], c_param_addr[2:0]}` = 5 bit | 189 是 4-bit 地址（16 单元），刚好够 3 voice × 5 nibble = 15 |
| 数据宽度 | 4-bit | 匹配 189（4-bit）和 283（4-bit 加法器），但 62256 是 8-bit 浪费一半 |

### A2. 累加器锁存 (174)

| 项 | 现状 | 问题 |
|---|---|---|
| `accum_q[5:0]` | 6-bit reg | 一片 174 (6 D-FF) 刚好。但 **CLK 门控**：`c_adder_clk=1` 时才锁存，174 无 Enable |
| 解决 | — | (a) 门控时钟 `CLK = STEP_CLK & c_adder_clk` 需 74HC00；(b) 改用 377（有 Enable 但 8 位浪费） |

### A3. vol/wave 锁存 (174)

| 项 | 现状 | 问题 |
|---|---|---|
| `cur_vol_r[3:0]` + `cur_wave_r[2:0]` | 7-bit reg | **一片 174 只有 6 D-FF**，不够！需 2 片 174 或 1 片 273 |
| 锁存条件 | `param_oe_n=0 && param_addr=5/6` | 需译码 + Enable 控制 |

### A4. DAC 输出锁存 (273)

| 项 | 现状 | 问题 |
|---|---|---|
| `dac_out[7:0]` | 8-bit reg | 273 直接连即可。**唯一干净的**。但 RTL 没实例化 hc273，只声明 `output reg` |
| 锁存条件 | `c_out_latch=1` 时 | 273 也无 Enable，CLK 门控问题同 A2 |

### A5. SPFM 模块内的 reg

| 模块 | 现状 | 问题 |
|---|---|---|
| `spfm_373.latch[7:0]` | reg | 行为对（透明锁存），可直接用 74HC373 |
| `spfm_174.Q[5:0]` | reg | 行为对，可直接用 74HC174 |
| `spfm_377.Q[7:0]` | reg | 行为对，可直接用 74HC377 |
| 但 | — | 这三个模块**没实例化 hc373/hc174/hc377**，自己写了一遍 reg 版本 |

## B. 组合逻辑门电路（被 RTL 隐藏）

Verilog 的 `&`, `~`, `case` 在硬件里需要门电路。下面这些被 RTL 折叠了，没分配 IC：

### B1. SPFM 总线译码

| RTL | 实际需要 |
|---|---|
| `le = ~CS_n & ~WR_n & RST_n` | 3 输入与门 + 反相器 |
| `addr_req = ~CS_n & ~WR_n & ~A0` | 3 输入与非 |
| `data_req = ~CS_n & ~WR_n & A0` | 同上 |
| `addr_wr = u2_q[1] & ~u2_q[2]` | 2 输入与 + 反相 |
| `data_wr = u2_q[4] & ~u2_q[5]` | 同上 |

**合计**: 1 片 74HC10 (3×3 与非) + 1 片 74HC04 (6 反相) = **+2 IC**

### B2. 控制字解码

inst_dq[7:0] 已经是解码后的信号，无需额外门。但：

| RTL | 实际需要 |
|---|---|
| `add_b = (param_oe_n==0) ? do_r[3:0] : 4'b0` | **4-bit 2选1 mux** → 74HC157 |
| `add_c0 = adder_clr_n ? accum_q[4] : 0` | 1-bit 2选1，可塞进 157 多余通道 |
| `addr_R = {voice, param} + ...` | 实际是 `step[4:0]` 直连，无加法（仿真冗余） |

**合计**: 1 片 74HC157 = **+1 IC**

### B3. 7134 Right 地址生成

| RTL | 实际 |
|---|---|
| `addr_R = {7'b0, c_voice, 3'b0} + {9'b0, c_param_addr}` | 加法 + mux，实际就是 `step[4:0]` 直连 A0R-A4R |
| **简化**: `A0R-A2R = step[2:0], A3R-A4R = step[4:3]` | 无需加法器，但 RTL 写成加法是冗余 |

## C. 时序问题（仿真掩盖了）

### C1. ROM 输出延迟

| 信号 | t_pd | 影响 |
|---|---|---|
| inst_dq (39SF040) | 70ns | step 变化后 70ns 才稳定 |
| rom_dq (39SF040) | 70ns | 同上 |
| do_r (7134) | ~25ns | 较快 |
| 283 add_sum | ~10ns | HC 系列典型 |

**仿真里全是 0 延迟组合逻辑**，所以 `posedge STEP_CLK` 能读到稳定值。硬件里 `posedge` 时这些信号可能还没稳定（尤其 inst_dq 要 70ns）。

**STEP_CLK 周期 = 325ns**，半周期 162ns，理论上够（70ns ROM + 10ns 加法 = 80ns < 162ns）。但累加器锁存（A2）和 DAC 锁存（A4）的 setup/hold 要仔细算。

### C2. negedge 锁存的硬件等价

仿真里 `always @(negedge STEP_CLK)` 解决了 delta cycle 问题。硬件里**没有 negedge 锁存器**这种东西 —— 174/273 都是 posedge。

**硬件等价**: 用 STEP_CLK 的反相时钟驱动这些锁存器。需要 1 片 74HC04 的 1 通道产生 `~STEP_CLK`。

或者重新设计时序，全部走 posedge + 严格控制组合路径延迟。

### C3. 62256 读改写

仿真 `phase_mem[idx] <= add_sum` 一个周期完成。硬件 62256：

```
T=0   posedge STEP_CLK, step 更新
T=70  ROM 输出 inst_dq 稳定, RAM 地址稳定
T=140 RAM 数据有效 (t_AA=70)
T=150 283 输出稳定 (10ns)
T=162 negedge STEP_CLK → RAM 进入写相 (OE_n=1)
T=187 RAM I/O 高阻 (t_HZOE=25)
T=187+ WE_n=0, 283.S 驱动 RAM I/O → 写入
T=325 下一个 posedge
```

**问题**: 283 的 S 输出**一直活跃**。读相时它也在驱动 RAM I/O，和 RAM 的 DO 冲突。必须：
- (a) 加 74HC244 双向缓冲（+1 IC）
- (b) 让 283 的 B 输入 mux 在读相时为 0，S = A + 0 + 0 = A（直通），但仍可能冲突
- (c) 改用 189（双端口，DI/DO 分开），彻底解决

## D. 数据宽度不匹配

### D1. phase 是 20-bit，但数据通路是 4-bit

| 操作 | 仿真 | 硬件 |
|---|---|---|
| 一次累加 | `phase += freq` 20-bit | 5 次 4-bit 加法（283 是 4-bit） |
| 波表地址 | `phase[19:12]` 8-bit | 需要从 189 读 2 个 nibble 拼接 |

仿真里 `phase_v0[19:12]` 直接取高字节。硬件需要：
- nibble 3 累加完锁存到 A（1 片 174）
- nibble 4 累加完锁存到 B（1 片 174）
- 拼成 8-bit 给 ROM 地址

**+2 IC**（phase 高字节缓存）

### D2. 波表 ROM 地址 15-bit vs 19-bit

| 仿真 | 实际 |
|---|---|
| `{wave[2:0], vol[3:0], phase[19:12]}` = 15-bit | 39SF040 有 19-bit 地址，够用 |
| ROM 存 wave×vol 组合（8×16×256=32KB） | 见 E2 |

### D3. 7134 数据 8-bit，但只用低 4 位

| 信号 | 实际用 | 浪费 |
|---|---|---|
| freq nibble | 4-bit | 高 4 位接地 |
| wave index | 3-bit | 高 5 位接地 |
| vol | 4-bit | 高 4 位接地 |

**7134 是 8-bit 双端口 RAM**，用 8-bit 是合理的（CPU 写 8-bit 方便）。但 freq 用 4-bit 是为了匹配 283 加法器。

## E. 架构层面的冗余

### E1. 仿真里 phase_mem 和 phase_v0/1/v2 重复

```verilog
reg [3:0] phase_mem [0:14];        // 60 bit
reg [19:0] phase_v0, phase_v1, phase_v2;  // 60 bit
```

两套存同样的东西。硬件只要一套。仿真写两套是为了 `case(c_voice)` 选 phase_v* 方便，但硬件直接从 189 读对应地址即可。

### E2. 波表 ROM 存 wave×vol 全组合

| 方案 | ROM 容量 | 缺点 |
|---|---|---|
| **当前**: `{wave, vol, phase}` 全组合 | 8×16×128=16KB | 需要 cur_vol/cur_wave 锁存器，地址位宽 15-bit |
| **WSG**: 只存 `{wave, phase}`，vol 用模拟乘法 | 8×32=256B | 需要 DAC 后接模拟音量（CD4066 + 电阻网络） |

当前方案 ROM 大但锁存器多。WSG 方案 ROM 小但模拟部分复杂。

### E3. 62256 用 32KB 只存 15 个 nibble

62256 = 32K×8 = 262144 bit。实际用 15×4 = 60 bit。**利用率 0.02%**。完全可以用 74LS189 (16×4=64bit) 替代，且是双端口。

## G. 该砍哪些功能来精简

按「砍了能省最多 IC」排序：

### G1. 砍 vol 数字乘法，改模拟音量 (省 1-2 IC)
- 去掉 cur_vol_r 锁存 (省 174 半片)
- 波表 ROM 只存 `{wave, phase}`，容量从 16KB 降到 1KB
- 加 CD4066 (1 IC) + 电阻网络
- **净省 0-1 IC**（CD4066 抵消 174）

### G2. 砍 128 点波形，改 32 点 (省 2 IC)
- 波表 ROM 地址从 15-bit 降到 10-bit
- phase 高字节缓存从 8-bit 降到 5-bit
- **省 phase 高字节缓存 1-2 片 174**
- 音质下降（32 点 vs 128 点），但有低通滤波可接受

### G3. 用 189 替代 62256 (省时序麻烦，不省 IC)
- 189 双端口，无读改写冲突
- 无需 244 双向缓冲
- 但 189 停产，需找 74219/71822 替代

### G4. 砍 SPFM 总线，改并行接口 (省 3 IC)
- 去掉 373/174/377
- 直接用 8-bit 地址 + 8-bit 数据 + WR（主机直连）
- 主机需多几根线，但模块省 3 IC
- **省 3 IC**，但失去 SPFM 兼容性

### G5. 砍 3 通道，改 1 通道 (省大量 IC)
- phase 存储从 15 nibble 降到 5
- 整个数据通路简化
- 但失去和弦能力

## H. 结论

**当前状态**: 仿真跑通 = 行为正确。**硬件设计 = 0%**，因为 12 个 IC 里只有 5 个实例化了真实芯片，其余 7 个用 reg 冒充。真实 IC 数是 17-18。

**根本原因**: 设计从「Verilog 能仿真」出发，没从「硬件可实现」反推。每个 reg、每个 `?:`、每个 `case` 都该问「这在 PCB 上是什么芯片」，但没问。

**建议**:
1. 先决定架构方向（保持数字 / 改 WSG 风 / 折中）
2. 重写 RTL，**所有存储和锁存都用实例化的芯片**，不允许 reg 冒充
3. 重新仿真验证
4. 再画接线表

否则永远在「仿真 happy → 画图露馅 → 打补丁」循环里。
