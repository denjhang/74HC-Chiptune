## 1. 元件清单(12 IC + 9 电阻 + 1 电容)

从 `*PART*` 段提取封装,结合 `*NET*` 段信号命名逆向出真实型号。

| 位号 | 封装 | 网表信号前缀暗示 | **推断型号** | 推断功能 |
|------|------|------------------|-------------|----------|
| U1 | 24-pin | `AB*`, `/WR*`, `32H..1H`, `/GM`, `SOUND_ON`, `DB*` | **Namco 08xx/09xx 定制 WSG** | 主状态机/时序控制器 |
| U2 | 14-pin | `2H` 接 pin10, `3LK-157-S` 接 pin8 | **74HC04 / 74HC00** | 时钟反相/边沿整形 |
| U3 | 28-pin | `3M-DQ*` 接 11/12/13/15(数据), `32H..1H` 接 5-10(地址) | **2732/2764 EPROM** | wavetable 波形 ROM |
| U4 | 16-pin | `AB*` 经此选通到 U1,接 `3LK-157-S`(pin1, S 脚) | **74HC157** | mux:总线 AB 选择 |
| U5 | 16-pin | `DB*` 输出, `3K-158-*` 接入 | **74HC158** | 反相 mux:数据 DB |
| U6 | 16-pin | `3L-157-Y*` 输出, `2L-DQ*` 输入 | **74HC157** | mux:2L 数据选择 |
| U7 | 16-pin | `3L-157-Y*`, `2K-DQ*` | **74HC157** | mux:2K 数据选择 |
| U8 | 16-pin | `1K-283-*` 完全匹配 283 引脚 | **74HC283** | 4-bit 加法器 |
| U9 | 16-pin | `1K-283-S/C4` 接入, `1L-74-1Q` | **74HC174** | 6×D 触发器(频率锁存) |
| U10 | 28-pin | `3K-158-*`, `1M-DQ*`, `2K-DQ*` | **ROM/RAM** | 频率/参数存储 |
| U11 | 20-pin | R1-R4 接 Q1-Q4, `2M-273-*` 接入 | **74HC273 / 377** | 输出寄存器(8-bit) |
| U12 | 14-pin | `4066-O*`, RES-COM 接 pin1/4/8/11 | **CD4066** | **4 通道模拟开关** |

**电阻/电容**:
- R1-R4:接 U11 输出 Q1-Q4,另一端到 RES-COM → **通道权重电阻**
- R5-R8:接 4066 输出,另一端到 SOUND → **求和电阻**
- R9:VCC 上拉(2.2kΩ 典型)
- C1:SOUND 到 GND,低通滤波

## 2. 时钟链(关键发现)

从 `*NET*` 段提取所有 `?H` 信号:

```
1H  = U3.10, U1.24
2H  = U3.9,  U2.10, U1.23
4H  = U3.8,  U4.13, U1.22
8H  = U3.7,  U4.10, U1.6, U1.21
16H = U3.6,  U4.6,  U1.4, U1.20
32H = U3.5,  U4.3,  U1.2, U1.19
```

这是 **6 级时钟分频链**:`1H → 2H → 4H → 8H → 16H → 32H`,每级 ÷2。

Pac-Man 主时钟 = 6.144 MHz,所以:
- 1H = 3.072 MHz
- 2H = 1.536 MHz
- ...
- **32H = 96 kHz**(WSG 官方采样率,与 Pac-Man 技术文档一致)

**注意**:这些时钟都接 U3(ROM)的地址线!说明 U3 不是普通 ROM,而是**用低频时钟当地址的
"时序步进 ROM"**——它的输出 `3M-DQ0-3`(4-bit)就是**微码**,在不同 32H/16H/8H/4H 组合下
读出不同 step 的控制信号。

这跟 `wsg-4ch-128pt-full/` 的微码 ROM 设计**完全一致**!

## 3. 频率累加链(time-shared 加法器)

```
U10 (ROM/RAM)
   │ 2K-DQ[0:3] ──┐
   │              ├─→ U8 (74HC283) ── 1K-283-S[1:4] ──→ U9 (74HC174)
   │ 2L-DQ[0:3] ──┘                                              │
   │                                                              ▼
   │  3M-DQ[0:3] ──────────── 反馈回 U8 通过 U6/U7 mux 选不同输入
```

**真相**:U8 是**唯一的 1 片 4-bit 加法器**,通过 U6/U7 mux 在不同时钟相位选择不同的输入,
做 4-5 拍完成 16-bit 加法。这就是 Pac-Man 技术文档里说的:

> "The WSG uses a single 4-bit adder to perform all 16/20-bit frequency additions through time-sharing."

具体步骤(每个 32H 周期,4 拍):
1. 拍 0:U6/U7 选当前通道的 phase[3:0] → U8 加 freq[3:0] → U9 锁回 phase[3:0]
2. 拍 1:U6/U7 选 phase[7:4] + 进位 → U8 → U9
3. 拍 2:phase[11:8] + 进位
4. 拍 3:phase[15:12] + 进位

4 拍完成一个 16-bit 加法,32H=96kHz 时,实际 4 通道共享需要 16 拍 → 每通道 24kHz 更新。

**这就是 1980 年用 1 片 283 干完 4 通道 × 16-bit 加法的"魔法"**。

## 4. 输出混音(4066 + 电阻求和)★

**这是 psg4 要直接照搬的核心**。

### 网表摘录

```
*SIGNAL* RES-COM
U12.4  U12.8  U12.11  U12.1   R4.2  R3.2  R2.2  R1.2

*SIGNAL* 4066-O1
R5.2  U12.3

*SIGNAL* 4066-O2
R6.2  U12.2

*SIGNAL* 4066-O3
R7.1  U12.10

*SIGNAL* 4066-O4
R8.1  U12.9

*SIGNAL* SOUND
R5.1  R9.2  R6.1  R7.2  R8.2  C1.2
```

### 引脚对照(CD4066 标准 DIP-14)

| 4066 引脚 | 功能 | 网表连接 |
|-----------|------|----------|
| 1 | IN/OUT A | RES-COM |
| 2 | OUT/IN A | R6.2(4066-O2) |
| 3 | OUT/IN B | R5.2(4066-O1) |
| 4 | IN/OUT B | RES-COM |
| 5 | CTRL B | (未在 net,推测接 SOUND_ON 或通道使能) |
| 6 | CTRL A | (U11.9 = 2M-273-Q4) |
| 7 | GND | GND |
| 8 | IN/OUT C | RES-COM |
| 9 | OUT/IN C | R8.1(4066-O4) |
| 10 | OUT/IN D | R7.1(4066-O3) |
| 11 | IN/OUT D | RES-COM |
| 12 | CTRL C | (U11.6 = 2M-273-Q3) |
| 13 | CTRL D | (U11.5 = 2M-273-Q2) |
| 14 | VCC | VCC |

### 推断的电路

```
                 ┌─── R1 ───┐
                 │          │
   U11.Q1 ──┬──► │ ch_A in  │ 4066-O2 ── R6 ──┐
            │    │          │                 │
   U11.Q2 ──┼──► │ ch_B in  │ 4066-O1 ── R5 ──┤
            │    │          │                 │
   U11.Q3 ──┼──► │ ch_C in  │ 4066-O3 ── R7 ──┼──► SOUND ──► C1 ──► GND
            │    │          │                 │       │
   U11.Q4 ──┴──► │ ch_D in  │ 4066-O4 ── R8 ──┘       ▼
                 │          │                        功放
                 └──────────┘
                       ▲
                       │
                  RES-COM(公共节点)
```

**关键解读**:
1. U11 的 Q1-Q4 是 **4-bit 音量代码**(0000-1111),作为 4066 的 4 个开关控制端
2. R1-R4 一端接 U11 输出(逻辑电平),另一端全部接 RES-COM(模拟求和节点)
3. 4066 不是用作"通道切换",而是用作**逻辑电平到模拟电压的转换开关**:
   - Q1=1 时,R1 通过 4066 ch_A 接到 SOUND 节点
   - Q1=0 时,R1 悬空
4. R1-R4 选不同阻值(典型 R, 2R, 4R, 8R),构成**4-bit R-2R DAC**!

### 工作原理

当 U11 输出 vol 代码 = `1010`(= 10):
- Q1=0 → R1 悬空
- Q2=1 → R2 接通 → 贡献 VCC × (R_load/R2)
- Q3=0 → R3 悬空
- Q4=1 → R4 接通 → 贡献 VCC × (R_load/R4)

总电流 = VCC × (1/R2 + 1/R4) × R_load,等效电压 = 10/16 × VCC

**这就是 4-bit 音量 DAC!** 4066 充当"开关电阻网络"的选通器,R1-R4 是权重电阻。

### 多通道混音?

网表里只有**一组** R1-R4 + 一个 SOUND 节点。说明:
- Pac-Man 板**没有 4 通道同时输出**
- 而是**TDM**:U1 在不同时隙把不同通道的 vol 写到 U11,4066 + RC 低通做平均

跟 `wsg-4ch-128pt-full/` 的 TDM 思路完全一致,只不过这里 4066 取代了 R-2R DAC。

## 5. 控制信号(`/WR*`, `SOUND_ON`, `/GM`)

```
*SIGNAL* /WR0  → U3.4,  U1.15    (写选通 0:主机写 wavetable ROM?)
*SIGNAL* /WR1  → U6.3,  U1.16    (写选通 1:主机写参数?)
*SIGNAL* SOUND_ON → U11.1, U1.17 (静音控制,/MR 接 273 的 pin1)
*SIGNAL* /GM     → U3.20, U1.18  (Game Mode?推测是总线方向控制)
```

`SOUND_ON` 接 U11 pin1(273 的 /MR),说明**静音 = 异步清零输出寄存器**,这是个很
实用的设计——主机可以直接把所有声音关掉而不用逐通道写 0。

## 6. 对 psg4 设计的启示

### 6.1 必须照搬的

1. **4066 + R1-R4 当 4-bit 音量 DAC**(§4)
   - 不要再用 R-2R 电阻梯 + 273 的方案
   - 直接 4066 的 4 路开关 + 4 个权重电阻,简单可靠

2. **RC 低通 + 直接输出**(§4)
   - 不要 LM386 之前的运放,直接电阻分压 → LM386

3. **SOUND_ON 全局静音**(§5)
   - 加一个 SPFM 地址做全局静音,接 377 /MR

### 6.2 可以改进的(Pac-Man 限制于 1980 年代)

1. **U1 定制芯片 → SPFM 接口**
   - Pac-Man 用定制 WSG 是因为当时没有便宜的 8-bit 微控制器
   - 现在用 SPFM 总线接 STM32/ESP32,3 片 74HC 搞定

2. **time-shared 加法器 → 各通道独立 161**
   - Pac-Man 用 1 片 283 + 2 片 157 mux 是**因为 IC 贵**
   - 现在 161 便宜,每通道独立 1 片 161,省掉 mux 的复杂性

3. **微码 ROM → 不要**
   - Pac-Man 用微码 ROM 是因为定时步骤复杂(3 通道 × 4 拍加法)
   - psg4 各通道独立,直接 161 计数,**不需要微码**

### 6.3 Pac-Man 板的"实际 IC 数"

把 U1(定制)排除后,标准 74HC/CMOS IC 共 **11 片**(U2-U12)。

psg4 目标 10 片以内,等于把 Pac-Man 的 11 片砍到 10 片以下,具体方法:
- 砍 wavetable ROM(U3)→ 用方波代替
- 砍参数 ROM/RAM(U10)→ 用 SPFM 直接写
- 砍 time-shared 加法器需要的 mux(U4-U7)→ 各通道独立
- 保留 4066 + 273 + 174 + 373 等核心

详见 [psg4-architecture.md](psg4-architecture.md)。

## 7. 网表未解之谜

1. **U5 是 158(反相 mux)还是 157?**
   - 信号名 `3K-158-*` 强烈暗示 158
   - 158 是反相输出,可能用于"低有效控制信号"译码
   - 待原理图 PDF 确认

2. **U10 是 ROM 还是 RAM?**
   - 28-pin DIP,可能是 6116(RAM)或 2732(ROM)
   - 如果是 RAM,说明 Pac-Man 主机可以动态改波形
   - 待查 Pac-Man 技术文档

3. **4066 的 CTRL A/B/C/D 怎么驱动?**
   - 网表里 CTRL A(pin6)接 U11.9(2M-273-Q4),但 CTRL B(pin5)未在 net 中
   - 可能 CTRL B 接 VCC 常通,或者接 U1 的某个未命名输出
   - 这影响 4-bit DAC 的高位权重是否可控

4. **R1-R4 的阻值**
   - 网表只标封装(R0603),没标阻值
   - 如果是 R-2R-4R-8R 则是标准 DAC,如果是等值则是 4 通道独立混音
   - 影响 psg4 直接照搬时的电阻选型

这些问题留作后续从原理图 PDF / 实物照片考证。
