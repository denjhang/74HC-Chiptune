> 来源：`第七周年纪念版8位CPU《萤火虫1号》指令集.xls`（龙少官方）
> 整理时间：2026-07-05

## 一句话模型

**所有指令都是"目的 = 源"**。目的和源都从同一组总线节点里选，机器码高字节选源、低字节是立即数或 ALU 模式号。

## 总线节点（11 个）

| 节点 | 含义 | 备注 |
|------|------|------|
| **RF** | 寄存器堆（分页：0=SP, 2=返回值, 10/18/26=运算LOP/ROP/DOP, 34=返回地址, 37=缓存）| 程序员模型里 RF 是"当前页"的一个字节 |
| **A** | A 累加器 | ALU 主输入 |
| **A0 / A1 / A2** | 24 位地址寄存器的低/中/高字节 | A2A1A0 = 24 位地址，寻址 16MB |
| **RA** | RAM 地址寄存器 | 指向 RAM 当前字节 |
| **B** | B 输入寄存器 | ALU 第二输入 |
| **RAM** | RAM 数据口（RA 寻址的字节）| 读=RAM→，写=→RAM |
| **IO0 / IO1** | 两个独立 I/O 端口 | 独立寻址空间，非内存映射 |
| **ALU[XX]** | 查表 ALU，XX 选模式 | 地址=(A,B,XX)，输出回总线 |
| **PC** | 程序计数器 | 写入=跳转 |

> 译码控制器、IR、TSL（三态锁存）是数据通路内部节点，不直接对程序员暴露。

## 机器码编码

**16 位定长指令**（ROM 里每个地址 1 字节，1 条指令占 2 字节）：
```
高字节 = 操作码（选"源 = 目的"组合）
低字节 = XX（立即数 / ALU 模式号 / 未使用）
```

### 源选择（高字节高 4 位）

| 高4位 | 源 | 说明 |
|------|----|----|
| 0 | 立即数 XX | XX 直接当数据 |
| 1 | ALU[XX] | ALU 查表结果，XX 选模式 |
| 2 | RF | 寄存器堆 |
| 3 | RAM | RAM[RA] |
| 4 | IO0 | |
| 5 | IO1 | |

### 目的选择（高字节低 4 位）

| 低4位 | 目的 | 说明 |
|------|------|------|
| 0 | RF | |
| 1 | A | |
| 2 | A0 | |
| 3 | A1 | |
| 4 | A2 | |
| 5 | RA | |
| 6 | B | |
| 7 | **PC**（特殊：跳转指令）| 见跳转节 |
| 8 | RAM | |

### 跳转（特殊编码）

| 机器码 | 助记 | 功能 |
|--------|------|------|
| 07XX | `PC = A2A1A0` | 无条件跳转。**XX 最低位必须为 0**（对齐）|
| 17XX | `if(!(ALU[XX] & 0x01)) PC = A2A1A0` | 条件跳转：ALU[XX] 最低位为 0 则跳。XX 选 ALU 模式（通常是比较/减法，借位/相等编码在 bit0）|

> 条件跳转的判据是 **ALU 输出最低位**，这是 ISA 唯一的条件码机制——所有比较都要通过 ALU 模式编码到 bit0。

### 空操作

| 机器码 | 助记 |
|--------|------|
| FFFF | `NOP()`（**两字节都是 FF**）| CPU 控制信号全置空 |

## ALU 查表机制（关键）

ALU 不是组合逻辑运算器，是 **ROM 查表入口**：

```
ALU 地址 = (A, B, XX)   // A 当高位字节, B 当低位, XX 当段/页
ROM[ALU地址] → 输出字节 → 回总线
```

从 glow.cpp 的 `glow_cpu_init()` 看到的 ALU 模式（部分）：
- `ALU_ADD` / `ALU_ADD_C`（带进位加）
- `ALU_A_ADD_1`（A 自增 1）
- `ALU_EQUAL_C`（相等比较，结果编码到输出）

**ALU 表在初始化时由 C 编译器生成并烧进 ROM**（见 glow.cpp 13182 行起 `glow_cpu_init()`），可热改（SRAM + CR2032 + 下载器）。

## 对 SCC 路线的意义（核心洞察）

1. **SCC 波形查表 = ALU 查表，同一机制**：把波形表放进 ALU ROM，`A = ALU[wave_op]` 就是一条"读波形采样"指令
2. **多通道调度 = 几条传送指令**：phase 累加、查波形、乘音量，全是 `XX = YY` 传送 + ALU 查表，不需要专用状态机
3. **条件跳转够用**：循环、分支、通道轮询都能用 `17XX` 实现
4. **24 位地址空间（16MB）**：波形表、程序、数据混在同一空间，远超 PSG 的 8 位 period
