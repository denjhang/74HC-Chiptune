# MiniGlow 数据通路设计（RTL 依据）

> 创建时间：2026-07-05
> 硬约束：**15 片核心 IC 内**，可扩展（仅协处理器）
> 目标：对标 STC32G 音频核心（ADSR/序列/调度），所有乘除交协处理器

## 一、整体架构

```
                ┌─────────────────────────────────────┐
                │           控制信号 (HC138 译码)       │
                │  IR[15:8] → 译码 → 寄存器写使能/三态  │
                └─────────────────────────────────────┘
                              │ 控制位
                              ▼
   ┌────────┐  ┌────────┐  ┌────────┐  ┌────────┐  ┌────────┐
   │  PC    │  │  RA    │  │   A    │  │   B    │  │  IR    │
   │ HC161  │  │ HC273  │  │ HC273  │  │ HC273  │  │ HC273  │
   │ ×4=16b │  │ ×2=16b │  │  8b    │  │  8b    │  │ 16b    │
   └───┬────┘  └───┬────┘  └───┬────┘  └───┬────┘  └────────┘
       │           │           │           │
       │ PC[15:0]  │ RA[15:0]  │ A[7:0]    │ B[7:0]
       │           │           │           │
       ▼           ▼           ▼           ▼
   ┌─────────────────────────────────────────────┐
   │              8 位内部数据总线 DB[7:0]         │
   │   (HC244 三态缓冲隔离各源)                    │
   └─────────────────────────────────────────────┘
       │           │           │           │
       │           │           │           │
       ▼           ▼           ▼           ▼
   ┌────────┐  ┌────────┐  ┌────────┐
   │ ALU    │  │ 628512 │  │ IO 接口 │
   │HC283×2 │  │ SRAM   │  │ (排针)  │
   │ +HC85  │  │ 512KB  │  │ 协处理器│
   └────────┘  └────────┘  └────────┘
```

## 二、寄存器组（CPU 状态）

| 寄存器 | 位宽 | 实现 | 用途 |
|--------|------|------|------|
| **PC** | 16 位 | HC161 ×4 | 程序计数器（段 0 内 64KB 寻址）|
| **RA** | 16 位 | HC273 ×2 | 数据指针（段内偏移，配 SEG 拼 19 位访问 628512）|
| **SEG** | 3 位 | HC273 半个（占 RA 同片或单独）| 段号（A18-A16）|
| **A** | 8 位 | HC273 | 累加器，ALU 第一输入 |
| **B** | 8 位 | HC273 | ALU 第二输入 |
| **IR** | 16 位 | HC273 ×2 | 指令寄存器（{opcode, xx}）|
| **ZP** | 16 位 | （复用 RA？）| 跳转目标—— TBD |

**ZP 复用 RA**：跳转前 RA 当临时跳转目标寄存器，跳转完 RA 不影响后续（跳转后 RA 一般要重设）。这样省一组 16 位寄存器（2 片 HC273）。

→ **跳转指令语义改为 `PC = RA`**（先设 RA = 目标地址，再 JMP）

## 三、ALU 设计（HC283 + HC85）

ALU 只需 ADD/SUB/CMP：

```
      A[7:0] ──┐
                ├── HC283 ×2 级联 = 8 位加法器
      B[7:0] ──┘    (SUB 用 B 取反 + 1，HC86 异或门实现取反)
                    (ADD_C 接上次进位 CY_in)
                    输出 SUM[7:0] + CY_out

      A[7:0] ──┐
                ├── HC85 ×2 级联 = 8 位比较器
      B[7:0] ──┘    (输出 A>B / A=B / A<B 标志)
                    EQUAL_C 用于 JCC 判据
```

**逻辑运算（AND/OR/XOR/NOT/移位）怎么办？**

砍掉查表 ROM 后，逻辑运算没硬件支持。两个出路：
- **出路 A**：CPU 不支持逻辑运算（STC 调度代码用 if/switch，逻辑位运算少）
- **出路 B**：加几片门（HC08 AND / HC32 OR / HC86 XOR），但占片数

**STC wt.c 看了一遍**——逻辑运算用得少（`& 0x1F`、`>> 5` 几处）。可以：
- 短期：用软件实现（A AND B = 通过 ALU 加法 + 巧妙编码，慢但能跑）
- 长期：加 1 片 HC86（异或，4 路）补 XOR，其他暂软件

**决策**：先按 ALU 只做 ADD/SUB/CMP 设计（HC283 + HC85 = 3 片），逻辑运算软件实现。如果实测 ADSR 代码慢，再加门补。

## 四、指令译码（HC138）

机器码 16 位 = {opcode[7:0], xx[7:0]}。opcode 高 4 位 = 源选择，低 4 位 = 目的选择。

**译码用 2 片 HC138**：
- HC138 #1：译 opcode[7:5]（源选择 5 种：imm/ALU/RF/RAM/IO）+ 跳转指令
- HC138 #2：译 opcode[3:0]（目的选择：RF/A/RA/B/RAM/IO0/PC）+ SEG/RA_L/RA_H

→ **2 片 HC138** 共 16 个译码输出，覆盖所有指令。

## 五、628512 地址空间分配（512KB = 8 段 × 64KB）

```
SEG (A18-A16) | 段 | 地址范围 (RA=16位偏移) | 用途
--------------|----|----------------------|------------------
     000      | 0  | 00000-0FFFF          | 程序 ROM
     001      | 1  | 10000-1FFFF          | RF + 全局变量 + 栈
     010      | 2  | 20000-2FFFF          | 通道参数缓冲（CPU↔协处理器）
     011      | 3  | 30000-3FFFF          | 协处理器输出缓冲
     100      | 4  | 40000-4FFFF          | 协处理器私有数据（如波形表预存）
     101      | 5  | 50000-5FFFF          | 预留（乐谱/采样）
     110      | 6  | 60000-6FFFF          | 预留
     111      | 7  | 70000-7FFFF          | 预留
```

地址拼接：`addr[18:0] = {SEG[2:0], RA[15:0]}`

## 六、协处理器接口（标准化排针）

CPU 板上的排针，所有协处理器模块遵守此接口：

```
协处理器插槽（CPU 板侧，2×10 排针 = 20 针）：

  Pin 1:  VCC (+5V)
  Pin 2:  GND
  Pin 3:  CLK (8MHz 系统时钟)
  Pin 4:  RST_n
  
  Pin 5-12: D[7:0]      (双向数据总线，CPU 写参数/读结果)
  Pin 13: CP_TYPE[0]    (协处理器类型，模块上拉/下拉)
  Pin 14: CP_TYPE[1]
  Pin 15: CP_TYPE[2]
  Pin 16: CP_REG_IDX[3:0] (寄存器索引，CPU 写)
  Pin 17: CP_REG_DATA[7:0] (寄存器数据，CPU 写)  ← 这俩怎么分要算
  Pin 18: CP_START       (CPU 写 1 启动)
  Pin 19: CP_STATUS      (协处理器输出 DONE)
  Pin 20: CP_OUT         (协处理器输出)
```

**精确引脚数待算**——20 针可能不够（REG_IDX 4 位 + REG_DATA 8 位 + OUT 8 位 + ...）。可能要 2×20 = 40 针。这部分后续在协处理器设计时定。

CPU 侧：通过 IO0 寄存器组访问协处理器接口（IO0 是 8 位，分时复用地址/数据）：
- IO0 = address（CP_REG_IDX）+ 写脉冲 → 协处理器锁存
- IO0 = data（CP_REG_DATA）+ 写脉冲 → 协处理器存数据
- IO0 = read（CP_OUT）→ 读协处理器输出

| # | 型号 | 位号 | 功能 |
|---|------|------|------|
| 1 | **HC628512** | U_RAM | 512KB SRAM（程序/RF/参数/输出/协处理私有）|
| 2 | HC161 | U_PC0 | PC[3:0] |
| 3 | HC161 | U_PC1 | PC[7:4] |
| 4 | HC161 | U_PC2 | PC[11:8] |
| 5 | HC161 | U_PC3 | PC[15:12] |
| 6 | HC273 | U_RA_L | RA[7:0] |
| 7 | HC273 | U_RA_H | RA[15:8] |
| 8 | HC273 | U_A | A[7:0] |
| 9 | HC273 | U_B | B[7:0] |
| 10 | HC273 | U_IR_L | IR[7:0]（xx）+ SEG[2:0] 复用 |
| 11 | HC273 | U_IR_H | IR[15:8]（opcode）|
| 12 | HC283 | U_ALU_LO | ALU 低 4 位加法 |
| 13 | HC283 | U_ALU_HI | ALU 高 4 位加法 |
| 14 | HC138 | U_DEC | 指令译码（源/目的选择）|
| 15 | HC00 | U_CTRL | 控制逻辑（NAND，写使能/三态控制）|

**待补的可能**（突破 15 片的话）：
- HC85（比较器，JCC 判据）—— 能否用 HC283 减法 + 检零代替？SUB 后判结果零 = EQUAL。**可以！** 用 HC283 做 SUB，结果零标志从 HC283 输出 + 一个 HC08（4 与门）检全零。HC08 可能要 1 片，挤进 HC00 那片？HC00 是 NAND，能改用 HC04（反相）+ 与门组合。**这细节留给 RTL 时再定**。
- HC244（总线驱动）—— 如果各寄存器自带三态输出（HC273 没有！HC373/HC374 才有三态），那 HC273 输出要经三态门挂总线，需要 HC244。

**问题**：HC273 不是三态输出。要让 A/B/RA/IR 等寄存器共享 8 位总线 DB[7:0]，必须：
- 方案 a：用 HC374（三态 D 触发器）替代 HC273 → 多花片数（HC374 比 HC273 贵）
- 方案 b：HC273 输出经 HC241/HC244 三态缓冲挂总线 → 每个寄存器配 1 片 HC244（不现实，太占片数）
- 方案 c：HC273 输出经 HC157（4×2 选 1）选通到总线 → 用 MUX 代替三态，每片选 1 个源

**这是关键设计点**——总线仲裁方式决定片数。

## 八、总线仲裁（决定片数的关键）

### 萤火虫原版做法（参考）

萤火虫原版用 HC16374（**16 位三态 D 触发器**，5 片）+ HC16244（16 位三态缓冲）。每个寄存器都是三态输出，挂公共总线，HC138 选谁驱动。

但 miniglow 用 HC273（非三态），且分立 8 位器件，每个寄存器配三态缓冲要 1 片 HC244，太贵。

### miniglow 做法：MUX 总线（HC157）

用 HC157（4 位 2 选 1 MUX）选通源到总线：
- 8 位总线 = 2 片 HC157（每片 4 位 2 选 1）
- 但 HC157 是 2 选 1，多个源要级联（4 选 1 = 3 片 HC157）

或者：**用 HC138 译码选源，所有寄存器输出经线或（OC 门）/ 或三态门挂总线**。

**最干净的做法**：**直接用 HC374 替代 HC273**。HC374 是三态输出 D 触发器，自带 /OE。这样：
- 每个寄存器 1 片 HC374（同片数）
- /OE 由 HC138 控制，选谁驱动总线
- 不需要额外的 HC244

**修正：HC273 全换 HC374**（片数不变，但功能升级为三态）。

**HC244 不要了**（HC374 自带三态）。
