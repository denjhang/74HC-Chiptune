# MiniGlow ISA 子集（基于萤火虫原版精简）

> 创建时间：2026-07-05
> 设计原则：**适度精简 + 指令集复用 glowcc**
> 项目目标：**通用音源主板** —— CPU 是固定基础，协处理器可换/可加（模块化）
>   - CPU 只做调度 + ADSR + 序列（整数加减比较够）
>   - 所有乘除/查表交协处理器（wave-mix / PSG / FM op-chain 等可选模块）
>   - CPU 通过标准化 IO 接口操作协处理器，不关心它内部算什么
> 砍掉标准：STC 移植代码（ADSR/序列/调度）完全用不上 + 萤火虫原版冗余

## 一、精简依据（先看 STC 代码用什么）

调度核跑的是 STC_Chiptune 的 wt.c / scc.c 等。分析这些代码的运算需求：

```c
// wt.c env_tick（典型 ADSR）：
cnt = env_cnt; step = env_step;
if (cnt >= step) { env_cnt = cnt - step; return; }
env_cnt = 250; lvl = level;
switch (env_state) {
  case 1: lvl++; if (lvl >= 31) {...} level = lvl; break;
  case 2: if (lvl > 0) lvl--; ... break;
  ...
}

// wt.c render（典型调度）：
pos += step; wt_ch[ch].pos = pos;
idx = (pos >> 5) & 0x7F;
wave_val = wt_waves[(wi << 7) | idx];   // ← 这部分交协处理器
```

**STC 调度代码的运算特征**：
- ✅ 8/16 位整数加减（ADSR 计数、phase 累加）
- ✅ 比较 + 分支（if/switch）
- ✅ 数组读写（寄存器组、变量）
- ✅ 位移（`>> 5`、`<< 7`）—— 但萤火虫靠 ALU 实现
- ❌ **不用 64 位（long long）**
- ❌ **不用浮点（double）**
- ❌ **不用乘法**（ADSR 不乘，乘音量交协处理器）
- ❌ **不用除法/取模**

## 二、机器码精简表（对照完整萤火虫 ISA）

完整萤火虫 ISA 见 `Glowworm-1/docs/isa.md`。下表标 **保留 ✅ / 砍掉 ❌ / 改造 ⚠️**：

### 总线节点（11 → 8）

| 节点 | 完整版 | miniglow | 理由 |
|------|--------|----------|------|
| RF | ✅ | ✅ 保留 | 寄存器堆，必需 |
| A | ✅ | ✅ 保留 | 累加器，必需 |
| **A0** | ✅ | ❌ 砍 | 数据 RAM 寻址低字节——miniglow 用 RA+段寄存器寻址，不需要 |
| **A1** | ✅ | ❌ 砍 | 同上 |
| **A2** | ✅ | ❌ 砍 | 同上 |
| RA | ✅ | ✅ 保留 | RF/段内寻址指针，必需 |
| B | ✅ | ✅ 保留 | ALU 第二输入，保留（ADD/SUB 需要） |
| RAM | ✅ | ✅ 保留 | RAM 数据口，必需 |
| IO0 | ✅ | ✅ 保留 | 协处理器接口，必需 |
| **IO1** | ✅ | ❌ 砍 | 一个 IO 口够（IO0 接协处理器寄存器组）|
| ALU | ✅ | ✅ 保留 | 查表 ALU，必需（萤火虫精髓）|

**砍 3 个节点（A0/A1/A2/IO1）**——剩 8 个。

### 数据寻址方式（关键改动）

完整版用 `A2A1A0` 24 位地址访问大 RAM。miniglow **砍 A0/A1/A2**，改用：

```
SEG 寄存器（A18-A16，3 位段号）+ RA（A7-A0，段内 256 字节偏移）
  → 拼 19 位地址：{SEG, 7'b0, RA} 不对，应该是 {SEG, RA, 8'b?}
```

**等等——这里要重新想清楚地址拼接**：

628512 是 19 位地址 = 8 段 × 64KB（段 3 位）× ... 不对，19 位 = 3 位段 + 16 位段内 = 8 × 64KB = 512KB ✓

但 RA 只有 8 位（256 字节段内偏移），不够 16 位段内偏移。

**两个解法**：
1. RA 扩到 16 位（2 片 HC273）—— 段内 64KB 寻址，配 3 位 SEG = 19 位全空间
2. RA 8 位 + 段寄存器扩到 11 位（段号 11 位 × 256 字节 = 512KB）

**选解法 1**（RA 16 位）—— 萤火虫原版 RA 是 8 位，扩到 16 位改动小，且 16 位 RA 配 3 位 SEG = 19 位地址，结构清晰。

> 决策：**RA 扩到 16 位**（2 片 HC273），SEG 3 位（1 片 HC161 或半个 HC273），拼 19 位地址访问 628512。

## 三、ISA 变化总结

### 砍掉
1. **节点**：A0、A1、A2、IO1（4 个）
2. **指令**：所有以 A0/A1/A2/IO1 为目的或源的（约 30+ 条机器码）
3. **ALU 模式**：MUL_L、MUL_H、DIV、MOD（4 个，乘除交协处理器）

### 改造
1. **RA 扩到 16 位**（原 8 位）→ 立即数/RF/IO → RA 的指令要分两次（低/高字节）
2. **跳转目标改用 ZP 寄存器**（16 位，原 A2A1A0 24 位）→ 跳转指令前先设 ZP
3. **新增 SEG 寄存器**（3 位段号）→ 新增 `SEG = imm` 指令

### 保留（指令集复用 glowcc 的核心）
- 所有立即数 / ALU / RF / RAM / IO0 互传指令（机器码 00/01/05/06/08/0A/10/11/15/16/18/1A/21/25/26/28/29/2A/31/35/36/39/3A/41/45/46/48/49/4A）
- ALU 的 ADD/SUB/CMP/AND/OR/XOR/NOT/移位/自增自减（22 种）
- 跳转 07XX/17XX（语义不变，目标改 ZP）
- NOP FFFF

## 四、glowcc 改动评估

miniglow 复用 glowcc，但要做这些后端改动：

| 改动 | 工作量 | 影响 |
|------|--------|------|
| 删除 A0/A1/A2 寄存器分配 | 中 | 代码生成不再用 A0/A1/A2 |
| 删除 IO1 相关 | 小 | 一处 |
| RA 改 16 位（代码生成分两次写）| 中 | LCC 后端 RA 操作改 |
| 跳转目标改 ZP（不是 A2A1A0）| 中 | PC_ASGN_A2A1A0 等函数重写 |
| 新增 SEG 寄存器 + 段切换 | 中 | 地址空间管理改造 |
| 砍 long long / double 支持 | 大 | 但能简化大量代码（移除 64 位/浮点 ALU 表生成）|

**总工作量**：中（1-2 周细致改动）。但改完 glowcc 仍能编译 C 代码（STC 的 ADSR/序列），只是不支持 64 位/浮点。

## 五、片数估算（基于本 ISA + 项目目标）

### ALU 实现决策：HC283 + HC85（不查表 ROM）

**理由**：CPU 当通用音源主板的调度器，**所有乘除交协处理器**，CPU 永远不碰乘除。所以 ALU 只要能做：
- ADD / ADD_C（16 位加法进位链，ADSR/phase 必需）
- SUB / SUB_C（同上）
- EQUAL_C（比较）

这几个用 **HC283（4 位加法器）×2 + HC85（4 位比较器）+ 几个门** 实现，1 拍出结果。**不需要 ALU ROM 查表**（萤火虫精髓里的查表 ALU 在这里反而冗余——查表的本质是"用 SRAM 当多功能 ALU"，但迷你版 ALU 功能极简，HC283 直接实现更省）。

### 片数明细

```
【628512 SRAM】            = 1 片  (512KB，分 8 段：程序/参数/RF/输出/协处理私有/预留)
【PC 16 位】HC161 ×4        = 4 片  (16 位程序计数器)
【RA 16 位】HC273 ×2        = 2 片  (16 位数据指针，配 SEG 拼 19 位)
【SEG 寄存器】复用某 HC273 的 3 位 = 0 片 (附在 PC 或 RA 旁)
【A、B 寄存器】HC273 ×2     = 2 片
【IR + 临时锁存】HC273 ×1   = 1 片
【指令译码】HC138 ×2        = 2 片
【ALU】HC283 ×2 + HC85 ×1   = 3 片  (8 位 ADD/SUB + 比较)
【控制门】HC00/HC04         = 1 片
【总线驱动】HC244 ×1        = 1 片

合计：~14 片核心 IC（接近目标 12-15）
```

加晶振/去耦/连接器/协处理器接口插座 ≈ 18 片 BOM（CPU 部分）。

### 协处理器模块化（"音源显卡"）

CPU 板提供**标准化协处理器插槽**，可插入不同协处理器模块：

```
┌─────────────────────────────────────┐
│ 协处理器模块插槽（CPU 板上的排针）    │
│                                     │
│  数据总线 D0-D7          (双向)      │
│  CP_TYPE[2:0]            (协处理器类型，模块上拉/下拉告知 CPU)│
│  CP_REG_INDEX[3:0]       (寄存器索引)│
│  CP_REG_DATA[7:0]        (寄存器数据)│
│  CP_START                (启动信号)  │
│  CP_STATUS               (完成标志)  │
│  CP_OUT_L/H              (输出)      │
│  GND / VCC / CLK                     │
└─────────────────────────────────────┘

可选模块：
  - wave-mix 协处理器（覆盖 SCC/WT）：~10 片
  - PSG 计数器协处理器（覆盖 SN/AY/SAA/NES/GB）：~10 片
  - FM op-chain 协处理器（YM2413）：~6 片
  - 用户自制协处理器（开放标准，可扩展）
```

**模块化的好处**：
- CPU 板设计一次定型，迭代协处理器不动 CPU
- 不同协处理器可以共存（多个插槽）或单独使用
- 协处理器出问题不影响 CPU 调试

## 六、设计决策已定（基于"协处理器可变"项目目标）

| 决策点 | 结论 | 理由 |
|--------|------|------|
| ALU 实现方式 | **HC283 + HC85**（不查表 ROM）| CPU 不算乘除，简单 ALU 用硬件直接实现更省 |
| 跳转目标 ZP | **16 位**（不跨段）| CPU 程序在段 0，跨段由数据访问处理 |
| glowcc | **另起 miniglowcc**（专精调度核）| 保留原 glowcc 给完整萤火虫 |
| ALU ROM 段 | **释放**（不分配）| 用不到查表 ALU |
| 协处理器接口 | **标准化排针**（CP_TYPE/REG/START/STATUS/OUT）| 模块化，可换可加 |

### CPU↔协处理器接口协议（标准化）

CPU 不管接什么协处理器，都通过这组寄存器操作：

```c
// CPU 视角（IO0 寄存器组）
#define CP_TYPE     (*((u8*)0x00))  // 只读：协处理器类型（上电时 CPU 探测）
#define CP_REG_IDX  (*((u8*)0x01))  // 读写：寄存器索引
#define CP_REG_DATA (*((u8*)0x02))  // 读写：寄存器数据
#define CP_START    (*((u8*)0x03))  // 写 1 启动
#define CP_STATUS   (*((u8*)0x04))  // bit0 = DONE
#define CP_OUT_L    (*((u8*)0x05))  // 输出低字节
#define CP_OUT_H    (*((u8*)0x06))  // 输出高字节

// 通用操作流程（CPU 不关心 CP_TYPE 是什么）：
void coproc_set_param(u8 idx, u8 data) {
    CP_REG_IDX = idx;
    CP_REG_DATA = data;
}

u16 coproc_run() {
    CP_START = 1;
    while (!(CP_STATUS & 1));
    return (CP_OUT_H << 8) | CP_OUT_L;
}

// 不同协处理器的参数语义不同，CPU 程序按 CP_TYPE 分支：
switch (CP_TYPE) {
    case CP_WAVE_MIX:  // SCC/WT
        coproc_set_param(0, step_l);
        coproc_set_param(1, step_h);
        coproc_set_param(2, vol);
        sample = coproc_run();
        break;
    case CP_FM:
        coproc_set_param(0, phase_m_l);
        // ... FM 专用参数
        sample = coproc_run();
        break;
}
```
