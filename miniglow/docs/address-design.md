# MiniGlow 地址空间与寄存器设计（关键决策）

> 创建时间：2026-07-05
> 状态：**决策中** —— 这是 ISA 设计的前提，必须先定

## 一、确认的硬件基础

### 628512 SRAM（已查 datasheet 确认）

- **容量**：512K × 8 bit = **512 KB**（4 Mbit）
- **地址线**：A0-A18（19 位）
- **数据线**：DQ0-DQ7（8 位双向）
- **封装**：DIP-32（也有 SOP/TSOP）
- **控制**：CE（片选）、OE（输出使能）、WE（写使能）
- 模型已建：`rtl/hc628512.v`

### 用量决策：1 片 628512（用户拍板）

512KB 单片分地址段使用，A18-A16 做段选择：

```
A18 A17 A16 | 段 | 地址范围      | 容量  | 用途
------------|----|--------------|------|------------------
 0  0  0   | 0  | 00000-0FFFF  | 64KB | 程序 ROM
 0  0  1   | 1  | 10000-1FFFF  | 64KB | ALU 表 / 或扩展
 0  1  0   | 2  | 20000-2FFFF  | 64KB | RF + 全局变量
 0  1  1   | 3  | 30000-3FFFF  | 64KB | 通道参数缓冲
 1  0  0   | 4  | 40000-4FFFF  | 64KB | 波形表预存区
 1  0  1   | 5  | 50000-5FFFF  | 64KB | 预留（乐谱/采样）
 1  1  x   | 6-7|               | 128KB| 预留
```

段译码：1 片 HC138（3-8 译码）把 A18-A16 译成 8 个段选信号。

## 二、核心矛盾：谁来产生 19 位地址？

萤火虫原版有 PC（24 位取指）+ A2A1A0（24 位数据寻址）+ RA（8 位 RF 寻址）。迷你版砍了 A0/A1/A2，**数据寻址没了寄存器**。

### 出路对比

| 出路 | 数据寻址寄存器 | 段切换 | 片数 | 限制 |
|------|-------------|--------|------|------|
| **1. RA 扩到 16 位** | RA 16 位（2 片 HC273）+ 段寄存器 1 片 HC161 | 改段寄存器 | 3 片 | RA 16 位寻址 64KB（段内），切段靠段寄存器 |
| **2. PC 复用** | PC 19 位（取指+数据共用）| 加模式位 | 0（PC 已有）| 取指和数据严格分时，ISA 加模式切换 |
| **3. RA 8 位 + 段寄存器** | RA 8 位（1 片 HC273）+ 段寄存器 1 片 HC161 | 改段寄存器 | 2 片 | RA 段内只能 256 字节，访问大表要切段 |
| **4. ALU 不查表，砍 ALU 段** | 用 HC283+HC85 实现 ALU | RA 8 位够 | 1 片 HC273 | 砍了 ALU 表寻址压力，但 ALU 功能受限 |

### 各出路的影响

**出路 1（RA 16 位）**：最像"小萤火虫"，数据访问灵活，但多 2 片 IC。
- 总片数评估：CPU 部分 ~14 片（比目标 12 多 2 片）

**出路 2（PC 复用）**：最省片子，但 ISA 复杂（模式切换），且取指和数据竞争 PC。
- 实际上 8 位机很多用这个（8051 的 DPTR 也是取指/数据共用思路）
- 但萤火虫原版不这么做，偏离架构

**出路 3（RA 8 位 + 段寄存器）**：最省片子（2 片），接近 8051 的 R0-R7 + 段寄存器模式。
- RA 段内 256 字节够 RF + 局部变量
- 访问波形表/ALU 表（跨段）要"切段"指令
- 切段开销：每次切段 2-3 拍，调度核主要在段 2（RF）工作，偶尔切段 1（ALU）或段 4（波形），可接受

**出路 4（砍 ALU 表，用 HC283）**：
- 萤火虫精髓之一是"查表 ALU"，砍了就不太像萤火虫
- 但 HC283 + HC85 也能做 ADD/SUB/CMP，只是没 MUL/DIV（迷你版本来就不要 MUL/DIV）
- ALU 段释放 → 数据空间更宽裕

## 三、组合方案

| 组合 | 描述 | CPU 片数 | 评价 |
|------|------|---------|------|
| **A. 出路 3 + 出路 4** | RA 8 位 + 段寄存器 + ALU 用 HC283（不查表）| ~10 片 | 最省，但偏离查表 ALU 精髓 |
| **B. 出路 3 + 保留 ALU 表** | RA 8 位 + 段寄存器 + ALU 查表（在段 1）| ~11 片 | 平衡——查表 ALU 保留，段切换开销小 |
| **C. 出路 1 + 保留 ALU 表** | RA 16 位 + 段寄存器 + ALU 查表 | ~13 片 | 最像小萤火虫，灵活但片多 |

## 四、我的倾向：方案 B

**RA 8 位 + 段寄存器 + 保留 ALU 表**：
1. 片数 ~11（接近目标 10-12）
2. 保留萤火虫"查表 ALU"精髓
3. 段切换开销可接受（主要工作在段 2，偶尔切段 1/4）
4. RA 8 位够 RF + 变量（256 字节）
5. 数据宽表（波形）跨段访问靠"切段 + 偏移"指令

### ISA 增加的指令（vs 萤火虫原版）

- `SEG = imm`（设段寄存器 A18-A16）—— 新增
- `RA = imm` / `A = RF[RA]` / `RF[RA] = A`（段内寻址，沿用萤火虫）
- ALU 查表：`SEG = 1; RA = alu_offset; A = RAM[RA]; ALU_op...`（切段 + 设 RA + 读）

### 寻址示例

```c
// 访问 RF（段 2，常态）
SEG = 2;
RA = 0x10; A = RF;   // 读 RF[0x10]

// 查 ALU 表（段 1）
SEG = 1;
RA = alu_addr;        // ALU 表内偏移
A = RAM;              // 读 ALU 结果
SEG = 2;              // 切回 RF 段

// 读波形表（段 4）
SEG = 4;
RA = wave_offset;
sample = RAM;
SEG = 2;
```
