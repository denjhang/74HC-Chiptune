输出锁存	74HC574 (8-bit锁存)	1

低通滤波	RC 滤波	-

核心其实就三步循环：ACC += STEP → ROM[ACC高8位] → 锁存输出 → DAC

累加器也可以省掉 4 颗 161，直接用 2 颗 74HC574（锁存）+ 4 颗 283（加法）组成，让 ROM 地址线 = 锁存器输出，每 CLK 周期：STEP + 当前值 → 锁存 → ROM 地址更新。

最精简可以压到 7 颗 IC：4x 283 + 1x 574 + 1x 28C64 + 1x 04。

# 74系列门电路 WT 合成器计划

## 目标

用 74 系列逻辑门 + SRAM + ROM 搭建最简 WT 波形表合成器，不依赖 MCU。

## 工具链

- **Verilog 编写**: 任意编辑器
- **综合**: [Yosys](https://github.com/YosysHQ/yosys) + [yosys74](https://github.com/roughengineer/yosys74) (74系列门库)
- **仿真**: [Icarus Verilog](https://github.com/steveicarus/iverilog) + GTKWave
- **流程**: Verilog → Yosys 综合到 74xx 网表 → Icarus Verilog 仿真验证 → 手动布线/原理图

## WT 核心逻辑

```
每 CLK 周期:
  ACC += STEP          // 16-bit 相位累加器
  ADDR = ACC[15:8]     // 高 8 位作为 ROM 地址
  OUT  = ROM[ADDR]     // 查表输出
  DAC  = R-2R(OUT)     // 8-bit DAC
```

## 硬件估算 (单通道)

| 功能 | 元件 | 数量 |
|------|------|------|
| 相位累加器 (16-bit ADD+REG) | 74HC283 x4 + 74HC574 x1 | 5 |
| 波形 ROM | 28C64 (8Kx8 EEPROM) | 1 |
| 时钟振荡 | 晶振 + 74HC04 | 1 |
| DAC | R-2R 电阻网络 (9个电阻) | - |
| 低通滤波 | RC 滤波 | - |
| **合计** | | **7 颗 IC** |

## 元件映射 (Verilog → 74 门)

| Verilog 构造 | 74 系列实现 |
|-------------|------------|
| `always @(posedge clk) reg <= val` | 74HC574 (8-bit 锁存) |
| `assign sum = a + b` | 74HC283 (4-bit 加法器) |
| `rom[addr]` | 28C64 / 28C256 (EEPROM) |
| `sram[addr]` | HM62256 (32Kx8 SRAM) |
| `assign y = a & b` | 74HC08 (AND 门) |
| `assign y = a \| b` | 74HC32 (OR 门) |
| `assign y = ~a` | 74HC04 (NOT 门) |
| `assign y = sel ? a : b` | 74HC157 (MUX) |
| 计数器 | 74HC161 (4-bit 同步计数器) |

## 开发步骤

1. 搭建工具链 (Yosys + Icarus Verilog + yosys74)
2. 写 WT 核心 Verilog (相位累加器 + ROM 查表)
3. Icarus Verilog testbench 验证输出波形
4. Yosys 综合到 74xx 网表，确认资源用量
5. 扩展: 多通道、包络、频率控制接口
6. 面包板/PCB 原型验证

Gigatron是如何设计出来的，他们应该有专门的综合工具吧？还是说本身就是资深老工程师，直接靠徒手？
Gigatron 的设计者 Marcel van Kervinck 和 Walter Belgers 都是资深工程师。Marcel 是荷兰计算机科学家，Walter 是信号处理专家。

他们的做法其实比较"手工"：

没有用综合工具，直接画逻辑图 + 手写汇编
核心就是一块 AT28C256 ROM（指令解码+控制信号全部存在 ROM 里），加上几颗 74HC 逻辑芯片做 PC、ALU、寄存器
本质是一台 ROM 微码控制的 TTL 计算机，不是传统 CPU 设计流程
所有时序都是人脑算的，用 Excel 表格规划微码
他们能这么做是因为：

Gigatron 架构极其精简（8-bit，哈佛结构，单总线）
指令集只有十几种
控制逻辑全部压缩到 ROM 查表里，不需要复杂的门级设计
对于我们的 WT 合成器来说，复杂度比 Gigatron 低很多 — 不需要指令解码、不需要 ALU，就一个累加器 + ROM 查表。用 Yosys + yosys74 综合完全够了，不需要徒手。

他们架构很牛逼啊，底层就是个8位cpu，好像只有七条指令集还是多少，然后模拟16位vcpu，
对，Gigatron 的设计非常精妙：

硬件层：8-bit TTL CPU，~30 颗 74HC 芯片，指令集只有 6 条（LD, AND/OR/XOR, ADD, ST, JMP, OUT）
vCPU 层：用 ROM 里的微码解释执行 16-bit 虚拟指令，在 8-bit 硬件上模拟 16-bit 运算
巧妙之处：硬件只做最简单的事（取指、解码、执行），复杂逻辑全靠软件层在 vCPU 上跑
我们的项目里 Gigatron 音源就是移植的这个 4 通道波形合成部分。他那个完整的 TTL 计算机才是真正的工程艺术 — 用最少的门电路实现了一台能跑 BASIC、显示图形的计算机。

Gigatron 的核心思路：硬件极简，软件（微码/ROM）做复杂的事。

映射到纯硬件 WT 合成器：

Gigatron 教我们的：

ROM 不只存波形，还能存控制逻辑 — 他用 ROM 做指令解码器，我们可以用 ROM 做包络曲线、甚至是简单的 ADSR 状态机查表
加法器 + ROM 查表 = 万能 — 他的 ALU 本质上就是 ROM 查表，我们的频率步进也是加法器 + ROM
时分复用 — 他用一套硬件分时模拟多寄存器，我们可以用一套累加器 + ROM 分时服务多通道
如果只做声音，最简方案：


硬件（~10颗IC）:
  1x 计数器 (节拍/通道切换)
  4x 加法器 (相位累加)
  1x 锁存器 (当前相位)
  1x 波形ROM (波形数据)
  1x 包络ROM (ADSR查表)
  1x 控制ROM (通道调度+状态机)
  1x 加法器 (混音累加)
  1x 时钟
关键启发：用控制 ROM 代替随机逻辑。不需要为 ADSR 状态机画一堆门电路，把状态转移表烧进 ROM，用计数器当地址，输出就是控制信号。这样硬件数量可以压到最低。
