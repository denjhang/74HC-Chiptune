## 一句话定位

PSG3 v0.5 rev.d 已验证 ROM 查表波形完美 (256 点, 256 种波形). PSG4 v0.6 在此基础上
**扩展通道数** (方波×N + 波形查表×N + LFSR 噪音), 并探索模拟乘法调制.
**不碰 PCM** (rev.e3 验证 PCM 在面包板有硬上限, 战略剔除).

---

## ⚠️ 基于 PSG3 v0.5 的哪些结论 (必读)

PSG4 必须建立在 PSG3 已验证的结论上, 不要重走弯路:

| 结论 | 来源 | PSG4 应用 |
|------|------|----------|
| ROM 查表 = 任意波形最佳方案 | rev.d ✅ 上板 | 波形通道一律 ROM 查表, 不用分频链/CD4029 |
| HC161 地址计数器 → ROM → DAC | rev.d ✅ | 波形通道核心架构不变 |
| **ROM 直连 DAC, CS=GND (不接 clk)** | rev.e3 ✅ 上板 | DAC 透明模式, 不用 HC273, 不用 clk |
| HC161 TC 直连任何 TTL 不需整形 | rev.e2/e3 ✅ | 级联/ROM 地址/计数器使能, 全直连 |
| HC161 Q 不能直 DAC (分频抽头) | rev.a ✅ | 波形数据必须来自 ROM, 不能用 Q |
| 高频不用 CD 系列 | rev.a ✅ | 一律 HC/HCT |
| PCM 在面包板有硬上限 | rev.e3 ✅ | **PSG4 不做 PCM** |
| period reload 天然变速扫描 | rev.d ✅ | 频率控制不变 (HC161 计满回零) |

> 完整发现记录: `PSG3 v0.5/docs/findings-2026-07-14.md` (12 节, 含铁律 + PCM 实验).

## PSG4 vs PSG3 的区别

| | PSG3 v0.5 (rev.d) | PSG4 v0.6 |
|---|---|---|
| 通道数 | 方波×1 + 噪音×1 + 波形×1 | **扩展 (方波×N + 波形×N + 噪音×1)** |
| 波形来源 | ROM 查表 256 点 ✅ | ROM 查表 (不变, 已验证) |
| DAC 接法 | CS=clk (rev.d 旧版) | **CS=GND (rev.e3 验证更干净)** |
| PCM | rev.e3 验证后剔除 | **不做** |
| 模拟调制 | 无 | **探索: MC1496/CA3080 乘法** (FM/AM/环形) |
| 总线 | YM2413 (8 寄存器) | 扩展 (更多通道 → 更多 reg 或地址译码) |
| 波形可写 | 否 (ROM 固化) | ROM 固化为主 (简单可靠), SRAM 可写留探索 |

## 核心架构 (基于 rev.d, 不变)

```
3×HC161 (period12) → freq_tc → HC161 地址计数器 (8-bit) → ROM → TLC7524 (CS=GND) → 运放 → 喇叭
                                       ↑                    ↑
                                 TC 直连级联         SST39SF010 (256 点 × 256 波形)
                                 (不需整形)          reg5 全 8 位选波形页
```

**rev.d 验证过的完整链路**, PSG4 每个波形通道复用此架构, 只扩展通道数.

---

## 扩展通道的方向

PSG3 是 3 通道 (方波/噪音/波形). PSG4 扩展:

### 方向 1: 多波形通道 (最直接)
- 复制 rev.d 波形通道 ×N (各自独立 period/vol/wave_sel + ROM + DAC)
- 代价: 每通道 ~8 片 (3 HC161 period + 2 HC161 地址 + 1 ROM + 2 DAC)
- 4 波形通道 = ~32 片 + 总线译码
- ROM 可共享 (一片 ROM, 地址高位区分通道) 或独立 (各通道一片 ROM)

### 方向 2: 模拟乘法调制 (MC1496/CA3080)
- 波形通道输出 → MC1496 做 FM/AM/环形调制 → 运放 → 喇叭
- 两路波形 (载波 + 调制) → MC1496 → 丰富音色, 不增加数字复杂度
- 库存: MC1496 ✅, CA3080 ✅ (见 inventory.md)

### 方向 3: WSG (波形查表专用, 不带方波/噪音)
- PSG = 方波 + 噪音为主, WSG = 纯波形查表
- WSG3 v0.6 已有设计 (NCO + TDM), 但 PSG4 走 period reload 更简单
- 两者可并存 (同一套总线)

> ⚠️ 以上都是**设计思路, 未验证**. 具体走哪个方向待定.

## 寄存器分配 (PSG3 总线扩展)

PSG3 用 reg0-5 (6 个). PSG4 扩展通道需要更多 reg:

| reg | 地址 | PSG3 用途 | PSG4 可能扩展 |
|-----|------|----------|--------------|
| reg0/1 | 0x01/0x02 | 方波 CH0 | 不变 |
| reg2 | 0x04 | 噪音 CH1 | 不变 |
| reg3/4/5 | 0x08/0x10/0x20 | 波形 CH2 (rev.d) | CH2 不变 |
| reg6-7 | 0x40/0x80 | 预留 | **CH3 波形 (period/vol/wave) 或地址译码扩展** |

> 8 寄存器 (独热码) 只够 2 方波 + 1 噪音 + 1 波形 (+ 预留 2).
> 更多通道要改总线地址译码 (HC138 译码, 从 8 扩到 256 地址), 或分页.

## DAC 接法铁律 (rev.e3 验证, PSG4 必须遵守)

- **ROM 查表 DAC: CS=GND (透明)**, 不接 clk.
- clk 适合分频链叠加 (rev.a), **不适合 ROM 查表/PCM**.
- ROM 输出直连 DAC DB0-7, 无需 HC273 锁存.
- 详见 findings 第 7 条.

---

## PCM 为什么剔除 (rev.e3 结论)

- 8kHz 采样率量化噪音大 (2-4x 才好听), 22kHz+ 才干净 (要换 39SF040).
- 面包板 ROM 高速查表耦合噪声 = 面包板音质上限 (要走 PCB).
- PCM 音量远小于波形 (RMS 差 3-10 倍, 要独立运放).
- 16-bit PCM 要 27C1024 + 双 TLC7524 + 差分放大 (成本高).
- **结论: PCM 留到专门项目 (PCB + 16-bit + 运放), PSG4 不做.**

> PCM 降噪改进方法 5 条 (22kHz / 16-bit / 串电阻 / PCB 隔离 / 差分放大) 见 findings 第十二节.
