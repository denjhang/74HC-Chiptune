# PSG4 v0.6 交接文档

> PSG4 v0.6 = PSG3 v0.5 波形通道升级 — DCO + RAM 波表查表 (避免 NCO)
> 创建时间：2026-07-10
> 状态：**架构设计阶段，待实现**

## 一句话定位

PSG3 v0.5 的波形通道（DCO 计算式波形）升级版——把 CD4029 的内置阶梯波形换成 **RAM 查表任意波形**。
频率控制仍然用 **period reload**（HC161 计满回零），不用 NCO 相位累加。
波形可实时写入 SRAM（和 WSG3 v0.6 的 PROM 固化不同）。

## 和 PSG3 v0.5 的区别

| | PSG3 v0.5 波形通道 | PSG4 v0.6 |
|---|---|---|
| 波形来源 | CD4029 计数器实时计算（阶梯=三角/锯齿） | **SRAM 查表（任意波形）** |
| 波形类型 | 方波/三角/锯齿/反锯齿（数学波形） | 任意（正弦/钢琴/弦乐/人声...） |
| 波形可写 | 否（固定 4 种） | **是（实时写 SRAM）** |
| 频率控制 | period reload (不变) | period reload (不变) |
| 加法器 | HC283 做比较 (不变) | HC283 做比较 (不变) |
| 核心改动 | — | CD4029 → 地址计数器 + SRAM |

## 核心架构

```
3×HC161 (period12 上计数) → freq_tc → 地址计数器 → SRAM[addr] → HC273 → TLC7524 → 运放
                                    ↑                ↑
                              HC161 级联        6116 存波形
                              (地址递增)        (32点/任意长度)
```

**关键改动：把 PSG3 的 "CD4029 Q → DAC" 换成 "地址计数器 → SRAM → HC273 → DAC"。**

```
PSG3 v0.5:  freq_tc → CD4029(Q0-3) → HC283比较/HC08 AND → TLC7524(DB)
                              ↑ Q值=波形幅度(阶梯)

PSG4 v0.6:  freq_tc → 地址计数器(A0-N) → SRAM[A] → HC273(保持) → TLC7524(DB)
                                    ↑                    ↑
                              HC161级联            SRAM出任意波形采样值
```

## 频率控制（不变，复用 PSG3）

```
freq = 4MHz / (波形步数 × (4096 - period12))
period12 = {reg4[7:4], reg3[7:0]} = 12-bit
两套 period 查找表: 锯齿族 / 三角 (PSG3 已有, 复用)
```

> PSG3 的 period reload 天然就是"变速扫描"波形表——永远完整走一遍波形,
> 高频时走得快, 低频时走得慢。不需要 NCO 的"变步进"。

## 地址计数器方案

波形表长度决定地址位数:
- 32 点 (SCC 兼容): 5-bit 地址 (HC161 ×1 的 Q0-4)
- 64 点: 6-bit (HC161 ×1 的 Q0-5)
- 256 点: 8-bit (HC161 ×2 级联)

**回绕逻辑**: 地址到波形末尾回 0, 用 HC283 比较器检测 + reload (和 PSG3 的 period 计数器同款机制)

```
HC161 ×N (地址计数器)
  ├── CP ← freq_tc (每 period 步走一次)
  ├── PE ← addr_tc (到末尾 reload 回 0)
  └── Q → SRAM 地址 A0-N

HC283 (比较器, 检测地址到末尾)
  ├── A ← addr (HC161 Q)
  ├── B ← 波形长度 (固定值或 reg 设置)
  └── A>=B → addr_tc → HC161 PE
```

## 波形数据格式

- SRAM 6116: 2K×8, 存波形采样值
- 8-bit 无符号 (0-255, 直接进 TLC7524 DB0-7)
- 或 4-bit (0-15, 进 DB4-7, 低 4 位接 GND) — 和 PSG3 兼容
- 波形写入: 总线写 SRAM (和写 reg 同协议, 地址译码区分)
- 可存多个波形 (2K 足够存 64 个 32 点波形)

## 波形写入接口

总线写波形数据时:
1. 选 SRAM 地址段 (reg 地址译码区分 "参数" vs "波形数据")
2. 总线 D0-7 → SRAM 数据口
3. 地址计数器输出 → SRAM 地址口 (播放时自动递增)
4. 或: 总线写地址 → SRAM 地址口 (写入时由 CPU 控制地址)

> 具体方案待定: 双口 RAM (CY7C130) 免仲裁, 或 HC157 mux 切换地址来源

## 寄存器分配 (复用 PSG3 总线)

| reg | 地址 | 用途 |
|-----|------|------|
| reg0/1 | 0x01/0x02 | 方波通道 (v0.4 不变) |
| reg2 | 0x04 | 噪音 (v0.4 不变) |
| reg3 | 0x08 | 波形 period[7:0] (PSG3 不变) |
| reg4 | 0x10 | 波形 period[11:8] \| vol (PSG3 不变) |
| reg5 | 0x20 | 波形长度/选择 (改: 原 duty/wave/mode → 波形号/长度) |
| reg6-7 | 0x40/0x80 | **波形 RAM 写入** (新增) 或预留 |

## 和 WSG3 v0.6 的关系

| | PSG4 v0.6 | WSG3 v0.6 |
|---|---|---|
| 频率控制 | period reload (HC161 计满回零) | NCO (acc += freq) |
| 加法器 | **不要** | 要 (HC283 做 NCO) |
| RAM | **SRAM 存波形** (可写) | LS189 存 acc/freq 状态 (不可写波形) |
| 微码 ROM | **不要** | 要 (TDM 时序) |
| 波形 | SRAM 任意可写 | PROM 固化 |
| 核心芯片 | ~16 片 | 10 片 |
| 原理 | 简单 (并行 period) | 复杂 (串行 TDM) |
| 音色丰富度 | 高 (任意波形) | 低 (8种固定) |

两者用**同一套总线接口**（PSG3 v0.5 总线层），可以并存。

## PSG5 展望 (后续)

PSG4 (DCO + RAM 波表) 之后:
- PSG5 = PSG4 + 模拟调制 (MC1496/CA3080)
- DCO 出波形 → MC1496 做 FM/AM/环形调制 → 运放 → 喇叭
- 声音可复杂很多, 但不需要 NCO/相位累加
- 模拟调制芯片不增加数字复杂度
