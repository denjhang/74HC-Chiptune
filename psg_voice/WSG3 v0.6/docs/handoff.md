# WSG3 v0.6 交接文档

> WSG3 v0.6 = Pac-Man Namco WSG 功能等效复刻 (10 片核心 + 复用 PSG3 总线接口)
> 创建时间：2026-07-10
> 状态：**架构设计阶段，待实现**

## 一句话定位

复刻 1980 年 Pac-Man Namco WSG 的核心算法——3 通道波表合成，4-bit 加法器分时做 20/16-bit 相位累加，PROM 查波形表。
**总线接口复用 PSG3 v0.5 已验证的 YM2413 总线层**，不重新设计，接口层永不膨胀。

## 和旧版 wsg3 的区别

| | 旧版 wsg3（已放弃） | WSG3 v0.6 |
|---|---|---|
| 总线接口 | 自己设计 SPFM 接口 → 5-7 片隐藏门 | **复用 PSG3 v0.5 总线层（13 片，已验证）** |
| 隐藏门 | ~60 个（cp273/we_n/clr 等 TDM 控制逻辑） | **目标 0**（所有逻辑实例化） |
| 芯片膨胀 | 14→19→21 片（停不下来） | 核心 10 片 + 总线 13 片 = **固定 23 片** |
| TDM 控制 | assign 三元表达式（隐藏门） | 微码 ROM 直出控制信号（无隐藏门） |

## 核心架构（10 片）

```
总线层 (复用 PSG3 v0.5, 13 片, 已验证)
    │ reg_addr[7:0] + reg_data[7:0] + rst_n
    ▼
┌──────────────────────────────────────────────┐
│ WSG 核心 (10 片)                              │
│                                              │
│  HCNT 计数器 (HC161×1, 6-bit)                │
│    │ tdm_step[3:0] + sub_cyc[1:0]            │
│    ▼                                         │
│  微码 ROM (SST39SF040, 16步×16bit 控制字)    │
│    │ 控制信号 (无隐藏门, ROM 直出)            │
│    ├──────────────────────────────────┐       │
│    │                                  │       │
│  74LS189 ×2 (acc RAM + freq RAM)     │       │
│    │ acc[3:0]    freq[3:0]           │       │
│    ▼            ▼                    │       │
│  HC283 (4-bit 加法器: acc+freq+carry)│       │
│    │ sum[3:0] + carry                 │       │
│    ▼                                  │       │
│  HC174 (6-bit 移位锁存, 累积进位)     │       │
│    │ phase[4:0] (高位当波形ROM地址)   │       │
│    ▼                                  │       │
│  波形 ROM (SST39SF040, 8波×32点×4bit)│       │
│    │ wave_sample[3:0]                 │       │
│    ▼                                  │       │
│  HC273 (输出锁存: wave+vol)           │       │
│    │                                  │       │
│    ▼                                  │       │
│  CD4066 / TLC7524 (音量乘法+DAC) ◄───┘ vol[3:0]│
│    │                                         │
│    ▼                                         │
│  模拟输出 → 运放 → 喇叭                       │
└──────────────────────────────────────────────┘
```

## 微码 ROM 时序（16 步 TDM，原版 Pac-Man）

| 步 | 通道 | 操作 | 微码 (16-bit) |
|----|------|------|--------------|
| 0-4 | ch0 | 频率 nibble 0-4 + 累加 nibble 0-4 (清零开始) | 1101_1110_1111_0111 (步0) / 1101_1110_1111_1111 (步1-4) |
| 5 | ch0 | **输出**: 查波形ROM × 音量 → 锁存 | 1011_1111_1111_1111 |
| 6-9 | ch1 | 频率 nibble 0-3 + 累加 nibble 0-3 | 同上 |
| A | ch1 | **输出** | |
| B-E | ch2 | 频率 nibble 0-3 + 累加 nibble 0-3 | |
| F | ch2 | **输出** | |

微码位定义（据技术文档）:
```
bit [15:12] = 1M ROM 控制 (1011=锁存输出, 1101=仅读)
bit [11:8]  = 273 CP 控制 (1110=上升沿, 1111=保持)
bit [7:4]   = 174 CLR 控制 (0111=清零, 1111=正常)
bit [3:0]   = 174 CLK 控制 (0111=锁存, 1111=保持)
```

## 芯片清单 (10 片核心)

| 位号 | 型号 | 功能 | 库存 |
|------|------|------|------|
| W1 | 74HC161 | HCNT 6-bit 计数器 (TDM 时序) | ✅ |
| W2 | SST39SF040 | 微码 ROM (16步控制字) | ✅ |
| W3 | 74LS189 | acc RAM (累加器值, 16×4) | ✅ |
| W4 | 74LS189 | freq RAM (频率值, 16×4) | ✅ |
| W5 | 74HC283 | 4-bit 加法器 (acc+freq+carry) | ✅ |
| W6 | 74HC174 | 6-bit 移位锁存 (进位链+相位) | ✅ |
| W7 | SST39SF040 | 波形 ROM (8波×32点) | ✅ |
| W8 | 74HC273 | 输出锁存 (wave+vol) | ✅ |
| W9 | 74HC157 | 地址 mux (CPU写 / TDM扫描) | ✅ |
| W10 | 74HC158 | 数据反相 mux (CPU写 / 加法回写) | ✅ |

> 音量乘法+DAC: 用 TLC7524 (库存有) 或 CD4066+R-2R (原版方式)
> 库存全部确认有 (见 docs/inventory.md, PSG3 v0.5 共享)

## 总线接口（复用 PSG3 v0.5）

**完全复用 PSG3 v0.5 的总线层（wiring-table-bus.md），不改任何接线。**
- 13 片: 2×LS373 + HC374(地址) + 8×HC374(reg) + HC04 + 3×HC08
- 地址分配: WSG 占 reg6/reg7 或新译码段 (具体待定)

> **接口层永不膨胀原则**: WSG 核心只接收 reg_data + reg_addr + rst_n,
> 不自己处理总线协议。所有 CS/WR/A0 时序在总线层完成。

## RAM 寄存器映射 (74LS189, 16×4)

| 地址 | U6 (acc) | U7 (freq/vol) |
|------|----------|---------------|
| 0-4 | ch0 累加器 (5 nibble) | ch0 频率 (20-bit) |
| 5 | ch0 波形号 (3-bit) | ch0 音量 (4-bit) |
| 6-9 | ch1 累加器 (4 nibble) | ch1 频率 (16-bit) |
| A | ch1 波形号 | ch1 音量 |
| B-E | ch2 累加器 (4 nibble) | ch2 频率 (16-bit) |
| F | ch2 波形号 | ch2 音量 |

## 波形数据

- PROM 82s126 原版: 8 波 × 32 点 × 4-bit = 256 字节
- 数据格式: **有符号 4-bit** (0-15 表示 -8~+7, MAME: `(data & 0x0F) - 8`)
- 用 SST39SF040 存 (256 字节填充到 512KB)
- 波形不能实时写 (PROM 固化), 这是 WSG 和 PSG4 的区别

## 和 PSG4 v0.6 的关系

| | WSG3 v0.6 | PSG4 v0.6 |
|---|---|---|
| 频率控制 | NCO (acc += freq, 4-bit分时累加) | period reload (HC161 计满回零) |
| 加法器 | 要 (HC283 做 NCO 累加) | **不要** |
| RAM | 要 (acc + freq 状态) | **不要** (374 寄存器) |
| 微码 ROM | 要 (TDM 时序控制) | **不要** (并行, 无 TDM) |
| 波形 | PROM 固化 (8种固定) | SRAM 可写 (任意波形) |
| 核心芯片 | 10 片 | ~16 片 |
| 原理复杂度 | 高 (串行 TDM + 移位累加) | 低 (并行 period reload) |

两者用**同一套总线接口**（PSG3 v0.5 总线层），可以并存。
