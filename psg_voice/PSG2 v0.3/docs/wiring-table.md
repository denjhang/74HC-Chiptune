## 二、占空比扩展原理

v0.2 的 toggle_q（U5 Q1）是 50% 方波（频率 f）。占空比扩展在它后面加分频链：

```
Q1 (U5, 50%, f) ──→ FF3 (U8 半片, ÷2) ──→ Q2 (50%, f/2)
Q2 ──→ FF4 (U8 另半片, ÷2) ──→ Q3 (50%, f/4)

AND 组合 (U9):
  duty_50  = Q1                    (原频, 50%)
  duty_25  = Q1 AND Q2             (f/2, 25%)
  duty_625 = Q2 AND Q3             (f/4, 25% 变体)
  duty_125 = duty_25 AND Q3        (f/4, 12.5%)  ← 复用 duty_25

U10 (HC153) 4选1: bit0-1 选 duty_50/25/125/625 → U7.REF
```

| 挡位 | bit0-1 | 占空比 | 频率 | 实测（A4 period=183）|
|------|--------|--------|------|---------------------|
| 0 | 00 | 50% | f（原频）| 438 Hz ✅ |
| 1 | 01 | 25% | f/2 | 219 Hz ✅ |
| 2 | 10 | 12.5% | f/4 | 110 Hz ✅ |
| 3 | 11 | 25% | f/4（变体）| 110 Hz ✅ |

> **特性**：占空比变窄时频率同步降低（每级÷2）。软件移调补偿（选窄挡时 period 减半回原音高）。
> **顺带扩展音域**：12.5%/25%@f/4 挡可下探到 C0（原 period 极限只能到 C3）。

## 三、U6 改动（HC374 加占空比位）

**v0.2 现状**（据 `psg_ft232h_v02.py` 驱动）：
- 软件：`self._c = vol & 0x0F` → 音量放 C 口低 4 位（C0-C3 = HC374 bit0-3）
- 硬件：HC374 Q0-Q3 → TLC7524 DB4-DB7

**v0.3 改动**：音量完全不动（bit0-3），占空比放 **bit4-5**（v0.2 这两位接地，现在启用）。

| Pin | v0.2 | v0.3 | 连接 |
|-----|------|------|------|
| 9 (D0) | 音量 bit0 | 音量 bit0（不变）| ← FT232H C0 |
| 8 (D1) | 音量 bit1 | 音量 bit1（不变）| ← FT232H C1 |
| 7 (D2) | 音量 bit2 | 音量 bit2（不变）| ← FT232H C2 |
| 6 (D3) | 音量 bit3 | 音量 bit3（不变）| ← FT232H C3 |
| 15 (D4) | GND | **占空比 bit0** | ← FT232H C4 |
| 14 (D5) | GND | **占空比 bit1** | ← FT232H C5 |
| 13 (D6) | GND | **mode_sel** | ← FT232H C6 |
| 12 (D7) | GND | **REF 选择 (ref_sel)** | ← FT232H C7 |
| 2 (Q0) | → U7.DB4 | → U7.DB4（不变）| 音量 |
| 3 (Q1) | → U7.DB5 | → U7.DB5（不变）| 音量 |
| 4 (Q2) | → U7.DB6 | → U7.DB6（不变）| 音量 |
| 5 (Q3) | → U7.DB7 | → U7.DB7（不变）| 音量 |
| 16 (Q4) | 悬空 | → **U10.A（HC153 地址 bit0）** | 占空比地址 |
| 17 (Q5) | 悬空 | → **U10.B（HC153 地址 bit1）** | 占空比地址 |
| 18 (Q6) | 悬空 | → **U11.S0（HC4053 开关 X 控制，Pin11）** | mode_sel |
| 19 (Q7) | 悬空 | → **U11.S2（HC4053 开关 Z 控制，Pin9）** | REF 选择 |

> ✅ **音量接线零改动**，v0.2 的 Q0-Q3→DB4-7 保持原样。
> 占空比 bit4-5（Q4-Q5）、mode bit6（Q6）、REF bit7（Q7）v0.3 新启用。

## 四、U8 — 74HC74（占空比分频链，新增）

双 D 触发器，两个都做 T 触发器（D=Q̄）分频。

| Pin | 信号 | 连接 |
|-----|------|------|
| 1 | /CLR1 | ← rst_n（或接 +5V，复位靠软件）|
| 2 | D1 | ← Q1_n（Pin 6，T 触发器接法）|
| 3 | CLK1 | ← **U5.Q1**（toggle_q，原方波输出）|
| 4 | /PRE1 | → +5V（不置位）|
| 5 | Q1 | → U9.A1（AND 输入，duty_25/duty_125 用）|
| 6 | Q1_n | → U8.D1（反馈）= Q2 信号 |
| 13 | /CLR2 | ← rst_n |
| 12 | D2 | ← Q2_n（Pin 8，T 触发器）|
| 11 | CLK2 | ← **U8.Q1**（U8 半片1 输出 Q2，级联）|
| 10 | /PRE2 | → +5V |
| 9 | Q2 | → U9.A2/A3（duty_625/duty_125 用）= Q3 信号 |

## 五、U9 — 74HC08（占空比 AND 组合，新增）

四 2 输入与门，用 3 个，第 4 门闲置（接 GND，输出悬空）。

**信号定义**（先说清来源）：
- **Q1** = U5（HCT74）Pin 9，v0.2 原 toggle 输出（50% 方波，频率 f）
- **Q2** = U8（新 HC74）Pin 5，Q1 经 FF3 ÷2（50%，频率 f/2）
- **Q3** = U8（新 HC74）Pin 9，Q2 经 FF4 ÷2（50%，频率 f/4）

| 门 | 输入 | 输出 | 含义 |
|----|------|------|------|
| 门1 (Pin 1,2→3) | Q1, Q2 | duty_25 | 25%@f/2 |
| 门2 (Pin 4,5→6) | Q2, Q3 | duty_625 | 25%@f/4 变体 |
| 门3 (Pin 9,10→8) | duty_25, Q3 | duty_125 | 12.5%@f/4（复用 duty_25）|
| 门4 (Pin 12,13→11) | GND, GND | 闲置 | 接 GND，输出悬空 |

| Pin | 信号 | 连接 |
|-----|------|------|
| 1 (A1) | ← U5.P9（Q1）| duty_25 输入1 |
| 2 (B1) | ← U8.P5（Q2）| duty_25 输入2 |
| 3 (Y1) | duty_25 | → U10.1C1 + U9.P9（门3 复用）|
| 4 (A2) | ← U8.P5（Q2）| duty_625 输入1 |
| 5 (B2) | ← U8.P9（Q3）| duty_625 输入2 |
| 6 (Y2) | duty_625 | → U10.1C3 |
| 9 (A3) | ← U9.P3（duty_25）| duty_125 输入1 |
| 10 (B3) | ← U8.P9（Q3）| duty_125 输入2 |
| 8 (Y3) | duty_125 | → U10.1C2 |
| 12 (A4) | → GND | 闲置 |
| 13 (B4) | → GND | 闲置 |
| 11 (Y4) | — | 悬空 |
| 14 | VCC | +5V |
| 7 | GND | GND |

> **第 4 门闲置**：与门只能做 AND，第 4 个 AND 想不出有用功能（75% 占空比要 OR/德摩根，HC08 做不了）。

## 六点五、U11 — 74HC4053（双 SPDT：REF 切换 + mode 切换，1 片搞定两个二选一）

三 SPDT 模拟开关，用 2 路（Z 切 REF，X 切 mode），Y 空闲。
**取代原 HC86**（mode 二选一）+ **承担 REF 切换**，1 片顶 2 片。

### 为什么能合并

HC86 原来做"tc_hi/pe_n 二选一"（mode 切换），HC4053 做"占空比变体/Q0 二选一"（REF 切换）。
**两个都是 2 选 1**，HC4053 有 3 路 SPDT（X/Y/Z），用 2 路刚好，省掉 HC86。

### 开关分配

| 开关 | 输入 0 (bit=0) | 输入 1 (bit=1) | 控制脚 | 输出 | 功能 |
|------|---------------|---------------|--------|------|------|
| **Z** | HC153 占空比变体 (1Y) | Q0 抽头 | S2 (Pin9) ← bit7 | Z-COM → U7.REF | REF 选择 |
| **X** | tc_hi | pe_n | S0 (Pin11) ← bit6 | X-COM → U5 7474 FF1.D | mode 选择（方波/白噪）|
| Y | 悬空 | 悬空 | S1 (Pin10) → GND | — | 空闲 |

### 从 v0.2 改装

**第 1 步**：拆 v0.2 的两根线：
- U5.Q1（toggle）→ U7.REF 直连（REF 路径）
- tc_hi → U5 7474 FF1.D 直连（1D 路径）

**第 2 步**：插 U11（74HC4053），接 2 路 SPDT：
- Z0（Pin12）← U10.HC153 1Y（占空比变体）
- Z1（Pin13）← Q0 抽头（HC161 计数器最低位）
- Z-COM（Pin14）→ U7.REF（TLC7524 Pin15）
- X0（Pin14... 见逐脚表）← tc_hi
- X1 ← pe_n（U4 HC00 第1路输出）
- X-COM → U5 7474 FF1.D（取代原 tc_hi 直连）
- S2（Pin9）← U6.Q7（bit7，REF 控制）
- S0（Pin11）← U6.Q6（bit6，mode 控制）
- S1（Pin10）→ GND（Y 开关不用）
- E（Pin6）→ GND（常开）
- VCC（Pin16）→ +5V，GND（Pin8）→ GND，VEE（Pin7）→ GND

### 工作方式

**bit7（REF）**：0=占空比变体，1=Q0 抽头
**bit6（mode）**：0=tc_hi（方波），1=pe_n（白噪）

> **U5（7474）FF1 接线改动**：原 v0.2 是 D1 ← tc_hi，现在 D1 ← **U11 X-COM**。
> FF1 用 clk 同步采样，消毛刺（v0.1 验证原理）。
>
> **⚠️ 白噪是硬件物理延迟现象**：pe_n 经 HC00 + HC4053 传输门延迟，7474 采样落点随 period
> 微妙变化 → 白噪。iverilog 仿真复现不了（理想零延迟），只能上板实测。
> 详见 design.md 5.8 节。

## 六点六、U11 — 74HC4053 逐脚表（双 SPDT）

| Pin | 信号 | 连接 |
|-----|------|------|
| 12 (Z0) | ← U10.HC153 1Y（占空比变体）| duty50/25/125/625（bit7=0 选）|
| 13 (Z1) | ← Q0 抽头（HC161 最低位）| 调制泛音（bit7=1 选）|
| 14 (Z-COM) | → U7.REF（TLC7524 Pin15）| REF 输出 |
| 9 (S2) | ← U6.Q7（bit7）| 开关 Z 控制 |
| 15 (X0) | ← tc_hi | 方波（bit6=0 选）|
| 1 (X1) | ← pe_n（U4 HC00 第1路输出）| 白噪（bit6=1 选）|
| 2 (X-COM) | → U5 7474 FF1.D（Pin2）| reload 源输出 |
| 11 (S0) | ← U6.Q6（bit6）| 开关 X 控制 |
| 4 (Y0), 5 (Y1), 3 (Y-COM) | 悬空 | 开关 Y 不用 |
| 10 (S1) | → GND | 开关 Y 控制（不用）|
| 6 (E) | → GND | 使能（低有效，常开）|
| 16 (VCC) | → +5V | |
| 8 (GND) | → GND | |
| 7 (VEE) | → GND | 不用负电源 |

> **bit 真值**：
> - bit7=0 → Z-COM = Z0 = HC153 占空比变体
> - bit7=1 → Z-COM = Z1 = Q0 抽头（乘法调制）
> - bit6=0 → X-COM = X0 = tc_hi（方波）
> - bit6=1 → X-COM = X1 = pe_n（白噪）

### 为什么用 HC4053（不是 HC153 / 4066 / CD4099）

- **HC153 数字多路器**：会整形边沿，吃掉 Q0 高频谐波 → 丢失泛音 ❌
- **CD4066 模拟开关**：要独热码控制，2 选 1 要非门（库存没多余非门）❌
- **CD4099 + 4066**：2 片配合太复杂 ❌
- **HC4053 三 SPDT 模拟开关**：1 位控制 2 选 1，天然互斥（SPDT 不会两路同通），
  模拟通路保真，**1 片搞定** ✅

## 七、U7 改动（TLC7524 REF + DB 输入）

| Pin | v0.2 | v0.3 |
|-----|------|------|
| 15 (REF) | ← U5.Q1（toggle_q 固定）| ← **U11.HC4053 Z-COM（2 选 1: 占空比变体/Q0）** |
| 4-7 (DB4-7) | ← U6.Q0-Q3（音量）| **不变**（← U6.Q0-Q3 音量码）|

其余不变（OUT1→喇叭，DB0-3/RFB/CS/WR→GND）。

> **数据流（修正后，自洽）**：
> - REF = HC4053 选 "占空比变体（HC153 输出）" 或 "Q0 抽头"
> - DB4-7 = HC374 音量码（Q0-Q3，v0.2 不变）
> - DB0-3 = GND（固定 0，v0.2 不变）
> - 输出 = REF × (音量码/256)（乘法衰减，音量控制生效，和 v0.2 一致）
>
> **占空比变体经 REF 路径**（不是 DB 路径）：duty50/25/125/625 都是 1-bit 方波，
> 接 REF 当波形参考，DB 接音量码衰减幅度。**音量控制方案不变**（v0.2 已验证）。
> bit7=1 时 REF 切 Q0 抽头（Q0 调制），DB 仍是音量码，音量照常衰减。

## 八、控制字格式（HC374，8 bit）

```
bit0-3: 音量 (16 级, Q0-Q3 → DB4-7, 与 v0.2 一致)
bit4-5: 占空比挡 (00=50% / 01=25% / 10=12.5% / 11=25%@f/4变体)
bit6:   mode (0=方波, 1=白噪)  ← HC4053 开关 X
bit7:   REF 选择 (0=toggle 纯净方波, 1=Q0 乘法调制)  ← HC4053
```

驱动写入：`self._c = (vol & 0x0F) | ((duty & 0x03) << 4) | ((mode & 1) << 6) | ((ref & 1) << 7)`

## 九、写时序（FT232H）

| 操作 | FT232H 动作 |
|------|------------|
| 写 period | C0-C7=period, D4(LE) 上升沿 → U1 锁存 |
| 写控制 | C0-C3=音量, C4-C5=占空比, C6=mode, D5(A0) 上升沿 → U6 锁存 |
| 选方波/白噪 | 写 bit6 = 0/1 |
| 选占空比挡 | 写 bit4-5 = 00/01/10/11 |
| 静音 | 写 bit0-3 = 0000 |
| 复位 | D6(RST) 拉低再拉高 |

### U16 — 74HC374（噪音控制寄存器）

A1（FT232H D7）选通，CP 上升沿锁存。

**控制字格式**（与方波风格统一：音量在 bit0-3）：
```
bit0-3: 音量 (16 级, Q0-Q3 → DB4-7)
bit4-5: 频率挡 (00=÷2 / 01=÷4 / 10=÷8 / 11=÷16, 仅独立模式)
bit6:   绑定开关 (0=独立分频 / 1=直通 square_tc)
bit7:   预留
```

| Pin | 信号 | 连接 |
|-----|------|------|
| 1 (/OE) | → GND | 常输出 |
| 11 (CP) | ← **A1（FT232H D7 经 LS373）** | 写脉冲 |
| 9 (D0) | ← FT232H C0 | 音量 bit0 |

### U4 借用 — HC00 剩余门（绑定 2 选 1 + RST 灌种）

v0.2 方波通道 U4（HC00）第 1 路（PE 反相）在用，**第 2/3/4 路空闲给噪音**。

**门 2（Pin 4,5→6）：RST 反相**（死锁防护灌种）
| Pin | 信号 | 连接 |
|-----|------|------|
| 4 (A2), 5 (B2) | ← rst_n（输入短接当反相器）| |
| 6 (Y2) | rst_n_inv（= ~rst_n）| → U4 门4（绑定 2 选 1 用）|

**门 3+4（Pin 9,10→8 + 1,2→3 复用？）**：绑定 2 选 1

绑定逻辑：`noise_clk = bind ? square_tc : ind_clk`
用 NAND 实现（德摩根）：`noise_clk = NAND(NAND(ind_clk, ~bind), NAND(square_tc, bind))`

需要：
- NAND(ind_clk, ~bind) — 1 门
- NAND(square_tc, bind) — 1 门
- NAND(两路, 两路) 汇总 — 1 门

但 HC00 只剩 3 路（门2/3/4），门2 已做 RST 反相，**只剩门3/4 两路不够 3 个 NAND**。

**修正方案**：RST 灌种不用独立反相器，直接用 `serial_in = (!rst_n) ? 1 : feedback` 的行为级（硬件上 serial_in 接到一个 OR：`feedback OR rst_n_inv`，rst_n_inv 可以和绑定共用反相）。

详细门级实现留到硬件调试，接线表先标功能。

| 门 | 功能 | 输入 | 输出 |
|----|------|------|------|
| 门1 (Pin 1,2→3) | PE 反相（v0.2 方波用）| tc_hi | pe_n |
| 门2 (Pin 4,5→6) | 绑定反相（~bind）| U16.Q6（bind_sw）| bind_inv |
| 门3 (Pin 9,10→8) | NAND(ind_clk, bind_inv) | ind_clk, bind_inv | n_ind |
| 门4 (Pin 12,13→11) | NAND(square_tc, bind) + 汇总 | 见下 | noise_clk |

> ⚠️ **门级细节**：HC00 剩 3 门做"绑定 2 选 1"需要 3 个 NAND（反相+两路NAND+汇总），但 RST 灌种也要反相。
> 实际硬件实现时，RST 灌种可以接到 HC164 的 MR 脚控制（RST 期间 MR=0 清零，释放后靠 serial_in 灌种），
> 或者 serial_in 逻辑合并到绑定 NAND 网络。**门级接线在硬件调试时 finalize**，RTL 行为级已验证功能正确。

## 十三、写时序（FT232H，完整）

| 操作 | FT232H 动作 |
|------|------------|
| 写 period | C0-C7=period, D4(LE) 上升沿 → U1 锁存 |
| 写方波控制（音量+占空比）| C0-C3=音量, C4-C5=占空比, D5(A0) 上升沿 → U6 锁存 |
| 写噪音控制（音量+频率+绑定）| C0-C3=音量, C4-C5=频率, C6=绑定, **D7(A1) 上升沿 → U16 锁存** |
| 复位 | D6(RST) 拉低再拉高 |

> ⚠️ **数据总线复用**：C0-C7 是方波和噪音共用的数据总线。写方波用 A0(D5) 选通 U6，写噪音用 A1(D7) 选通 U16，互不干扰（写时另一个的 CP 保持低，不锁存）。
