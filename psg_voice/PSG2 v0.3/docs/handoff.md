# PSG2 v0.3 交接文档

> PSG2 两通道 PSG（CH0 方波三层音色 + CH1 LFSR 白噪音）
> 交接时间：2026-07-03
> 状态：**CH0 方波通道上板验证通过（占空比/mode/REF 全功能正常），CH1 噪音通道 6 片定稿待上板**

## 🌍 项目独特性（全球搜证 2026-07-02）

**用最少 74 门做 PSG 原型并真上板，全球开源社区目前没有同类项目。**

- mengstr/Discrete-AY-3-8910: 104 片，仅仿真器画图，未做硬件
- rejunity/tt05-psg-ay8913: 1500-1600 门 FPGA/ASIC，非 74 离散逻辑
- 本项目：**7 片起步**（v0.2 已上板出声）→ **按需扩展**到 17 片

**与 yosys 综合路线本质对立**（Pepijn de Vos 等）：
- 他们的 breathing LED 就要 5 片，综合器不懂芯片级优化
- **本项目全实例化**：RTL 直接实例化真实 74 芯片，无隐藏门，74 TTL 高利用率
- iverilog 只做仿真，不做综合；架构由人定，AI 做验证

## 🎯 核心认知：仿真盲区 + 面包板发现价值（2026-07-02 实证）

**iverilog 是逻辑仿真器，不是电路仿真器**。以下音色现象从原理上无法模拟：

- **HC4053 白噪**（pe_n 进 7474）：HC00 真实传播延迟产物，仿真零延迟复现不了
- **Q 抽头乘法调制泛音**（Q0 进 TLC7524.REF）：HC161 输出模拟瞬态/非理想边沿，逻辑仿真看不到

**流程的第三条腿——面包板插针探索**：
1. iverilog 仿真：验证你想得到的逻辑
2. WAV 试听：验证仿真结果的音色
3. **面包板插针：发现你想不到的现象** ← v0.3 的 Q 抽头泛音和白噪都是这样发现的

## 一、CH0 方波通道（11 片，三层音色叠加）

### 寄存器（HC374, A0=D5 选通，8 bit 用满）

```
bit0-3: 音量 (4 bit, Q0-Q3 → TLC7524 DB4-7, 与 v0.2 一致)
bit4-5: 占空比挡 (00=50% / 01=25% / 10=12.5% / 11=25%@f/4)
bit6:   mode (0=方波, 1=白噪) ← HC4053 开关 X
bit7:   REF (0=占空比变体, 1=Q0抽头) ← HC4053 开关 Z
```

驱动写入：`(vol & 0x0F) | ((duty & 0x03) << 4) | ((mode & 1) << 6) | ((ref & 1) << 7)`

### 芯片清单

| 层 | 芯片 | 功能 |
|----|------|------|
| 基础（v0.2 7片）| HC373+2×HC161+HC00+HCT74+HC374+TLC7524 | 方波+音量 |
| 占空比 | HC74（分频链 Q2/Q3）+ HC08（AND 组合）+ HC153（4选1→DB）| bit4-5 选 4 挡 |
| mode+REF | **HC4053（双 SPDT）** | bit6 切 mode（tc_hi/pe_n），bit7 切 REF（占空比变体/Q0）|

**HC4053 合并**：原 HC86（mode 二选一）+ REF 开关合并成 HC4053 一片（3 路 SPDT 用 2 路），省 1 片。

### 数据流（已修正自洽）

```
分频链 Q1/Q2/Q3 → HC08 AND → HC153 选占空比变体 (duty50/25/125/625)
                                         ↓
HC161 Q0 抽头 ──────────────→ HC4053 Z (bit7选) ──→ TLC7524.REF
tc_hi / pe_n ──────────────→ HC4053 X (bit6选) ──→ 7474 FF1.D (reload源)
HC374 Q0-Q3 (音量码) ──────────────────────────→ TLC7524.DB4-7 (衰减, v0.2不变)
```

**音量控制与 v0.2 一致**（REF 接波形，DB 接音量码，乘法衰减）。

### 面包板意外发现的音色（iverilog 复现不了）

| 现象 | 原理 | 触发 |
|------|------|------|
| Q0 进 REF 出高音喇叭泛音 | HC161 Q 抽头非理想边沿 × DAC 乘法 | bit7=1 |
| pe_n 进 1D 出白噪 | HC00 传播延迟 + 7474 采样竞争 | bit6=1 |
| 占空比 25%/12.5% | 分频链 AND 组合（数字逻辑，可仿真）| bit4-5 |

## 二、CH1 LFSR 白噪音通道（6 片，定稿）

### 寄存器（HC374, A1=D7 选通，与方波风格统一）

```
bit0-3: 音量 (Q0-Q3 → DB4-7)
bit4-5: 频率挡 (00=÷2 / 01=÷4 / 10=÷8 / 11=÷16, 仅独立模式)
bit6:   绑定开关 (0=独立分频 / 1=直通 square_tc)
bit7:   预留
```

### 芯片清单

CD4070（max-length 反馈 Q7⊕Q5⊕Q4⊕Q3，周期 255）+ HC164（LFSR 主体）+ HC161（独立分频）+ HC153（频率选通）+ HC374 + TLC7524。

死锁防护：RST 期间 serial_in=1 灌种（借 HC00 剩余门）。绑定：直通 square_tc（音越高噪音越密）。

**已砍**：周期噪音（Q0 自反馈，试听死锁/退化方波，无实用性）。

---

## 三、控制线（FT232H，12 根）

- C0-C7: 数据总线
- D4 = LE（period 写选通）
- D5 = A0（方波 HC374）
- D6 = RST
- D7 = A1（噪音 HC374）

## 四、v0.3 工作流程（全程记录，供下个窗口参考）

### 阶段 1：架构规划（design.md）
1. 讨论 PSG2 定位（2 通道：方波 + 噪音）
2. LFSR 方案迭代：HC164+4070+4060+4066 → 砍 4060（用 64kHz 晶振）→ 砍 4066（HC153 选通）→ 定 6 片
3. 寄存器位分配：**看驱动 py**（`self._c = vol & 0x0F` = 音量在 bit0-3），不臆测
4. datasheet 查证：HC164 MR 无置位、HC153 上下半部共享地址、CD4070≠HC86 引脚

### 阶段 2：RTL + 仿真（iverilog + vvp）
- 命令：`PATH="/d/Program Files/oss-cad-suite/bin:/d/Program Files/oss-cad-suite/lib:$PATH" iverilog -g2012 ...`
- inline `PATH=... cmd` 一行写，不用 export（export 会 RC=127）
- 仿真发现：max-length taps Q7⊕Q5 周期只有 30 → 改 Q7⊕Q5⊕Q4⊕Q3 周期 255
- 仿真发现：周期噪音死锁 → 试 CD4078 防护 → 试 ~Q0 → 全砍，只要白噪

### 阶段 3：WAV 试听（耳朵验收）
- TB 生成 64kHz 采样 WAV，试听发现仿真"正确"但听感不对（周期噪音死锁无声）
- 绑定模式方向反（CET 门控让噪音频率和音高成反比）→ 改直通 square_tc

### 阶段 4：面包板探索（意外发现）
- 占空比扩展：7474 分频链 + HC08 AND + HC153，4 挡占空比
- **Q0 进 REF 出泛音**（乘法调制，仿真看不到）
- **pe_n 进 1D 出白噪**（硬件延迟，仿真看不到）
- HC4053 合并 mode+REF 两个二选一（砍 HC86）

### 阶段 5：接线表（wiring-table.md）
- 逐脚标注，Pin 号查 rtl/*.v 模型的 datasheet 注释（不凭记忆）
- 多次修正：U6 Pin 号、HC153 引脚、HC4053 Z0 接 HC153（不是直接接 REF）
