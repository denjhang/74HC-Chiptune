**与 yosys 综合路线本质对立**：全实例化，无隐藏门，74 TTL 高利用率，人脑做架构 AI 做验证。

## 🎯 PSG2 完成清单

### CH0 方波通道（11 片，三层音色叠加）✅ 上板验证通过

**寄存器**（HC374, A0=D5 选通）：
```
bit0-3: 音量 (4 bit, Q0-Q3 → TLC7524 DB4-7)
bit4-5: 占空比挡 (00=12.5% / 01=25% / 10=6.25% / 11=50%, 据实测)
bit6:   ref (0=占空比变体, 1=Q0调制) ← HC4053 开关 Z (bit6/bit7 实测对调)
bit7:   mode (0=方波, 1=白噪)        ← HC4053 开关 X
```

**上板测试结论**：
- ✅ 占空比 4 挡 / 方波白噪切换 / REF 切换 全部正常
- ⚠️ 音量 vol 0-8 递增正常，vol 9+ 到顶（疑 REF 路径阻抗，待查）
- ✅ Q0/Q1 管乐器音色（Q0=小号，Q1=萨克斯，Q 抽头模拟瞬态直接当音色源）
- ✅ mode 白噪（采样竞争，频率锁死在主 clk；占空比窄时音量小+频率高）

### CH1 噪音通道（rev.a: HC374 + 模拟注入，7 片 + 2 电容）✅ 上板验证通过

**核心创新：D[0] 注入模拟噪声，LFSR 退化成采样器，永不锁死**

HC00 门3（反相器）+ 2×0.1μF 电容构成张弛振荡器（实为模拟宽带噪声源）：
- C1 并联门3 输入↔输出（正反馈回路）
- C2 并联门3 两输入（稳定振荡）
- 振荡信号经 HC00 门4（NAND）注入 LFSR 的 D[0]
- D[0] 是真随机（物理噪声），不可能收敛 → **永不锁死**

**上板测试结论**：
- ✅ 一上电就出噪音（振荡器持续注入）
- ✅ 频繁切频率挡不锁死（稳定几分钟）
- ✅ 独立模式 4 挡（÷2=32k / ÷4=16k / ÷8=8k / ÷16=4k）全部实用
- ⚠️ 绑定模式只有高音区（方波 2000-32000Hz）实用，低频不实用（HC374 架构限制）

## ⭐ 核心认知（2026-07-02 至 2026-07-08 持续深化）

### 1. 仿真盲区 + 面包板发现价值

iverilog 是**逻辑仿真器，不是电路仿真器**。以下现象从原理上无法模拟：
- **mode 白噪**：7474 采样竞争（HC00 延迟让 pe_n 跳变沿偏移到 clk 上升沿附近，采样落在建立时间窗内）。⚠️ 必须经过 7474 D 触发器采样才出噪声
- **Q 抽头音色**：HC161 输出的非理想边沿瞬态 + reload 混叠（Q0=小号，Q1=萨克斯）。**不是 TLC7524 乘法调制**——直接把音响接 Q 芰点就是这个音色
- **泛音随音高循环变化**：reload 边沿与 Q 跳变沿的相位关系循环（车轮效应）

### 2. 实践创新原则（最高优先级）

**有些问题还是得用户想办法创新，不要纠结，在实践中找答案。**

LFSR 锁死问题的解决路径（典型案例）：
- AI 反复纠结"HC86 做 OR 要几个门"、"XOR 换 XNOR"——全是纸上推演，没解决
- AI 错误归因"国产芯片质量"，用户纠正"任意芯片都锁死"
- 用户创新：HC00 + 2×0.1μF 电容 = 模拟噪声源，D[0] 注入，LFSR 退化成采样器，**永不锁死**

教训：仿真/理论给方向，**实测给答案**。纠结门级逻辑时跳出来想。**实践中的创新 > 书本标准做法。**

### 3. 噪音方波同源

方波和噪音在 74 离散逻辑里**同源**——都是"计数器 + D 触发器"，区别只在 D 接哪个信号：
- D 接 tc_hi → 方波
- D 接 pe_n → 采样竞争 → 噪音

**改一根线就能切换**。这让 CH1 独立 LFSR 通道的必要性打折扣。

### 4. 芯片架构影响噪音音域（2026-07-08 实测对比）

| 芯片 | 绑定模式实用频率 | 原因 |
|------|-----------------|------|
| HC374/LS374 (rev.a) | 2000-32000Hz | 并行 D 触发器，无级联累积，低频音色薄 |
| **HC164 (国产)** | **200-32000Hz** | 串入并出 8 级链，Q7 延迟累积，低频音色厚 |

**HC164 绑定模式低频也有用（200Hz），HC374 不行。** 这是架构差异，不是质量差异。

### 5. 启动电路是关键，不是芯片型号

2026-07-08 实测：**国产 HC164 也能做 LFSR**，之前失败完全是启动电路缺陷（RC 上电一次性脉冲不够）。HC164/HC374/LS374 都能做 LFSR，关键是启动电路（持续模拟注入才稳）。

## 📋 下一站：查表机路线（SCC 的前置技术）

**SCC/FM 不是"更复杂的音源电路"，是专用音频查表计算机。** HC161 只能计数，多通道调度要状态机，状态机 = 微程序 ROM 驱动 = 查表机/CPU 架构。SCC 绕不开查表机。

| 项目 | 片数 | 能力 | SCC 适用度 |
|------|------|------|-----------|
| fsm_player | 9 | 顺序读 ROM，无分支 | PCM ✅ / SCC ❌ |
| SCC 专用查表机 | ~14-16 | 微程序 + 音频数据通路 | ⭐ 目标 |
| 39040cpu | 22 | 通用图灵机 | ✅ 通用 |

**开发顺序**：PSG2 ✅ → fsm_player/39040 查表机 → SCC(写程序) → FM

详见 design.md 8.6 节 + CLAUDE.md 4.5 节"查表机谱系"。
