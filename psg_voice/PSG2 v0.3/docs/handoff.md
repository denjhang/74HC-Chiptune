**与 yosys 综合路线本质对立**：全实例化，无隐藏门，74 TTL 高利用率，人脑做架构 AI 做验证。

## 🎯 核心认知（2026-07-02 至 2026-07-07 持续深化）

### 1. 仿真盲区 + 面包板发现价值

iverilog 是**逻辑仿真器，不是电路仿真器**。以下现象从原理上无法模拟：
- **mode 白噪**：7474 采样竞争（HC00 延迟 + clk 采样落在跳变沿建立时间窗）。⚠️ 必须经过 7474 D 触发器采样才出噪声，TC/pe_n 直接接喇叭只听到刺耳高频方波
- **Q 抽头音色**：HC161 输出的非理想边沿瞬态 + reload 混叠（Q0=小号高泛音，Q1=萨克斯低泛音）。**不是 TLC7524 乘法调制**——直接把音响接 Q 芰点就是这个音色。泛音随编程 period 循环变化（车轮效应）

**流程的第三条腿——面包板插针探索**：
1. iverilog 仿真：验证你想得到的逻辑
2. WAV 试听：验证仿真结果的音色
3. **面包板插针：发现你想不到的现象** ← 意外音色/物理现象的来源

### 2. ⭐ 实践创新原则（2026-07-07 用户确立，最高优先级）

**有些问题还是得用户想办法创新，不要纠结，在实践中找答案。**

LFSR 锁死问题的解决路径（典型案例）：
- AI 反复纠结"HC86 做 OR 要几个门"、"XOR 换 XNOR"、"CD4078 全 0 检测"——全是纸上推演，没解决实际问题
- AI 错误归因"国产 HC374 质量差"，用户纠正"原装 LS374 一样锁死"——**任意移位寄存器都会锁死，是固有特性**
- 业界标准做法（RC 灌种/全 0 检测/TR-909 启动）都是补丁，没根治
- **用户创新**：HC00 + 2×0.1μF 电容构成模拟噪声源，D[0] 注入持续噪声，LFSR 退化成"采样器"，**永不锁死**

教训：
- 仿真/理论给方向，**实测给答案**
- 纠结门级逻辑时，跳出来想"有没有完全不同的路"
- 74 芯片的模拟特性（延迟、瞬态、振荡）是宝藏，不是干扰
- **实践中的创新 > 书本上的标准做法**

### 3. 噪音方波同源（2026-07-07 认知突破）

方波和噪音在 74 离散逻辑里是**同源**的，都是"计数器 + D 触发器"：
- D 接 tc_hi（确定性相位）→ 方波
- D 接 pe_n（HC00 延迟让跳变沿偏移到 clk 附近）→ 采样竞争 → 噪音

**改一根线就能切换**。这让 CH1 独立 LFSR 通道（6-7 片）的必要性打折扣——0 片的 mode 白噪可能够用。

## 一、CH0 方波通道（11 片，三层音色叠加）

### 寄存器（HC374, A0=D5 选通，8 bit 用满）

```
bit0-3: 音量 (4 bit, Q0-Q3 → TLC7524 DB4-7, 与 v0.2 一致)
bit4-5: 占空比挡 (据硬件实测标签: 00=12.5% / 01=25% / 10=6.25% / 11=50%)
bit6:   ref (0=占空比变体, 1=Q0调制) ← HC4053 开关 Z (bit6/bit7 实测与接线表文档相反)
bit7:   mode (0=方波, 1=白噪)        ← HC4053 开关 X
```

⚠️ **bit6/bit7 据硬件实测对调**（mode=bit7, ref=bit6），软件已按硬件实际对齐。

### 芯片清单

| 层 | 芯片 | 功能 |
|----|------|------|
| 基础（v0.2 7片）| HC373+2×HC161+HC00+HCT74+HC374+TLC7524 | 方波+音量 |
| 占空比 | HC74（分频链）+ HC08（AND 组合）+ HC153（4选1）| bit4-5 选 4 挡 |
| mode+REF | HC4053（双 SPDT）| bit6 切 mode，bit7 切 REF |

### 上板测试结论（2026-07-03）
- ✅ 占空比 4 挡 / 方波白噪切换 / REF 切换 全部正常
- ⚠️ 音量 vol 0-8 递增正常，vol 9+ 到顶（疑 REF 路径 HC153→HC4053 串联阻抗致 TLC7524 输出饱和，待查）
- ✅ Q0/Q1 管乐器音色（小号/萨克斯，Q 抽头模拟瞬态直接当音色源）
- ✅ mode 白噪（采样竞争，频率锁死在主 clk，跟 period 无关；占空比窄时音量小+频率高）

## 二、CH1 噪音通道（rev.a: HC374 + 模拟注入，7 片 + 2 电容）

### 演进历程（重要的试错记录）

| 版本 | 方案 | 问题 |
|------|------|------|
| 原版 | HC164 LFSR + RST 灌种 | 上板锁死（删灌种赌上电随机值，实测频繁锁死）|
| rev.a 初版 | HC374 + RC 上电脉冲 | 频繁切频率挡锁死（RC 放完电无法再触发）|
| **rev.a 定稿** | **HC374 + HC00 振荡器（模拟噪声注入）** | **✅ 稳定几分钟无锁死** |

### 核心创新：D[0] 注入模拟噪声

HC00 门3（反相器）+ 2×0.1μF 电容构成**张弛振荡器**（实为模拟宽带噪声源）：
- C1 并联门3 输入↔输出（正反馈回路）
- C2 并联门3 两输入（稳定振荡）

振荡信号经 HC00 门4（NAND）注入 LFSR 的 D[0]。LFSR 退化成"模拟噪声采样器"：
- LFSR 移位/反馈结构提供"数字骨架"（频率挡/绑定仍有效）
- D[0] 是真随机（物理噪声），不可能收敛 → **永不锁死**

**这绕过了业界标准防锁死补丁，用 74 芯片模拟特性根治数字逻辑死穴。**

### 寄存器（HC374, A1=D7 选通）

```
bit0-3: 音量 (Q0-Q3 → DB4-7)
bit4-5: 频率挡 (00=÷2 / 01=÷4 / 10=÷8 / 11=÷16, 仅独立模式)
bit6:   绑定开关 (0=独立分频 / 1=直通 square_tc)
bit7:   预留
```

### 芯片清单（rev.a）

| 位号 | 型号 | 功能 |
|------|------|------|
| U12 | CD4070 | 白噪反馈 Q7⊕Q5⊕Q4⊕Q3 |
| U13 | **HC374/LS374** | **8 位 LFSR 主体（替代 HC164）** |
| U14 | HC161 | 独立分频 |
| U15 | HC153 | 频率选通 4 选 1 |
| U16 | HC374 | 噪音控制寄存器 |
| U17 | TLC7524 | 噪音 DAC |
| U18 | HC00 | 绑定 2 选 1 |
| U4 借 | HC00 门2/3/4 | 振荡器防锁死（门1 仍 PE 反相）|

**接线表**：`docs/wiring-table-rev-a.md`（rev.a 独立文件，原版 `wiring-table.md` 保留对比）

### 上板测试结论（2026-07-07）
- ✅ 一上电就出噪音（RC 振荡器持续注入）
- ✅ 频繁切频率挡不锁死（稳定几分钟）
- ⏸️ 4 挡频率音色 / 绑定模式 待详细测

## 三、控制线（FT232H，12 根）

- C0-C7: 数据总线
- D4 = LE（period 写选通）
- D5 = A0（方波 HC374）
- D6 = RST
- D7 = A1（噪音 HC374）

## 四、查表机谱系 → SCC 路线（2026-07-03 架构认知）

**SCC/FM 不是"更复杂的音源电路"，是专用音频查表计算机。** HC161 只能计数，多通道调度要状态机，状态机 = 微程序 ROM 驱动 = 查表机/CPU 架构。SCC 绕不开查表机。

| 项目 | 片数 | 能力 | SCC 适用度 |
|------|------|------|-----------|
| fsm_player | 9 | 顺序读 ROM，无分支 | PCM ✅ / SCC ❌ |
| SCC 专用查表机 | ~14-16 | 微程序 + 音频数据通路 | ⭐ 目标 |
| 39040cpu | 22 | 通用图灵机 | ✅ 通用 |

**修正后的开发顺序**：PSG v0.3 ✅ → fsm_player/39040 查表机 → SCC(写程序) → FM

详见 design.md 8.6 节 + CLAUDE.md 4.5 节"查表机谱系"。
