**本项目核心原则**（与综合路线的区别）:
- **全实例化，无隐藏门**: RTL 直接实例化 hc161/hc164/cd4070 等真实芯片模型，不写抽象行为级让综合器猜
- **74 TTL 高利用率**: 每片芯片的每个门都有用途（HC00 剩余门借给噪音通道，不浪费）
- **人脑做架构，AI 做验证**: 架构决策由人（选什么芯片/怎么连线），AI 负责 RTL 建模+仿真+WAV 试听+接线表

**设计哲学**：
- **7 片起步**（v0.2 基础方波+音量，已上板出声）→ **按需扩展**到 16 片
- 每加一个功能都经仿真+WAV 试听确认值得那几片芯片（周期噪音试听不可用就砍掉）
- 实用主义原型，不是 1:1 复刻专用 IC

**AI 辅助流程**（glm + iverilog，**不是 yosys 综合**）：
架构规划→RTL→仿真→WAV 试听→接线表→上板，全流程闭环。
iverilog 只做**仿真验证**（功能对不对），不做综合（不替你决定用几片芯片）。
遇到的问题都是接线/门类型/电平，无架构硬伤。流程方法论见 CLAUDE.md 5.5 节。

## 一句话目标

v0.3 从 v0.2 的单通道方波，扩展到 **两通道**（PSG1→PSG2 的版本跨越）。
核心设计原则：**纯数字电路，不用 RC/电阻网络，每通道复用已验证的方波逻辑**。

## 一、设计原则（从 v0.2 带过来的铁律）

1. **已验证的方波通道直接复制，不搞共享/查表逻辑**。稳定性优先于省芯片。
2. **每通道方波规格**：5 片做频率（2×HC161 + HC373 + HC00 + HCT74），2 片做音量（HC374 + TLC7524）。
3. **TLC7524 节省无源器件**：v0.2 证实 TLC7524 内置 R-2R，省掉所有外部电阻网络/RC。v0.3 延续。
4. **波形数据走 TLC7524 的 DB（数字），REF 接固定/方波参考**。
5. **每通道固定一种波形（不切换）**：CH0=方波, CH1=LFSR噪音, 每通道独立设计。

## 二、通道架构（已定：固定波形通道，每通道独立设计）

### 决策：每通道固定一种波形，不做切换

经过讨论 (原第五章"复用HC161切波形"已弃用)，结论：
**波形切换太复杂**（period 计数与满幅度计数冲突，复用 HC161 要么不满幅度要么频率不对），
**独立通道设计反而简单稳定**——每通道一种波形，各管各的，符合"已验证逻辑直接复制"原则。

### 通道规划（最终，PSG2 = 2 通道）

> PSG2 聚焦两通道研究：方波（v0.2 复用）+ LFSR 噪音。
> 锯齿/三角为后续扩展方向，杜邦线有限时可能拆方波换锯齿/三角。

| 通道 | 波形 | 设计要点 |
|------|------|---------|
| CH0 | 方波+占空比+白噪+Q调制 | 复用 v0.2 toggle, 三层音色叠加 (占空比+mode+REF调制), 见第五/六章 |
| CH1 | LFSR 噪音 | HC164+4070+HC161+HC153 → TLC7524.REF (6片核心, 见第四章) |
| (扩展) | 锯齿波 | 独立计数器(满0-255) → TLC7524#1(DB0-7波形) → TLC7524#2(音量级联) |
| (扩展) | 三角波 | CD4029可逆计数(满0-255) → TLC7524#1(波形) → TLC7524#2(音量级联) |

### 各通道的 DAC 用法（关键区别）

**1-bit 波形（方波/噪音）→ 1 片 TLC7524**：
波形是 0/5V 模拟量，直接进 REF。DB4-7 接音量码。
```
波形(0/5V) ──→ TLC7524.REF
音量码   ──→ TLC7524.DB4-7  → 输出 = 波形 × 音量/256
```
(v0.2 已验证此法可行)

**8-bit 波形（锯齿/三角）→ HC273 毛刺滤除 + 2 片 TLC7524 级联**：
波形是 8 位数字，第一片做 D/A（DB0-7 接计数器），第二片做音量衰减。
**关键：HC161 的 Q 必须经 HC273 同步锁存才能喂 DAC**（详见下方毛刺滤除）。

```
计数器 Q0-7 ──→ [HC273 毛刺滤除] ──→ TLC7524#1.DB0-7  (REF=5V) → 阶梯波形模拟量
                                                              ↓
音量码 ──→ TLC7524#2.DB4-7 (REF=#1输出) → 衰减后波形
```
(乘法 DAC 级联，第一片输出 = 5V×(Q/256)，第二片 = #1输出 × 音量/256)

#### ⚠️ HC273 毛刺滤除（锯齿/三角音质刚需，PSG3 上板必加）

**问题**：HC161 的 Q0-7 在计数翻转瞬间有毛刺（4 个触发器不同步跳变，如 0111→1000 经过瞬态）。
直接喂 TLC7524 会把瞬态也转成电压 → 阶梯波上有尖刺噪音。

**为什么 TLC7524 自带的 WR 锁存不够**：
- TLC7524 的 WR/CS 是**电平透明锁存**（类似 373）：CS=0 且 WR=0 时 DB 直通，WR 上升沿锁存
- 透明期间（WR 低）Q 的毛刺会直通到 DAC 输出，挡不住
- 方波模式 WR/CS 接 GND（常透明）没问题，因为 DB 是音量码（基本不变）
- 锯齿模式 DB 不断变化，必须用**边沿寄存器**在 clk 沿干净采样

**解法（Gigatron 同款）**：HC161 和 HC273 共用 clk，clk 上升沿：
- HC161 更新 Q（新值，但输出还在跳变）
- HC273 锁存 HC161 的 Q（建立时间满足的话锁的是**上一个** clk 周期的稳定值）
- 等于延迟一拍，但完全同步，毛刺被挡

```
clk ──→ HC161.CP (Q 更新, 有瞬态)
  └──→ HC273.CP (锁存 HC161.Q 的稳定值, 无毛刺)
              ↓
         TLC7524.DB0-7 (干净阶梯波)
```

**芯片成本**：锯齿/三角通道因此各 +1 片 HC273（库存充足）。这是音质保证的刚需，不能省。
v0.2 方波通道不需要 HC273（toggle 是 1-bit 边沿干净信号，且 DAC 当衰减器不当生波形器）。

### 混音方式（待定）

各通道 TLC7524 输出怎么合并到喇叭：
- 电阻求和网络（无源，简单）
- 运放加法器（有源，隔离好）

## 四、LFSR 白噪音通道设计（最终定稿 2026-07-02）

> 纯白噪音通道（周期噪音经试听验证不可用，已砍，见 4.10 弃用记录）。
> 两种时钟模式：独立分频（4挡）+ 绑定方波（噪音跟音高）。
> max-length 8 位 LFSR（Q7⊕Q5⊕Q4⊕Q3，周期 255），种子非0 永不自锁。

### 4.1 芯片清单（6 片核心，0 额外防护芯片）

| 芯片 | 用途 | 说明 |
|------|------|------|
| CD4070 | 白噪反馈 | 门1-3: Q7⊕Q5⊕Q4⊕Q3 (max-length 周期255); 门4: 闲置 |
| HC164 | LFSR 主体 | 8 位串入并出 (Q0-Q7 全引出); MR 常接1, 靠 serial_in 灌种 |
| HC161 | 独立分频 | 64kHz ÷2/÷4/÷8/÷16 (自由计数, CEP/CET=1) |
| HC153 | 频率选通 | 上半部 4 选 1 (仅独立模式); 下半部闲置 |
| HC374 | 8 bit 寄存器 | 频率+绑定+音量 |
| TLC7524 | 1-bit DAC | REF=Q7, DB4-7=音量 |

**绑定开关**：借 v0.2 方波通道 HC00 剩余门做 2 选 1（独立时钟 vs square_tc），0 额外芯片。
**死锁防护**：RST 期间 serial_in=1 灌种子（借 HC00 剩余门反相 RST），白噪 max-length 自持。

### 4.2 寄存器（8 bit，1 片 HC374，与方波风格统一：音量在 bit0-3）

```
bit0-3: 音量 (4 bit = 16 级, Q0-Q3 → DB4-7)

bit4-5: 频率挡 (2 bit = 4 挡, 仅独立模式 bind=0 时生效)
        00 = HC161.Q0 (÷2,  32kHz 最高频白噪)
        01 = HC161.Q1 (÷4,  16kHz)
        10 = HC161.Q2 (÷8,   8kHz)
        11 = HC161.Q3 (÷16,  4kHz 低频杂音)

bit6:   绑定开关
        0 = 独立噪音 (时钟来自 HC161 分频, 噪音频率固定 4 挡)
        1 = 绑定方波 (noise_clk 直通 square_tc, 噪音频率 = 2×方波频率)

bit7:   预留
```

### 4.3 数据流（完整连线）

```
                        64kHz 全局晶振
                              │
                              ▼
                    HC161.CLK (自由计数, CET/CEP=1)
                              │
              ┌───────┬───────┴───────┬────────┐
             Q0      Q1              Q2        Q3
              │       │               │         │ (独立模式 4 挡分频)
              ▼       ▼               ▼         ▼
           153.1C0  153.1C1       153.1C2   153.1C3   ← HC153 上半部 4选1
                              (bit0-1 当地址)
                              153.1Y (独立时钟)
                                     │
                                     ▼
                    ┌── 2选1 (bind 控制) ──┐
                    │                       │
              bind=0: ind_clk        bind=1: square_tc (方波通道TC)
                    │                       │
                    └─────────┬─────────────┘
                              ▼
                         noise_clk ──→ HC164.CP
                    (2选1 借 HC00 剩余门: NAND(NAND(ind,!bind),NAND(tc,bind)))

HC164.Q7 ─┐
HC164.Q5 ─┤→ 4070 门1 (Q7⊕Q5)
HC164.Q4 ─┤→ 4070 门2 (Q4⊕Q3)  → 门3 汇总 → xor_fb (max-length 反馈)
HC164.Q3 ─┘
                              ↓
              serial_in = (!rst_n) ? 1 : xor_fb   (RST 灌种)
                              ↓
                         HC164.DSA (串入)

HC164.Q7 ──→ TLC7524.REF        (1-bit 波形)
HC374.bit4-7 ──→ TLC7524.DB4-7  (音量)
HC374.bit0-1 ──→ HC153 地址     (频率挡, 独立模式)
HC374.bit3   ──→ HC00 2选1 控制  (绑定开关)
```

### 4.4 绑定模式（直通 square_tc, 仿真验证 2026-07-02）

**bind=1 时 noise_clk 直通 square_tc**，噪音频率 = TC 频率 = 2×方波频率。

| 音 | period | 方波频率 | 噪音频率 (LFSR 时钟) | 听感 (实测 WAV) |
|----|--------|---------|---------------------|----------------|
| C3 | 11 | 131 Hz | 261 Hz | 低频颗粒可用 |
| C4 | 133 | 262 Hz | 520 Hz | 可用 |
| C5 | 194 | 516 Hz | 1032 Hz | 中频白噪可用 |
| C6 | 225 | 1032 Hz | 2065 Hz | 可用 |
| C7 | 240 | 2000 Hz | 4000 Hz | 中高频白噪可用 |

**音越高噪音越密**（嗨镲跟旋律走），符合 SN76489 绑定语义。
绑定模式下 freq_sel 频率挡失效（直通无分频）。

### 4.5 死锁防护（RST 灌种, 0 额外芯片）

HC164 的 MR 异步清零无置位能力（datasheet 查证），RST 后全 0 会死锁白噪反馈。
解法：**RST 期间 serial_in=1 持续灌种子**，8 拍后 LFSR=0xFF，白噪自持。
- 硬件：借 v0.2 方波通道 HC00 剩余门把 rst_n 反相，RST 时强制 serial_in=1
- 白噪 max-length 性质：种子非 0 永不归零（仿真验证 2000 拍 0 次归零）

|--------|------|-----|
| 白噪独立 4 挡 (÷2/÷4/÷8/÷16) | 全部可用 | 白噪_独立_÷{2,4,8,16}.wav |
| 绑定 C3-C7 (直通 square_tc) | 全部可用 | bind_C{3,4,5,6,7}.wav |

### 4.7 弃用方案记录

#### 周期噪音（已砍，试听验证不可用）

**思路**：bit2 选白噪/周期，周期模式 Q0 自反馈做窄脉冲底鼓。
**弃用原因**（试听 + 振幅分析验证）：
- Q0 自反馈 = 环形移位，必须种子恰好 1 个 1 才周期 8
- HC164 无预置能力，从全 0/全 1/任意态都会死锁或退化成 DC 直流
- 加 CD4078 检测全 0：周期模式归零后注入 1，但又退化归零，循环往复（断续）
- 反馈改 ~Q0：退化成纯方波振荡器，不是噪音
- 试听结果：周期噪音要么死锁无声，要么是方波，无实用性
**结论**：砍周期模式，bit2 预留。纯白噪 4 挡 + 绑定已够用（嗨镲/军鼓/环境）。

#### 旧绑定方案 CET 门控（已弃用，方向反）

**思路**：HC161.CET = bind ? square_tc : 1，绑定时 HC161 对 TC 计数再分频。
**弃用原因**：噪音频率 = (64kHz/period)/freq_sel，**period 越大（音越高）噪音越低**，
与"音越高噪音越密"的直觉相反。C3 有用 C7 几乎无声。
**结论**：改成 bind=1 时 noise_clk 直通 square_tc（见 4.4）。

## 五、方波占空比扩展（CH0，仿真验证通过 2026-07-02）

> 在 v0.2 纯净 toggle 方波（Q1，50%，频率 f）后加分频链 + AND 组合，
> 实现 4 挡占空比切换。**占空比变窄时频率同步降低**（每级÷2）。
> 顺带扩展音域到 C0（原 period 极限只能到 C3）。

### 5.1 芯片清单（新增 3 片）

| 芯片 | 用途 |
|------|------|
| HC74（U8）| FF3(÷2→Q2) + FF4(÷2→Q3)，分频链 |
| HC08（U9）| 3 个 AND 门做 duty 组合 |
| HC153（U10）| 4 选 1 占空比挡 |

v0.2 原 7 片（HC373+2×HC161+HC00+HCT74+HC374+TLC7524）+ 新增 3 片 = **方波通道共 10 片**。

### 5.2 分频链

```
Q1 (U5 toggle, 50%, f) ──→ FF3 (U8 半片1, T触发器÷2) ──→ Q2 (50%, f/2)
Q2 ──→ FF4 (U8 半片2, T触发器÷2) ──→ Q3 (50%, f/4)
```

两级 toggle 分频，Q1/Q2/Q3 都是 50% 方波，频率递降（f → f/2 → f/4）。

### 5.3 占空比组合（HC08 AND 门）

```
duty_50  = Q1                    (原频 f, 50%)
duty_25  = Q1 AND Q2             (f/2, 高电平只在 Q1=Q2=1 时 → 25%)
duty_125 = duty_25 AND Q3        (f/4, 复用 duty_25 → 12.5%)
duty_625 = Q2 AND Q3             (f/4, 25% 变体)
```

3 个 AND 用 HC08 的 3 门（门4 闲置）。duty_125 复用 duty_25 省一个门。

### 5.4 占空比挡位（仿真实测，A4 period=183）

| bit0-1 | 输出 | 占空比 | 频率 | 听感 |
|--------|------|--------|------|------|
| 00 | Q1 | 50% | f（438Hz）| 纯净方波 |
| 01 | Q1·Q2 | 25% | f/2（219Hz）| 窄脉冲，谐波多 |
| 10 | Q1·Q2·Q3 | 12.5% | f/4（110Hz）| 更窄，鼻音 |
| 11 | Q2·Q3 | 25% | f/4（110Hz）| 25%@低八度 |

### 5.5 特性

- **占空比变窄 → 频率降低**：25% 挡降一个八度，12.5% 挡降两个八度。
- **软件移调补偿**：选窄占空比挡时，period 减半/÷4 可回到原音高。
- **音域扩展**：12.5%@f/4 挡让原 period 范围（C3+）下探到 C0。
- **音色变化**：占空比越窄谐波越丰富（SID pulse wave 风格）。

### 5.6 寄存器改动（HC374）

**音量不动**（bit0-3，Q0-Q3 → DB4-7，与 v0.2 完全一致）。
**占空比放 bit4-5**（v0.2 这两位接地，v0.3 启用）。
**mode 放 bit6**（方波/白噪切换，见 5.8 节白噪发现）。

```
bit0-3: 音量 (16 级, Q0-Q3 → DB4-7, 与 v0.2 一致)
bit4-5: 占空比挡 (00=50% / 01=25% / 10=12.5% / 11=25%@f/4变体)
bit6:   mode (0=方波, 1=白噪)
bit7:   预留
```

驱动改动：`self._c = (vol & 0x0F) | ((duty & 0x03) << 4) | ((mode & 1) << 6)`（v0.2 是 `self._c = vol & 0x0F`）。

### 5.7 详细接线

见 `wiring-table.md`（U6 改动 + U8/U9/U10/U11 新增）。

### 5.8 白噪模式发现（bug 变 feature，硬件实测 2026-07-02）

**现象**：把 7474 FF1 的 D 输入从 tc_hi 改接到 pe_n（HC00 反相 tc_hi 的输出），
方波通道**意外输出高频白噪**，有实用性。

**原理**（硬件物理延迟产物）：
- HC00 的真实传播延迟（几 ns~几十 ns）使 pe_n 的跳变沿相对 tc_hi 偏移
- 7474 FF1 用 clk=64kHz 采样时，落点随 period 微妙变化（有时赶上 clk 沿，有时赶不上）
- toggle 翻转时机呈现"看似随机"的模式 → 输出白噪
- 这是**真实硬件的元现象**，iverilog 仿真复现不了（理想 HC00 零延迟）

**实现（HC4053 开关 X，与 REF 切换合并成一片）**：HC4053 双 SPDT，开关 X 切 FF1 的 D 输入源：
- mode=0: X-COM = tc_hi（正常方波）
- mode=1: X-COM = pe_n（白噪模式）

```
pe_n (HC00 反相 tc_hi) ─┐
                        ├→ HC4053 开关 X (mode 控制) ─→ 7474 FF1.D
tc_hi ──────────────────┘
```
HC4053 开关 Z 同时管 REF 切换（占空比变体/Q0），**一片双 SPDT 顶原 HC86 + REF 开关**。

**特性与局限**：
- ✅ 0 额外噪音芯片（方波通道自带，只 +1 片 HC86 做切换）
- ✅ 占空比链在白噪模式下仍生效（Q2/Q3 分频不受影响）
- ❌ 白噪音色依赖具体 HC00 的物理延迟（批次/温度/电压相关，不可控）
- ❌ 仿真无法预测（iverilog 理想门），只能上板实测
- ❌ 不如 LFSR 噪音可控（无频率挡/绑定）

**vs LFSR 噪音通道（CH1）**：
- HC86 白噪：0 额外噪音芯片（+1 HC86 切换），不可控，硬件元现象
- LFSR 通道：6 片，可控（频率挡/绑定/max-length），数字确定
- 两者可共存（CH0 自带白噪 + CH1 LFSR），也可只用 CH0 白噪省掉 CH1

### 5.9 完整音色矩阵（三层叠加，面包板探索期穷尽音色）

**CH0 方波通道的音色由三层独立切换叠加**：

| 层 | 路径 | 切换芯片 | 控制位 | 切换内容 |
|----|------|---------|--------|---------|
| 占空比 | DB（数字端）| HC153（U10）| bit4-5 | duty50/25/125/625 |
| mode | 1D（reload 源）| HC86（U11）| bit6 | tc_hi（方波）/ pe_n（白噪）|
| REF 调制 | REF（模拟端）| HC4053（SPDT 模拟开关）| bit7 | toggle/Q0 |

**理论组合**：4（占空比）× 2（mode）× 4（REF）= **32 种音色组合**。
（实际很多组合重复或无意义，探索期全保留，正式版按音乐需求精简。）

**REF 调制原理（面包板意外发现）**：
TLC7524 是乘法 DAC，输出 = REF × DB/256。REF 接 HC161 的 Q0 抽头（高频方波），
DB4-7 接 HC374 音量码（4-bit 衰减系数，不是方波）。
REF 方波的**非理想边沿/瞬态/谐波**经 DAC 转换后产生独特泛音 → 乘法调制音色。
不同 Q 抽头频率不同，泛音分布不同 → 每种独特。
（注：bit7=0 时 REF 接占空比变体 = 普通 1-bit 方波，bit7=1 时 REF 接 Q0 = 带谐波的方波）

**⚠️ 这些音色是硬件物理特性产物，iverilog 仿真复现不了**：
- Q 抽头的非理想边沿/瞬态/毛刺 → 模拟特性，逻辑仿真看不到
- 白噪的 HC00 传播延迟 → 批次相关，仿真零延迟
- 只有面包板实测能听到真实音色（详见 CLAUDE.md 5.5 节"仿真盲区"）

**实测音色表（面包板探索，2026-07-02）**：

| REF 源 | 听感 | mode | 备注 |
|--------|------|------|------|
| toggle | 纯净方波 | 方波 | 基准音色（占空比变体叠加）|
| Q0 | 高音喇叭+丰富泛音 | 方波 | 乘法调制, 高一个八度 |
| Q1 | 中频调制泛音 | 方波 | |
| Q2 | 低频调制 | 方波 | C5 时无声 (频率太低) |
| toggle | 白噪 | 白噪 | HC86 切 pe_n, 硬件延迟产物 |

#### HC4053 接线（REF 路径 2 选 1，1 片搞定）

HC4053 是三 SPDT 模拟开关（1 位控制 2 选 1，天然互斥）。只用 1 路（Z）：

```
HC4053 Z0 ← toggle (U5.Q1)      ─┐
HC4053 Z1 ← Q0 抽头 (HC161)     ─┤  2 选 1
HC4053 Z-COM → U7.REF            ─┘
HC4053 S2 ← bit7 (0=toggle, 1=Q0)
HC4053 E ← GND (常开)
```

**为什么用 HC4053 不用 HC153/4066/CD4099**：
- HC153 数字多路器会整形吃泛音 ❌
- CD4066 要独热码+非门，2 片 ❌
- CD4099+4066 两片配合太复杂 ❌
- **HC4053 一片 SPDT，1 位控制，模拟保真** ✅

#### HC4053 控制（bit7，0 新增 FT232H 引脚）

软件切 REF：写 HC374 设 bit7 = 0（toggle）/ 1（Q0），HC4053 立即切换。
剩余 2 路 SPDT（X/Y）空闲，以后可扩展（级联成 3 选 1/4 选 1）。

## 五点五、PSG3 架构预览（锯齿/三角波，下一版本规划）

> v0.3 上板验证后，PSG3 将加入锯齿/三角波通道。这是迈向 SCC 波形内存音源的第一步
> （从"toggle 出 1-bit 方波"升级到"DAC 生 8-bit 多电平波形"）。

### 核心区别：方波 vs 锯齿/三角

| | 方波 (v0.2/v0.3) | 锯齿/三角 (PSG3) |
|---|---|---|
| 波形本质 | 1-bit (0/5V 翻转) | 8-bit (0-255 阶梯) |
| DAC 角色 | **衰减器** (REF=波形, DB=音量) | **生波形器** (DB=波形数据, REF=5V) |
| 计数器输出 | TC→toggle (边沿干净) | Q0-7 直出 (**有毛刺, 需 HC273**) |
| 音量控制 | REF 上乘 (1 片 DAC) | 第二片 DAC 级联衰减 |
| 单通道芯片数 | 7-10 | 8-9 |

### 锯齿通道架构（8 片）

```
clk ──→ HC161×2 (满幅度 0-255 计数, period 控频)
              │ Q0-7 (有毛刺)
              ▼
         HC273 (clk 同步锁存, 滤毛刺)  ← Gigatron 同款解法
              │ Q0-7 (干净)
              ▼
         TLC7524#1 (DB0-7=波形, REF=5V) → 阶梯锯齿模拟量
              │
              ▼
         TLC7524#2 (DB4-7=音量, REF=#1输出) → 衰减后锯齿
```
芯片：2×HC161 + HC373 + HC00 + HC273 + HC374 + 2×TLC7524 = **8 片**

### 三角通道架构（8-9 片）

锯齿基础上把 HC161 换成可逆计数 + 方向控制：
- **CD4029**（4 位可逆计数器）×2 级联，或单芯片 + 方向逻辑
- **CD4027**（双 JK 触发器）做方向控制：计满→减，计到 0→加
- 计数 0→255→0→255... 形成三角（折线上升下降）
- 其余同锯齿（HC273 毛刺滤除 + 2 片 TLC7524 级联）

芯片：2×CD4029 + CD4027 + HC273 + HC374 + 2×TLC7524 ≈ **8-9 片**

### HC273 毛刺滤除（PSG3 音质关键，详见 2.1 节）

HC161/4029 的 Q 在计数翻转瞬间有毛刺（各位不同步跳变）。
TLC7524 自带的 WR 锁存是电平透明型（透明期间毛刺直通），**挡不住**。
**必须用 HC273 边沿寄存器**在 clk 沿采样稳定值（延迟一拍，完全同步，无毛刺）。

这是 Gigatron 用 HC273 的同一原因——纯数字解法，不依赖 RC 延迟。

### 从锯齿/三角到 SCC

PSG3 的锯齿通道本质是 SCC 的**固定波形特例**：
- 锯齿 = 波形表为 [0,1,2,...,255] 的 SCC（线性递增，不需存表）
- 三角 = 波形表为 [0,1,...,127,128,127,...,1,0] 的 SCC（折线）

PSG3 通了之后，把"HC161 计数直出"换成"HC161 当地址 → SRAM 读波形数据"，
**就是 SCC**。多一片 SRAM（库存 62256）+ 地址锁存，波形任意可编程。

```
PSG3 锯齿:  HC161(Q=波形数据) → HC273 → TLC7524
SCC:        HC161(当地址) → SRAM(存波形) → HC273 → TLC7524
                                  ↑ 多这一步, 波形可编程
```

---

> **弃用原因**：period 计数（控频率）与满幅度计数（0-255 控波形）本质冲突。
> 复用 HC161 的 Q 输出，锯齿幅度会随音高变（高音矮低音高），无法保证满幅度。
> 加独立 HC161 又导致频率不对（÷256 太低）。波形切换最少加 2-4 片但效果不理想。
> **结论：走固定波形通道（见第二章），每通道独立设计，简单稳定。**

### 5.1 核心洞察：HC161 的 Q 输出本身就是波形数据

方波通道现在用 HC161 (2片级联 8位) 计 period, 计满后 TC 触发 toggle 翻转出方波。
**但 HC161 的 Q0-Q7 是 8 位二进制计数输出** — 这个值本身就是阶梯锯齿波!

复用 HC161 后, 同一套计数器可以同时输出多种波形:
- **Q0-Q7 直出** = 锯齿波 (0→period 阶梯上升)
- **toggle 输出** = 方波 (现做法)
- **Q0-Q7 高低半周比较** = 三角/占空比方波

### 5.2 4 bit 余量的分配

方波通道现在: HC374 用 4 bit 做音量 (DB4-7), **剩 4 bit 空闲**。
这 4 bit 可以做:

| 候选分配 | bit0-1 | bit2-3 | 效果 |
|---------|--------|--------|------|
| A | 占空比 (2位=4挡) | 波形选择 (2位=4波形) | 方波占空比可调 + 切波形 |
| B | 占空比 (4位) | — | 方波占空比 16 挡精细调 |
| C | 波形 (4位) | — | 最多 16 种波形 (过度) |

倾向 **A (2位占空比 + 2位波形选择)**:
- 占空比 4 挡: 50%(方波)/25%/12.5%/75% (典型 PSG 用途)
- 波形 4 种: 方波/锯齿/三角/脉冲 (够用)

### 5.3 占空比怎么实现 (纯数字, 用 161 复用)

占空比 = 高电平占周期的比例。HC161 计数 0→period, 用一个**比较点**切高低电平:
- 计数值 < duty_threshold → 高电平
- 计数值 ≥ duty_threshold → 低电平

duty_threshold = period × 占空比比例。
**问题**: 这要幅度比较器 (HC85/4063), 库存没有。

**但有个纯数字的偷懒法**: 直接用 HC161 的高位 Q 做比较点!
- Q7 (bit7) = 0 时高, =1 时低 → 50% 方波 (本质就是 toggle 的另一半)
- Q6 (bit6) 做分界 → 25%/75%
- 用 2 bit 选 Q5/Q6/Q7 之一做分界 → 4 挡占空比, **0 额外芯片** (4066 选 Q 位)

### 5.4 波形选择怎么实现

HC161 的 Q0-Q7 直出是锯齿。要三角/方波, 是对 Q 的不同处理:
- 锯齿: Q0-Q7 直送 TLC7524 (波形DAC)
- 三角: Q0-Q7 的最高位(bit7)反向后异或回各位 (折半三角) — 需异或门
- 方波: 只取某高位 Q (bit7) = 当前 toggle 做法

波形选择 (2 bit) 用 4066/LS368 在这几种"Q 处理结果"之间选通。

### 5.5 复用思路的成本优势

如果复用 HC161, **一个通道就能出多种波形** (不再是固定波形):
- 方波通道加几片 4066/4070 → 升级成"可切波形通道"
- 不用为每种波形单独建通道

**待讨论**: 是走"固定波形通道" (CH0方波/CH1锯齿/...) 还是"可切波形通道" (每通道都能切)?
- 固定波形: 简单, 通道职责清晰, 但波形种类少
- 可切波形: 灵活, 4 bit 余量用满, 但要加 4066/4070 做选通逻辑

## 八点五、CH0 上板实测发现 → 频率精度架构分析（2026-07-03）

> v0.3 上板实测暴露的核心瓶颈: **period 8-bit 位宽不够, 高音区精度崩盘**.
> 这是后续 PSG3/v0.4/SCC 架构设计的核心驱动力.

### 8.5.1 实测发现（CH0 方波通道）

| 现象 | 根因 |
|------|------|
| C6 以上音准差, 要靠 Q0 顶上去 | period 8-bit 高音区量化太粗 (见下表) |
| 占空比越小, 噪音频率越高 | 占空比分频链 (Q2/Q3) 改变了 HC164 时钟源, 噪音和方波不是独立时钟 |
| 占空比扩展低音有效 | Q2/Q3 = f/2/f/4 相当于朴素 BLK (降八度), 低音区 period 用满 8-bit |
| 波形 + 噪音同时用, 噪音略变 | 两路共享 HC161/TC, 互相耦合 |
| Q0 调制和占空比无关 | Q0 是 HC161 固有输出, 不受 toggle/分频链影响, 是另一条时钟路径 |
| REF 路径 (HC153→HC4053) 阻抗致音量 vol 9+ 饱和 | 待查, 疑模拟开关串联驱动能力不足 |

### 8.5.2 有效音域 ≤ 2000Hz（实测, 重要设计约束）

**用户实测: 超过 2000Hz 基本听不见.** 因此"最高频率 32kHz (period=1)"这个指标
**对实际用途无意义** — 不需要追求高频上限, **精度才是瓶颈**.

> 约束更新: 有效音域 C3-C7 (~131-2093Hz), C5 以下精度好, **C6-C7 是精度短板**.

### 8.5.3 period 8-bit 的量化精度（核心问题）

`freq = 64kHz / (2 × (256-p))`, 高音区 (256-p) 小, 量化误差大:

| 音 | freq | 256-p | 8-bit 误差 | 12-bit 误差 |
|----|------|-------|-----------|------------|
| C3 | 131Hz | 244 | 0.04% ✅ | 0.0004% |
| C5 | 523Hz | 61 | 0.8% ✅ | 0.0004% |
| C6 | 1047Hz | 30 | **1.6% ❌** | 0.001% |
| C7 | 2093Hz | 15 | **3.0% ❌ (半音跳)** | 0.003% |

**C6-C7 区间 8-bit 每 1 档 period = 1.6-3% 跳变** (接近一个半音), 音准失控.
12-bit 把误差降到 0.01% 以下 (人耳极限 ~0.1%), 全音域解决.

### 8.5.4 解决方案对比: TP/FP (AY8910) vs BLK/FNUM (YM2163)

查证 AY8910 datasheet + YM2163 架构, 两种"频率拆两块"的本质区别:

**AY8910 (PSG, TP/FP 线性 period)**:
- 12-bit period = (Coarse<<8) | Fine, **完整 period 值拆高低字节**
- 下计数器从 TP 计到 0 翻转, `F = clk / (16 × TP)`
- TP/FP 拆分纯粹是 **8-bit 总线装 12-bit 的寄存器寻址问题**, 不是计算方式拆分
- 改频率 = period reload → **相位跳变** (PSG 方波无所谓)

**YM2163 (FM, BLK/FNUM 浮点)**:
- `freq = fnum × 2^blk × (clk/2^20)`, **浮点表示 (尾数 + 八度指数)**
- 相位累加器 (NCO): `phase[20-bit] += fnum << blk`, 高位查正弦表
- **改频率时 phase 连续累加, 不跳变** ← FM 调制必需 (相位跳变=爆音)
- BLK 选八度, FNUM 八度内精调, **每八度相对精度均匀**

| | AY8910 (TP/FP) | YM2163 (BLK/FNUM) | 本项目 v0.3 (现状) |
|---|---|---|---|
| 本质 | 定点 period (线性) | 浮点频率 (八度+尾数) | 定点 period (`256-p` 反向) |
| 改频率 | reload, **相位跳变** | 改 fnum, **相位连续** | reload, 相位跳变 |
| 精度分布 | 低音精, 高音粗 | **每八度均匀** | 低音精, 高音**极粗** |
| 适用 | PSG (方波) | FM (要相位连续+调制) | PSG (方波+占空比) |

### 8.5.5 本项目的结论

**本项目是 PSG (方波+占空比), 不是 FM** → 不需要 BLK/FNUM 的 NCO, 用 AY8910 式
period 扩展就够:
- **加 1 片 HC374 扩 period 到 10-12 bit** (低 4 位频率, 高 4 位波形选择/花样)
- HC161 计数器级联扩到位 (12-bit = 3×HC161, 10-bit = 2.5×)
- period 正向编码 (TP 大=低频, 不再用 `256-p` 反向)
- **clk 64kHz 够用** (有效音域 ≤2000Hz, 离 clk/2=32kHz 还有 16 倍余量, 瓶颈是位宽不是 clk)

**BLK/FNUM (NCO) 留给 SCC 路线**: SCC 用 SRAM 查表 + 相位累加器, 那时才需要
相位连续 (波形任意可编程, 切波形不爆音). 本项目从 period 计数器 → 12-bit period →
SCC (NCO) 是渐进路线, 不是一步到位.

**弃用方向**: 追求最高频 (period=1 → 32kHz 听不见, 无意义).
**保留方向**: 提高 period 位宽解决 C6-C7 精度.
