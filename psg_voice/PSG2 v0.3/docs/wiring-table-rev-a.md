# PSG2 v0.3 噪音通道接线表 rev.a (HC374 版)

> **rev.a vs 原版 (wiring-table.md 第十一节) 的区别**:
> - LFSR 主体: HC164 → **HC374** (国产 HC164 不稳定 + 上电全 0 死锁)
> - 反馈: CD4070 (不变, 与 HC86 功能等效)
> - 启动逻辑: **HC00 门3 + 2×0.1μF 电容构成张弛振荡器, 持续注入 D[0] 防锁死** (0 额外芯片)
>   (原方案 RC 上电一次性脉冲, 实测频繁切频率会锁死; 振荡器方案持续注入, 稳定)
> - 芯片数: 7 片 (持平)
>
> rev.a 只改噪音通道 (CH1). 方波通道 (CH0) 接线不变, 见原版 wiring-table.md.

## 二、LFSR 结构 (HC374 八 D 触发器)

**与 HC164 的本质区别**: HC374 是边沿 D 触发器, D 输入独立; HC164 是串入并出, 只有 DSA/DSB 两个串入.
HC374 的每位 D 单独可控, 上电启动逻辑可强制 D[0]=1 (HC164 做不到, 它的串入由反馈独占).

**LFSR 接法 (XOR 反馈, max-length 8 位, 周期 255)**:

```
D[0] ← 启动逻辑 (RC 覆盖 + feedback, 见第六节)
D[1] ← Q[0]    (移位: 前级 Q → 下级 D)
D[2] ← Q[1]
D[3] ← Q[2]
D[4] ← Q[3]
D[5] ← Q[4]
D[6] ← Q[5]
D[7] ← Q[6]

feedback (CD4070 门1-3) = Q[7] ⊕ Q[5] ⊕ Q[4] ⊕ Q[3]
```

**工作过程**:
1. 上电: RC 脉冲维持 ~5ms, D[0] 强制=1, LFSR 在 noise_clk 驱动下被 1 填满
2. RC 充电后 (~5ms): D[0] 切回 feedback, max-length LFSR 自持 (非 0 永不归零)
3. 永不锁死

## 六、U12 — CD4070 (XOR 反馈)

## 八、U4 借用 — HC00 门 2/3/4 (振荡注入防锁死)

> 方波通道 U4 (HC00) 门1 做 PE 反相 (v0.2 用), **门 2/3/4 借给噪音做防锁死**.
>
> ⚠️ **方案演进** (2026-07-07 实测):
> - 原方案: RC 上电一次性脉冲 (47kΩ+0.1μF), 只在上电时灌种.
>   实测问题: **频繁切换频率挡会锁死** (noise_clk 突变时 HC374 落入非常规态, RC 已放电无法再次触发).
> - 新方案: **HC00 反相器 + 电容构成张弛振荡器, 持续给 D[0] 注入抖动**, 永不锁死.
>   实测稳定运行几分钟无问题, 频繁切频率挡不再锁死.

**振荡器原理** (HC00 门3 反相器 + 电容):

HC00 门3 输入短接当反相器 (Pin 9,10 并联 → Pin 8 反相输出).
**两个 0.1μF 电容** (104):
- **电容 C1**: 并联在门3 的输入输出之间 (Pin 9/10 ↔ Pin 8) — 形成正反馈回路
- **电容 C2**: 并联在门3 的两个输入之间 (Pin 9 ↔ Pin 10) — 加强振荡稳定性

反相器 + 电容构成**张弛振荡器**: 电容充放电让输入在阈值附近反复跳变,
反相器持续输出方波脉冲 (不是上电一次性, 是持续振荡).
这个振荡信号 (NOT 节点) 经门4 注入 D[0], LFSR 永远无法收敛到死锁态.

**效果**: 等于给 LFSR 的 D[0] 注入持续的模拟抖动, LFSR 变成"半模拟半数字"噪声源.
比纯数字 LFSR (靠 max-length 防锁死) 更鲁棒, 比 mode 白噪 (采样竞争) 更可控
(LFSR 结构还在, 频率挡/绑定仍有效).

**⚠️ 本质揭露 (2026-07-07 用户实测)**:
用户把 RC 节点直接接音响, 听到**小声白噪** — 说明这个"振荡器"**根本就是个模拟噪声源**.
HC00 反相器 + 0.1μF 电容的时间常数极小, 振荡在 MHz 级, 输出是**宽带噪声** (不是干净方波).

所以这套方案的真相是: **LFSR 退化成"模拟噪声采样器"**.
- 每个 noise_clk 上升沿, HC374 采一个模拟噪声的瞬时值 (高/低) 到 D[0]
- LFSR 的移位/反馈结构提供"数字骨架" (输出有数字味道)
- 但 D[0] 是真随机 (物理噪声), 不是 LFSR 自己算出来的反馈
- **永远不可能锁死** — D[0] 是真随机, 不可能收敛到固定值

**为什么"频繁切频率挡锁死"问题消失了**:
原 RC 上电脉冲方案: 脉冲放完, D[0]=feedback (纯数字 LFSR 演化),
**任意移位寄存器都会锁死** (不是国产芯片问题 — 这次用的原装 LS374 一样锁死,
之前 HC164 也是类似情况. 移位寄存器的纯数字 LFSR 在 noise_clk 突变时
会落入伪锁死态, 这是固有特性, 不是芯片质量问题).
新方案: D[0]=持续模拟噪声, **没有"纯数字演化"这个过程**, 不可能锁死.

**教训: 还是要有创新精神**:
纯数字 LFSR 的"max-length 防锁死"是数学理论, 硬件上不可靠 (任意移位寄存器都会锁死).
业界标准做法 (RC 灌种/全 0 检测/TR-909 启动逻辑) 都是补丁, 没有根治.
本项目的创新: **D[0] 注入模拟噪声, 让 LFSR 退化成采样器** — 绕过"防锁死"这个问题,
直接让它不可能锁死. 这是用 74 芯片的模拟特性 (HC00+电容=噪声源) 解决数字逻辑的死穴.

**三个噪声源的统一认知** (项目所有噪声方案, 都基于 HC161/HC00 非理想特性):

| 噪声源 | 原理 | 锁死风险 |
|--------|------|---------|
| mode 白噪 (bit6, 5.8.1) | HC00 延迟 + 7474 采样竞争 | 无 (reload 自持) |
| HC164/LS374 纯数字 LFSR (已弃用) | 纯数字伪随机 (max-length) | **高 (任意移位寄存器都会锁死, 不是芯片质量问题)** |
| **HC374 LFSR + RC 模拟噪声注入 (本方案)** | **LFSR 骨架 + D[0] 真随机种子** | **无 (物理随机)** |

本方案最优: 结合 LFSR 的可控性 (频率挡/绑定) 和模拟噪声的不可锁死性.

| 9 (A3), 10 (B3) | **振荡节点** (两脚并联) | 门3 输入 (短接当反相器) + **C1/C2 电容接这里** | 振荡反相器输入 |
| 8 (Y3) | NOT(振荡节点) (持续振荡) | → U4.12 (门4 输入 b) + **C1 另一端** | 振荡反相器输出 |
| 12 (A4) | ← U4.6 (NOT feedback) | 门4 输入 a | |
| 13 (B4) | ← U4.8 (NOT 振荡) | 门4 输入 b | |
| 11 (Y4) | **feedback OR OSC (持续注入)** | → **U13.D0 (HC374 Pin 3)** | ⭐ 防锁死注入输出 |
| 14 | VCC | +5V | |
| 7 | GND | GND | |

**真值表验证**:

| 振荡节点 | NOT(振荡) (门3) | NOT(fb) (门2) | NAND (门4) | D[0] 含义 |
|---------|-----------------|---------------|------------|-----------|
| 高 (振荡正半周) | 0 | x | NAND(x,0) = **1** | **注入 1** (破坏收敛) |
| 低 (振荡负半周) | 1 | NOT(fb) | NAND(NOT fb, 1) = **fb** | **正常反馈** |

> 振荡正半周强制 D[0]=1 (破坏任何收敛态), 负半周让 feedback 接管.
> 两半周交替, LFSR 既保持 max-length 演化, 又被周期性"踢出"死锁陷阱.
