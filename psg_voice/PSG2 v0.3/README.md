# PSG2 v0.3 — 两通道 PSG (方波占空比 + LFSR 白噪音)

> 74HC 纯数字合成器, 基于 PSG1 v0.2 方波通道扩展。
> AI 辅助设计流程验证项目 (glm + iverilog + yosys, 2026-07-02 跑通)。

## 项目独特性（全球搜证 2026-07-02）

**用最少 74 门做 PSG 原型并真上板, 全球开源社区目前没有同类项目。**

### 对比 1: 同类 PSG 复刻项目

| 对比项目 | IC 数量 | 完成度 |
|---------|---------|--------|
| mengstr/Discrete-AY-3-8910 (最接近) | 104 片 | 仅 Digital 仿真器画图, 未做硬件 |
| rejunity/tt05-psg-ay8913 | 1500-1600 门 | FPGA/ASIC, 非 74 离散逻辑 |
| 日本博客 AY8910 项目 | — | 停更未完成 |
| **本项目 PSG2 v0.3** | **16 片** | iverilog+WAV 试听+真硬件上板 |

### 对比 2: HDL→74 综合路线 (Pepijn de Vos 等)

本项目与"用 yosys/ghdl 把 Verilog 自动综合成 74 门"的路线**本质对立**:

| | yosys 自动综合 (Pepijn de Vos) | **本项目: 全实例化** |
|---|---|---|
| 代表 demo | breathing LED = **5 片** | PSG 方波+音量 = **7 片** |
| 设计方法 | 行为级 Verilog → yosys 综合 → 映射 74 | **手写实例化真实 74 芯片模型** |
| 门利用率 | 极低 (综合器不懂芯片级优化) | **每个门都有用, 无隐藏门** |
| 可控性 | 黑盒 (综合结果不可预测) | **完全可控 (接线表逐脚标注)** |
| 实用性 | 无 (demo 玩具, 无人做复杂功能) | **真 PSG 原型, 上板出声** |

**本项目的核心原则** (与综合路线的区别):
- **全实例化, 无隐藏门**: RTL 直接实例化 hc161/hc164/cd4070 等真实芯片模型, 不写抽象行为级让综合器猜
- **74 TTL 高利用率**: 每片芯片的每个门都有用途 (HC00 剩余门借给噪音通道, 不浪费)
- **人脑做架构, AI 做验证**: 架构决策由人 (选什么芯片/怎么连线), AI 负责 RTL 建模+仿真+WAV 试听+接线表

**设计哲学**: 7 片起步 (v0.2 基础方波+音量, 已上板出声) → 按需扩展到 16 片。
每加一个功能都经仿真+WAV 试听确认值得那几片芯片 (周期噪音试听不可用就砍掉)。
实用主义原型, 不是 1:1 复刻专用 IC。

**AI 辅助流程** (glm + iverilog, 不是 yosys 综合): 架构规划→RTL→仿真→WAV 试听→接线表→上板, 全流程闭环。
iverilog 只做**仿真验证** (功能对不对), 不做综合 (不替你决定用几片芯片)。
遇到的问题都是接线/门类型/电平, 无架构硬伤。

## 定位

PSG2 = **两通道研究**:
- **CH0 方波三层音色叠加** (11 片, 探索期穷尽音色):
  - 占空比层 (HC153 切 DB): 50%/25%/12.5%/25%@f/4
  - mode 层 (HC4053 开关 X 切 1D): 方波 / 白噪
  - REF 调制层 (HC4053 开关 Z 切 REF): 占空比变体/Q0 乘法调制泛音
  - HC4053 一片承担 mode+REF 两个二选一 (3 路 SPDT 用 2 路)
  - 面包板意外发现: Q 抽头进 REF 出独特泛音, pe_n 进 1D 出白噪 (iverilog 仿真复现不了)
- **CH1 LFSR 白噪音**: max-length 8 位 LFSR, 独立 4 挡频率 + 绑定方波模式

## 长期愿景 (音源方向)

本项目是"用 74 门设计数字音源芯片"系列的**音源方向先发项目**。
声音反馈直观、单通道可验证, 适合先从这里入手。

| 阶段 | 内容 | 状态 |
|------|------|------|
| PSG1 v0.1-v0.2 | 方波 + 4bit 音量 + ADSR | ✅ 已上板出声 |
| **PSG2 v0.3** | **方波三层音色 (占空比+mode+REF调制) + LFSR 噪音** | **上板中** |
| (未来) SCC 波形内存 | 计数器查 RAM/ROM + DAC | 库存够 (SRAM/ROM/74), 下一个目标 |
| (未来) PCM/ADPCM | 地址计数 + ROM + DAC (+差分解码) | 与 SCC 共享 80% 硬件 |
| (未来) FM 合成 | 相位累加 + 正弦 ROM + 相位调制 | 最难, SCC 通了再上 |

**软件镜像**: STC32G144 的 sn76489.c/ay8910.c 是同一套音源算法的软件实现,
那些 .c 里验证过的算法 (频率/音量/包络/混音) 可直接"翻译"成 74 门电路。

同源延伸 (暂缓): 39040cpu (74 CPU), prog_timer (定时器)。音源路线跑通再回头。

后续扩展方向: 锯齿波 / 三角波 (满幅度计数器 + TLC7524 级联)。

> **4 片扩展** (v0.3 三层音色): HC74 + HC08 + HC153 + HC4053

## 寄存器 (两通道风格统一, 音量在 bit0-3)

### CH0 方波 (HC374, A0=D5 选通)
```
bit0-3: 音量 (16 级, Q0-Q3 → DB4-7)
bit4-5: 占空比挡 (00=50% / 01=25% / 10=12.5% / 11=25%@f/4)
bit6-7: 预留
```

### CH1 噪音 (HC374, A1=D7 选通)
```
bit0-3: 音量 (16 级, Q0-Q3 → DB4-7)
bit4-5: 频率挡 (00=÷2 / 01=÷4 / 10=÷8 / 11=÷16, 仅独立模式)
bit6:   绑定开关 (0=独立分频 / 1=直通 square_tc)
bit7:   预留
```

## 控制线 (FT232H, 12 根)
- C0-C7: 数据总线 (写 period / 方波控制字 / 噪音控制字)
- D4 = LE (period 写选通)
- D5 = A0 (方波 HC374)
- D6 = RST
- D7 = A1 (噪音 HC374)

## 关键设计决策

1. **方波占空比用分频链**: 7474 toggle 后串两级 ÷2, AND 组合出 25%/12.5%。占空比变窄时频率同步降低 (软件移调补偿), 顺带扩展音域到 C0。
2. **噪音砍周期模式**: Q0 自反馈的周期噪音试听死锁/退化成方波, 已砍。纯白噪 4 挡 + 绑定已够用。
3. **噪音绑定直通 square_tc**: 旧方案 HC161.CET 门控让噪音频率和音高成反比 (反直觉), 改成直通后音越高噪音越密。
4. **死锁防护 RST 灌种**: HC164 无置位能力, RST 后全 0 死锁。RST 期间 serial_in=1 灌种, 白噪 max-length 自持永不归零。
5. **max-length taps Q7⊕Q5⊕Q4⊕Q3**: 多项式 x⁸+x⁶+x⁵+x⁴+1, 周期 255 (2⁸-1)。

## 弃用方案 (防止重走弯路)

- **周期噪音** (Q0 自反馈): 试听死锁/退化方波, 已砍
- **旧绑定 CET 门控**: 噪音频率和音高成反比, 改直通
- **复用 HC161 切波形** (v0.1 弃用): period 计数与满幅度计数冲突
- **TLC7524 反接电压模式** (v0.2 弃用): 噪音 + 音量失效
