# PSG1 v0.2 设计文档

> **1 通道方波 + 4-bit 音量（TLC7524 衰减器）**
> 在 v0.1（单通道方波，无音量）基础上加音量调节。
> 状态：🚧 规划中

## 1. 目标（相对 v0.1 的扩充）

| 项 | v0.1 | v0.2 |
|----|------|------|
| 通道数 | 1 | 1（不变）|
| 音量 | 无（gate 开关）| **4-bit 数字音量（TLC7524）** |
| gate | 有（HC00 与门）| **去掉**（音量码 0 = 静音，替代 gate）|
| 总线 | period + gate + rst | **period + 音量选通(A0) + rst**（更标准的 YM 风格）|

**两个改进**：
1. toggle_q 不直接接喇叭，经 TLC7524 衰减，输出幅度由 4-bit 音量码控制
2. **去掉 gate**——音量码 0000 自然静音，HC00 的 gate 与门可拆掉（HC00 仅保留 PE 反相）

> 去掉 gate 让总线侧更标准：控制信号变成 **LE(写period) + A0(写音量) + RST**，这是 YM/SID 风格的雏形，为后续多通道的总线复用直接铺路。

## 2. 架构（v0.1 基础上：加音量、去 gate）

```
FT232H → LS373 转换
  D0-D7 ──┬──→ HC373 D0-D7  (period, LE 选通)
          └──→ HC374 D0-D3  (vol, A0 选通)
  LE  ────→ HC373 LE        (写 period 脉冲)
  A0  ────→ HC374 LE        (写音量脉冲, v0.1 的 gate 位置)
  RST ────→ HC161/HCT74 MR  (复位)

PSG 核心 (HC373→HC161→HC00→HCT74):
  HC373 period → HC161 计数 → HC00 PE反相 → HCT74 toggle
                                                  │
                                                  ▼ toggle_q (方波 0/5V)
                                             ┌──────────┐
                                        ───→ │ TLC7524  │ ───→ 喇叭
                              vol[3:0] ───→ │ (衰减器) │
                                             └──────────┘
```

**v0.1 改动点**：
- HC00 的第 2/3 路（gate 与门）拆掉，toggle_q 直接输出（不再经 gate 选通）
- HC00 仅保留第 1 路（PE 反相）
- toggle_q 改接 TLC7524 的 OUT1（参考输入）
- 新增 HC374（音量锁存）+ TLC7524（衰减器）

## 3. TLC7524 音量衰减器（零外部元件）

TLC7524 是乘法 DAC（MDAC），内部自带 R-2R 梯子。用**反向电压模式**，不需外接运放/电阻。

### 接法
| TLC7524 脚 | 接 | 作用 |
|-----------|---|------|
| OUT1 (Pin 1) | ← toggle_q（v0.1 的方波）| 参考输入（被衰减的信号）|
| OUT2 (Pin 2) | → GND | 模拟地 |
| GND (Pin 3) | → GND | |
| VREF (Pin 15) | → 喇叭 | **衰减后的方波输出** |
| VDD (Pin 14) | → +5V | |
| RFB (Pin 16) | 悬空 | 反向模式不用 |
| CS (Pin 12) | → GND | 常选通 |
| WR (Pin 13) | → GND | 常写入（直通模式）|
| D4-D7 (Pin 4-7 查手册) | ← 音量码 vol[3:0] | 4-bit 音量（接高位）|
| D0-D3 | → GND | 固定 0 |

### 4-bit 音量档位
vol[3:0] 接 TLC7524 的 D4-D7（D0-D3 接地），实际 DAC 输入 = vol << 4：

| vol | D 值 | 输出幅度 | 相对音量 |
|-----|------|---------|---------|
| 0000 | 0 | 0% | 静音 |
| 0001 | 16 | 6% | 极小 |
| 0100 | 64 | 25% | 小 |
| 1000 | 128 | 50% | 中 |
| 1100 | 192 | 75% | 大 |
| 1111 | 240 | 94% | 满音量 |

> **接 D4-D7 高 4 位（最优接法）**：①必须连续 4 位——vol 权值 1:2:4:8 才能让 16 档等距铺开（无碰撞、不浪费）；非连续位权值跳跃，16 个值会挤成 4 团，实际只剩 4 档有效。②必须接高段——满量程 D=240（94% 幅度，4.69V），信噪比最好；接 D0-3 只有 5.9%。即"连续"保不浪费、"高段"保幅度，D4-7 是 4 位硬件 DAC 的最优解。

## 4. 音量码怎么写入（A0 选通）

v0.2 用 A0 选通音量锁存器（HC374），D0-D7 复用写 period 和音量。**gate 去掉后，原 gate 的 GPIO 位置(D5) 改成 A0**：

| FT232H | 信号 | 作用 | v0.1 对应 |
|--------|------|------|----------|
| C0-C7 | D0-D7 | 复用数据总线 | 不变 |
| D4 | LE | period 写选通 → HC373 | 不变 |
| **D5** | **A0** | **音量写选通 → HC374** | v0.1 是 gate，现改 A0 |
| D6 | RST | 复位 | 不变 |
| D7 | 备用 | 留给后续（多通道地址位）| - |

> FT232H 仍 11 根线。原 gate(D5) 升级成 A0（地址选通），语义更标准，为多通道铺路。

### 写时序
```
写 period: D0-D7=period, LE(D4) 脉冲   → HC373 锁存
写音量:   D0-D3=vol,    A0(D5) 脉冲   → HC374 锁存
静音:     写 vol=0000   → TLC7524 输出 0 → 自然静音（不需 gate）
```

## 7. 设计决策记录

- **单通道+音量，不碰多通道**：把"音量"和"多通道"两个难点拆开。音量验证成功后再开 PSG2 v0.3 扩通道。
- **去掉 gate，用音量码 0 静音**：音量调节天然包含"开关"功能（vol=0 即静音），gate 与门多余。去掉后 HC00 仅保留 PE 反相，且释放出 D5 GPIO 给 A0（音量选通），总线侧变成标准 LE/A0/RST 三信号。
- **A0 选通取代独立音量线**：D0-D7 复用写 period/音量，A0 选通音量锁存器。这是 YM/SID 总线复用的最小验证，为多通道直接铺路。
- **TLC7524 当衰减器而非波形 DAC**：用户无电阻库存，TLC7524 内部 R-2R 正好做衰减，零外部元件。
- **正向接法（REF=方波输入/OUT1=输出/RFB=GND）**：R-2R 电压输出，Gigatron 风格。
  ⚠️ 初版误用"反接电压模式"（OUT1=输入/REF=输出/RFB 悬空），实测噪音+音量失效——datasheet SLAS061D 电压模式只保证 OUT1≤2.5V 线性、且寄生电容随码跳变致噪。改成正向接法后恢复。
- **音量接 D4-D7 高 4 位（最优）**：连续 4 位保 16 档不浪费（非连续位会挤成 4 团），高段保满量程幅度（94%），二者兼得。4 位硬件 DAC 物理上限即线性 16 档，对数曲线需另换架构。
