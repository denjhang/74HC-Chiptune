# PSG1 v0.2 设计文档

> **1 通道方波 + 4-bit 音量（TLC7524 衰减器）**
> 在 v0.1（单通道方波，无音量）基础上加音量调节。
> 状态：🚧 规划中

## 1. 目标（相对 v0.1 唯一扩充）

| 项 | v0.1 | v0.2 |
|----|------|------|
| 通道数 | 1 | 1（不变）|
| 音量 | 无（gate 开关）| **4-bit 数字音量（TLC7524）** |
| 其他 | — | 完全复用 v0.1 的 PSG 核心 |

**唯一新增**：在 v0.1 的 toggle_q 方波输出和喇叭之间，插入 TLC7524 做音量衰减。PSG 核心电路（HC373/HC161/HC00/HCT74）完全不动。

## 2. 架构（在 v0.1 基础上加一级音量）

```
v0.1 部分 (不动):
  FT232H → LS373 → [HC373 period锁存 → HC161计数 → HC00 PE反相+gate → HCT74 toggle]
                                                                        │
                                                                        ▼ toggle_q (方波 0/5V)
v0.2 新增:
                                                                   ┌─────────────┐
                                                              ───→ │ TLC7524      │ ───→ 喇叭
                                                     vol[3:0] ───→ │ (衰减器)     │
                                                                   └─────────────┘
```

**关键**：toggle_q 不再直接接喇叭，而是进 TLC7524 的参考输入，输出幅度由 4-bit 音量码控制。

## 3. TLC7524 音量衰减器（零外部元件）

TLC7524 是乘法 DAC（MDAC），内部自带 R-2R 梯子。用**反向电压模式**，不需外接运放/电阻。

### 接法
| TLC7524 脚 | 接 | 作用 |
|-----------|---|------|
| OUT1 (Pin 1) | ← toggle_q（v0.1 的方波）| 参考输入（被衰减的信号）|
| OUT2 (Pin 2) | → GND | 模拟地 |
| GND (Pin 3) | → GND | |
| VREF (Pin 15) | → 喇叭 | **衰减后的方波输出** |
| VDD (Pin 14) | → +5V | |
| RFB (Pin 16) | 悬空 | 反向模式不用 |
| CS (Pin 12) | → GND | 常选通 |
| WR (Pin 13) | → GND | 常写入（直通模式）|
| D4-D7 (Pin 4-7 查手册) | ← 音量码 vol[3:0] | 4-bit 音量（接高位）|
| D0-D3 | → GND | 固定 0 |

### 4-bit 音量档位
vol[3:0] 接 TLC7524 的 D4-D7（D0-D3 接地），实际 DAC 输入 = vol << 4：

| vol | D 值 | 输出幅度 | 相对音量 |
|-----|------|---------|---------|
| 0000 | 0 | 0% | 静音 |
| 0001 | 16 | 6% | 极小 |
| 0100 | 64 | 25% | 小 |
| 1000 | 128 | 50% | 中 |
| 1100 | 192 | 75% | 大 |
| 1111 | 240 | 94% | 满音量 |

> 接高位（D4-D7）而非低位，保证 vol=1 时也有 6% 幅度（听得见），分布合理。

## 4. 音量码怎么写入

v0.2 仍用 FT232H 直驱（不引入总线复用，那是多通道的事）。音量码 4-bit 用 FT232H 的 4 个 GPIO：

| FT232H | 用途 | 说明 |
|--------|------|------|
| C0-C7 | period[7:0] | v0.1 不变 |
| D4 | period_le | v0.1 不变 |
| D5 | gate | v0.1 不变 |
| D6 | rst_n | v0.1 不变 |
| **D7** | **vol[0]** | 新增（经 LS373 转换）|
| **C8+ 或其他** | vol[1-3] | 新增 3 根 |

> ⚠️ FT232H 可靠 GPIO 只有 C0-C7 + D4-D7（共 12 个）。v0.1 用了 11 个（C0-C7 + D4-D6），只剩 **D7 一个**。
> 4-bit 音量要 4 根线，缺 3 根。**这是 v0.2 的核心矛盾**。

### 解决方案（待选）

| 方案 | 说明 | 代价 |
|------|------|------|
| **A. 加 1 片 HC374 锁存音量** | FT232H 复用 D0-D7 写音量到 HC374（类似 period 锁存），用 1 根 LE 选通 | 多 1 片 HC374，但 FT232H 不加线（复用 D0-D7 + 新 LE）|
| B. 2-bit 音量（接 D7 + 借 1 根）| 只用 2-bit 音量（4 档），省线 | 音量精度低 |
| C. SPI+HC595 串行扩展 | 用 3 根线串行写音量 | 多 HC595，且 PSG 不再纯并行 |

**推荐方案 A**——引入"地址/数据复用"的雏形：D0-D7 既写 period 也写音量，用不同 LE 选通。这也为后续多通道（PSG2/PSG3）的总线架构铺路。

### 方案 A 接法（推荐）
```
FT232H D0-D7 (复用) ──┬──→ HC373#1 D0-D7 (period, LE=FT232H D4)
                       └──→ HC374 D0-D3 (vol低4位, LE=FT232H D7)

HC373#1 Q → period (v0.1 不变)
HC374 Q0-Q3 → TLC7524 D4-D7 (音量码)
```
- 写 period：D0-D7=period，D4 LE 脉冲
- 写音量：D0-D3=vol，D7 LE 脉冲
- **FT232H 仍 11 根线**（C0-C7 数据 + D4/D5/D6 + D7），无新增 GPIO

## 7. 设计决策记录

- **单通道+音量，不碰多通道**：把"音量"和"多通道"两个难点拆开。音量验证成功后再开 PSG2 v0.3 扩通道。
- **TLC7524 当衰减器而非波形 DAC**：用户无电阻库存，TLC7524 内部 R-2R 正好做衰减，零外部元件。
- **反向电压模式**：OUT1 当输入、VREF 当输出，不需运放（参考 smbaker/mikekohn 实战）。
- **方案 A（HC374 锁存音量）**：引入 D0-D7 复用雏形，为多通道总线架构铺路，FT232H 不加 GPIO。
- **音量接 D4-D7 高位**：保证 vol=1 也有可听幅度（6%），分布合理。
