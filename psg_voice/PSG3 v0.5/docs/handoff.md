# PSG3 v0.5 交接文档

> PSG3 v0.5 = v0.4 (方波 + 噪音) **全部保留** + **新增波形通道** (方波/三角/锯齿/反锯齿切换)
> 交接时间：2026-07-09
> 状态：**三通道 RTL + tb 0 错误，C仿真器通过，4MHz 架构，待上板验证**

## 一句话现状

v0.4 的方波通道 + 噪音通道**原样保留**（reg0/1/2, clk=sq_clk 预分频 64kHz）。
v0.5 **新增波形通道**（reg3/4/5, clk=4MHz 直连）——一个 CD4029 4-bit 计数器，wave_sel 切换 4 种波形，HC283 比较器 + HC08 AND 门双路径，mode_sel 1-bit 切换音色调制方式。
**三通道并存**，各自独立 TLC7524 输出，上板分别接喇叭对比音色。

**周期分两类**（核心：wave_sel[0]=fold 控制有无 HC112 折返）：
- 锯齿族（锯齿/方波/反锯齿, fold=0）：单向回绕 = **16步/周期**, `freq = 4MHz/(16×(4096-p12))`
- 三角（fold=1）：HC112 折返 0→15→0 = **30步/周期**, `freq = 4MHz/(30×(4096-p12))`
- 两套 period 查找表：`host/uni_period_table.h` (uni_period12_saw / uni_period12_tri)

## 波形通道原理（核心创新）

**一个 CD4029 4-bit 计数器**，wave_sel 切换 4 种波形。核心区别在**有无 HC112 折返**（wave_sel[0]=fold）：

```
3×HC161 (period12) → freq_tc → CD4029 (4-bit 波形计数器)
                                   │ Q0-3
                    ┌──────────────┴──────────────────┐
                    │ HC273 (滤毛刺)                  │
                    │    ┌──────────┬─────────────────┤
                    │    │ HC283比较 │ HC08 AND        │
                    │    │ Q<duty   │ Q&duty          │
                    │    └────┬─────┴────┬────────────┘
                    └─ HC153 选 ─────────┘
                              │ wave_sel + mode_sel
                    TLC7524#1 (波形) → TLC7524#2 (音量) → 喇叭
```

| wave_sel | 波形 | fold | dir | CD4029 行为 | 周期 | HC153 选 | duty4 作用 |
|----------|------|------|-----|------------|------|---------|-----------|
| 00 | 锯齿 | 0 | 0 | 单向加 0→15→0 | **16步** | mode_sel 选 | 比较阈值/AND掩码 |
| 01 | 三角 | 1 | - | HC112 折返 0→15→0 | **30步** | mode_sel 选 | 比较阈值/AND掩码 |
| 10 | 方波 | 0 | 0 | 单向加 0→15→0 | **16步** | **固定比较** | **占空比** |
| 11 | 反锯齿 | 0 | 1 | 单向减 15→0→15 | **16步** | mode_sel 选 | 比较阈值/AND掩码 |

**wave_sel 两 bit 拆解**：bit[1]=dir（0加/1减），bit[0]=fold（0单向16步/1折返30步）。
方波 = 锯齿底子（fold=0+dir=0）+ mode_sel=1（HC283 比较）。所有波形统一，无特殊判断，方波两个 mode 都可用。
**mode_sel 1-bit 切换音色调制**：1=HC283比较（阈值调制，三角→方波加奇次谐波），0=HC08 AND（位掩码，降精度加量化高频）。同一个 duty4，两种模式产生不同音色。选通芯片 HC157（四 2选1）。

> **关于占空比的历史教训**：v0.3 的占空比靠 toggle 后分频链，25%/12.5% 会**降八度**。
> v0.5 用 HC283 比较器，占空比完全独立于频率（不降八度）。

> **关于周期的认知（2026-07-09 修正）**：之前误以为"所有波形统一 30 步"。
> 实际锯齿族（无折返）是 16 步/周期，三角（有折返）是 30 步/周期。所以分两套 period 表。
> 这符合"提 clk 或降步数才能提高精度墙"——16步墙高（低音下不去到 B1），30步墙低（高音 G7 突破 1%）。

## 寄存器 (三通道并存, 独热码地址)

```
reg0 (0x01): 方波 period (8 bit)                    ← v0.4 方波通道
reg1 (0x02): 方波控制 vol/duty/mode/ref             ← v0.4 方波通道
reg2 (0x04): 噪音控制 vol/freq/bind
reg3 (0x08): 波形 period12[7:0]                     ← v0.5 波形通道
reg4 (0x10): 波形 period12[11:8] | vol[3:0]         ← v0.5 波形通道
reg5 (0x20): 波形 duty[3:0](bit0-3) | mode_sel(bit4) | wave_sel[1:0](bit5-6) | 预留(bit7)
              wave_sel[1]=dir (0加=锯齿/方波, 1减=反锯齿)
              wave_sel[0]=fold (0单向16步, 1折返30步=三角)
              方波 = 锯齿(dir=0,fold=0) + mode_sel=1(比较), 无特殊判断
              mode_sel: 1=HC283比较(阈值调制) 0=HC08 AND(位掩码)
```

频率和音量放 reg0+reg1（最重要的两个参数在一起）。

### 音高公式（两套，按波形周期）

```
锯齿族 (16步): freq = 4000000 / (16 × (4096 - period12))
              覆盖: B1(61Hz) ~ C8(4186Hz), B1~C8 <0.55%, 74/85音<1%
三角 (30步):  freq = 4000000 / (30 × (4096 - period12))
              覆盖: C1(32.7Hz) ~ C8(4186Hz), 84/85音<1% (G7=1.12%最差)
查找表: host/uni_period_table.h (uni_period12_saw / uni_period12_tri, 穷举优化)
```

> **为什么两套**：锯齿族 16 步墙高（4MHz/16=250kHz），低音下不去（B1 以下失效）；
> 三角 30 步墙低（4MHz/30=133kHz），低音能到 C1 但高音精度差。各取所长。

### 关于精度的认知（重要）

STC32G 的 AY8910/SN76489 仿真器精度好，是用了 **NCO（24-bit 相位累加）**，不是计数器 reload。
真 AY8910 芯片在 1.79MHz 下高音精度也差（12-bit 直接计数）。本项目走真硬件路线（74 计数器），
用 4MHz 高 clk 缓解，非 NCO。NCO 留给后续 SCC。

## 芯片清单

### 总线层 (13 片, v0.4 不变)
2×LS373(电平转换) + HC374(地址锁存) + 8×HC374(reg0-7 寄存器) + HC04(NOT) + 3×HC08(选通)
> 详见 wiring-table-bus.md. 物理接线不变, 只改 U5/U6/U7 (reg3/4/5) 的 Q 去向给波形通道.

### 方波通道 CH0 (8 片, v0.4 不变)
2×HC161(period 8-bit) + HC00(PE反相) + HCT74(toggle) + HC153(占空比选通) + HC08(占空比AND) + TLC7524
> clk=sq_clk (预分频 64kHz). reg0/1 控制. 详见 wiring-table-square.md (复制 v0.4).

### 波形通道 CH2 (14 片, v0.5 新)
3×HC161(period12) + 2×HC00(U23反相器组/U32 UD-mux) + CD4029(波形) + HC112(方向) +
HC283(比较) + HC08(AND) + HC153(选择) + HC273(滤毛刺) + 2×TLC7524(波形+音量级联) + HC04(clk反相)

> U23 HC00 (4门全用): ~freq_tc(PE) / ~CO(at_extreme) / ~dir / ~fold
> U32 HC00 (4门全用): uni_ud 2选1 mux (3门) + ~rst_n (门4)
> at_extreme = ~CO (CD4029 CO 真值表已含 CI 条件, freq_tc 冗余, 省 1 个 AND 门)
> clk=4MHz 直连. reg3/4/5 控制. 详见 wiring-table-uni.md.

### 预分频 (3 片, 方波+噪音用)
2×HC161(÷63, reload=193/0xC1) + HC00(PE反相). 4MHz÷63=63492Hz(-0.8%, 软件补偿)
> 详见 wiring-table-prescaler.md. sq_clk 给方波通道 + 噪音通道.

### 噪音通道 CH1 (7 片 + 2 电容, v0.4 不变)
CD4070 + HC374 + HC161 + HC153 + TLC7524 + HC00×2
> clk=sq_clk. reg2 控制. 详见 wiring-table-noise.md.

**合计: 45 片** (总线13 + 方波8 + 波形14 + 预分频3 + 噪音7)
> 三通道各自独立 TLC7524 输出, 上板分别接喇叭对比音色.

## 总线扩展架构规划（v0.5 之后）

### 核心思想：标准化声音模块 + 地址扩展

每个声音模块（PSG方波 / 波形 / 噪音 / 未来模拟调制）接口统一：
```
D0-D7 (8根数据) + CP (1根地址选通) + rst_n
```
模块内部自行分配 8 位数据的含义，对外只暴露标准总线接口。
扩展通道 = 加地址 + 插模块，不改总线层，不改其他模块。

### 地址空间分析

总线是地址+数据**分时复用**（第1拍写地址进 HC374 锁存，第2拍写数据进 reg），
所以 8 位总线在地址拍 = 完整 8 位地址（256 个可能），数据拍 = 完整 8 位数据，不争抢宽度。

当前独热码（one-hot）：8 位地址只选 8 个 reg（1 位选 1 个），地址利用率 3%。
改成二进制译码后，8 位地址 = 最多 256 个 reg。

### 译码方案对比（库存芯片）

| 方案 | 地址位数 | 地址数 | 译码芯片 | 适用场景 |
|------|---------|--------|---------|---------|
| 当前独热码 | 8 (独热) | **8** | 不需要 (HC08 直选通) | ≤8 reg, 最简单 |
| 1 片 HC154 | 4 | **16** | 1 片 154 | 2-3 声道 |
| 2 片 HC154 级联 | 5 | **32** | 2 片 154 (A4 选片) | 5 声道 |
| 1×HC138 + 8×HC154 | 7 | **128** | 9 片 | 12+ 声道, 实际够用 |
| 页寄存器方案 | 8+ | **256~4096** | 1×HC374 + 1×HC154 | 理论无限扩展 |

### 各方案接线

**2 片 HC154 = 32 地址**（推荐起步方案）：
```
A0-A3 → 两片 154 共享地址
A4    → 154#1 的使能 (A4=0 选 #1)
A4反相 → 154#2 的使能 (A4=1 选 #2)
data_strobe → 两片 154 共享第二使能
每片 154 输出 16 个低有效选通 → 共 32 个 reg CP
```

**1×138 + 8×154 = 128 地址**（大扩展）：
```
A0-A3 → 8 片 154 共享地址 (段内 16 选 1)
A4-A6 → 138 的 3 位地址 (8 段选 1)
138 的 8 个输出 → 分别接 8 片 154 的使能
data_strobe → 138 的使能
= 8 段 × 16 = 128 个 reg
```

**页寄存器 = 256+ 地址**（理论极限）：
```
1 片 HC374 当"页寄存器" (占 1 个固定地址)
  写页号 → 选当前页
1 片 HC154 译页内地址 (A0-A3 → 16 个页内地址)
页寄存器输出(页号) + HC154 输出(页内选通) 组合 → 最终 CP

页寄存器 8 位 = 256 页 × 16 地址/页 = 4096 个 reg
访问流程: 先写页寄存器(切页), 再写页内地址
代价: 每次跨页多一次写操作
芯片数: 1 片 HC374 + 1 片 HC154 + 少量门 = 极少
```

### 每声道地址占用

| 通道类型 | reg 数 | 用途 |
|---------|--------|------|
| 方波 (v0.4) | 2 | period + vol/duty/mode |
| 噪音 (v0.4) | 1 | vol/freq/bind |
| 波形 DCO (v0.5) | 3 | period12 + vol + duty/wave/mode |
| 模拟调制 (未来) | 2-4 | VCA/滤波参数/MC1496 控制等 |
| **每声道合计** | **6-10** | |

- 32 地址 = 3-5 个完整声道
- 128 地址 = 12-20 个声道（实际用不完）
- 256+ 地址 = 理论上不设上限

### 结论

- **32 地址起步**（2 片 HC154）够 3-5 声道，接线简单
- **128 地址**（1×138 + 8×154）是实际项目的上限，12+ 声道
- **256 地址**用页寄存器方案最省芯片（1 片 374 + 1 片 154），但跨页要额外写操作
- 总线协议（YM2413 复用 + /CS 事务保护）完全不变，扩地址只在译码层动

## RAM + 总线仲裁：用 RAM 替代大量 374 寄存器

### 核心矛盾：锁存器 vs RAM

声音通道的参数（period/vol/duty）需要**持续在场**驱动振荡器。

| | HC374 寄存器 | RAM (6116) |
|---|---|---|
| 输出 | Q 端持续保持（零延迟） | 读出是瞬时的（给地址→等→出数据→消失） |
| 芯片数 | 1 参数 = 1 片 374 | 1 片存 2K 参数 |
| 适合 | 参数少、要持续输出 | 参数多、可分时读 |

**矛盾**：RAM 省芯片但读出是瞬时的，通道逻辑不能直接接 RAM 的 Q 端。

### 三种解法

**解法1：RAM 当仓库 + 374 当当前值（没省 374）**
```
写参数: 总线 → RAM (存起来, 大仓库)
用参数: RAM → (读出) → 374 (锁存当前值) → 通道逻辑
```
RAM 存大量备选参数（如多个音色预设），切换时从 RAM 读到 374。
但 374 还是要的——没省芯片，只是 RAM 当了"参数库"。

**解法2：分时轮询 TDM（YM2608/YM2151 路线）**
```
拍0: 读RAM通道0的period → 算通道0 → 输出采样0
拍1: 读RAM通道1的period → 算通道1 → 输出采样1
...
拍N: 回到通道0
```
1 个计算单元轮流处理所有通道，不需要每通道 374。
但需要"算"的逻辑（累加器/查表），不再是纯硬件振荡器。
**这就是 NCO/查表机路线**——CLAUDE.md 四点五节的 fsm_player/39040 方向。

**关键认知（2026-07-10）**：TDM 分发不需要通用 CPU 的分支/循环。
因为动作序列**完全固定**——查表机的微程序 ROM 按地址递增读，
每个地址存一条"读 RAM 哪个地址 → 分发到哪个通道"的固定指令。
不用判断不用跳转，纯顺序执行，地址计数器线性递增（HC161）。

```
微程序 ROM (SST39SF, 顺序读, 固定序列循环):
  拍0: RAM地址=ch0_period → 读RAM → 锁存到通道0的period
  拍1: RAM地址=ch0_vol    → 读RAM → 锁存到通道0的vol
  拍2: RAM地址=ch1_period → 读RAM → 锁存到通道1的period
  拍3: RAM地址=ch1_vol    → 读RAM → 锁存到通道1的vol
  ...固定序列, 到末尾回绕重来...
```

= **fsm_player 思路的升级版**：fsm_player 顺序读 ROM 输出 PCM，
这里改成"顺序读微程序 ROM 控制 RAM 分发"。
RAM 当参数仓库，查表机当快递员，省掉 N×M 片 374。

**解法3：双口 RAM（CY7C130，库存有）**
```
左口: CPU 写参数 (地址+数据, 随时写)
右口: 通道逻辑持续读 (地址固定, 等效 Q 持续输出)
```
右口固定地址 = 等效"可写的 374"。1 片双口 RAM = N 个虚拟锁存器。
但右口一次只读 1 个地址，多通道要多片双口 RAM 或分时切换右口地址。

### 总线仲裁（多主设备争 RAM）

当 RAM 同时被 CPU（写参数）和通道逻辑（读参数）访问时，需要仲裁：

```
       ┌─── 仲裁器 ───┐
CPU ──→│  谁先访问?    │──→ RAM
通道 ──→│  分时/优先级  │
       └──────────────┘
```

简单方案：**CPU 优先，通道插空**。通道逻辑在每个音频采样周期有固定窗口读 RAM，
CPU 在窗口外写。类似 YM2413 的 /CS 事务保护，但粒度更细（采样级）。

双口 RAM 天然解决仲裁——两边独立口，不需要仲裁器，但要库存里有双口 RAM（CY7C130 有）。

## YM2608 的 ADPCM 和 PCM 音源

### YM2608 怎么用 DRAM

YM2608 (OPNA) **外挂** 256K DRAM 存 ADPCM 采样（不是内置 DRAM）：
- ADPCM = 自适应差分 PCM（4-bit 压缩的 PCM，6-bit→4-bit 压缩比）
- YM2608 内部有 ADPCM 解码器 + **地址发生器**（硬件状态机）

播放流程：
1. CPU 写寄存器：设定采样**起始地址 + 结束地址 + 播放频率**
2. YM2608 内部地址计数器从起始地址开始，按频率递增
3. 每读一个字节 → ADPCM 解码 → PCM 样本 → DAC 输出
4. 到达结束地址 → 停止或循环

**不用翻页**——线性地址计数器直接遍历 DRAM。CPU 只设起止参数，不用逐字节搬运。

### PCM 音源的本质

PCM 音源 = **地址计数器 + ROM/RAM + DAC**：
```
地址计数器 (按频率递增) → ROM/RAM[地址] → DAC → 输出
                                    ↑ 循环 (地址回起点)
```

和 NCO 的对比：
```
NCO:  phase += step; idx = phase>>N; out = sin_table[idx]   (查正弦表)
PCM:  addr += step; out = pcm_rom[addr]                      (查录音采样)
```
**结构完全一样**——PCM 就是"查表内容是录音而不是正弦"的 NCO。

### 用 74 系列做 PCM 的方案

```
HC161×N (地址计数器, 按频率递增)
    │ 地址
    ▼
SRAM/ROM (存 PCM 采样, 6116/SST39SF)
    │ 采样值 (8-bit)
    ▼
HC273 (采样保持, 滤毛刺)
    │
    ▼
TLC7524 (DAC, 数模转换)
    │
    ▼
运放 (低通滤波, 去采样噪声)
    │
    ▼
喇叭
```

- 地址计数器：N 片 HC161 级联（采样长度决定位数：8 位=256 样本，16 位=64K 样本）
- 频率控制：HC161 的 reload 值（类似 PSG 的 period），或步进值（NCO 式累加）
- ROM/SRAM：存 PCM 数据。ROM（SST39SF）适合固定音色，SRAM（6116）适合可写
- HC273：采样输出在地址跳变瞬间有毛刺，边沿锁存滤掉
- TLC7524：8 位 DAC，已经在用

**fsm_player 就是这个架构的简化版**（CLAUDE.md 四点五节，9 片，顺序读 ROM 回放）。

### ADPCM vs PCM

| | PCM | ADPCM |
|---|---|---|
| 每采样位数 | 8-16 bit | 4 bit（压缩） |
| 解码 | 直接读 | 差分解码（要解码器） |
| 音质 | 好 | 稍差（有量化噪声） |
| 存储 | 大 | 小（省一半） |
| 74 实现 | 简单（地址+ROM+DAC） | 复杂（要差分解码逻辑） |

→ 用 74 系列做 **PCM 比 ADPCM 简单得多**。ADPCM 要解码器（查表+累加），等于查表机。
→ PCM 只要地址计数器 + ROM + DAC，不需要"算"。

### PCM 音源和当前 PSG3 的关系

PSG3 v0.5 的波形通道（DCO）是**计算式波形**（CD4029 折返生成三角/锯齿）。
PCM 是**查表式波形**（ROM 里存录好的采样，地址递增回放）。

| | DCO (当前 PSG3) | PCM (未来方向) |
|---|---|---|
| 波形来源 | 计数器实时计算 | ROM 预录采样 |
| 频率控制 | period (reload 值) | 地址步进值 |
| 波形类型 | 方波/三角/锯齿（数学波形） | 任意（钢琴/弦乐/人声） |
| 音色丰富度 | 低（只有数学波形+调制） | 高（任意录音） |
| 芯片 | CD4029+HC283+HC153+... | HC161+ROM+HC273+DAC |
| 和 fsm_player 关系 | 不同路线 | **fsm_player 就是简化版 PCM** |
