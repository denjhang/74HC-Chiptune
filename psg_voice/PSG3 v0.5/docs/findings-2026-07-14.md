# PSG3 v0.5 上板发现 (2026-07-14)

> 本次上板验证的关键发现. 彻底搞清楚了 74 系列 DAC 波形生成的可行与不可行.
> **核心结论: 计数器分频链叠加方案能做锯齿/方波, 做不了高质量三角. 三角要 NCO/查表.**

---

## 一、实验历程 (按时间)

### 1. CD4029 做波形计数器 → 失败

**原 v0.5 设计**: CD4029 (CMOS 4-bit 可逆计数器) 做波形计数, Q0-3 → HC273 滤毛刺 → TLC7524.

**现象**: 4029 的 Q1/Cout 相对方波, 但 **Q2/Q3/Q4 都是方波+噪音**. 整体输出垃圾混叠声.

**根因**: CD4029 是 CMOS 计数器, 4MHz 下卡 fmax 边缘 (5V 下典型 4-6MHz). 内部异步级联链 (Q1→Q2→Q3→Q4 逐级传递) 来不及稳定, 高位输出带噪音.

**教训**: ⚠️ **高频 (≥1MHz) 不用 CD 系列**. CD4029/CD40161 等 4000 系列 CMOS 留给低频/模拟开关. 高频用 HC/HCT (HC161/HCT74, fmax 30-60MHz).

### 2. HC161 做波形计数器 → 部分失败

**方案**: 换 HC161 (HC 同步计数器, fmax 30MHz+), CET/CEP=freq_tc 控制步进, Q0-3 → DAC.

**现象**:
- HC161 任何 Q 直接听: 只有混叠谐波, 无噪音 (比 CD4029 好)
- Q0-3 → 273 → DAC (透明模式): **白噪**
- Q0-3 → 273 → DAC, CS 接 clk (同步采样): **方波→占空比方波→谐波方波→不稳定方波, 每八度循环**

**根因 (核心)**: HC161 的 Q0/Q1/Q2/Q3 是**分频链抽头**, 不是"同步的计数值":
- Q0 = clk/2 (最高频)
- Q1 = clk/4
- Q2 = clk/8
- Q3 = clk/16 (最低频)

R-2R DAC 是二进制加权 (DB7×8 + DB6×4 + DB5×2 + DB4×1). 把 Q0-3 接 DB4-7 = **4 个不同频率的方波叠加**, 不是"一个锯齿的 4 个位".

各位频率不同 → 叠加结果完全取决于 4 个方波的**相位关系** → 相位关系随 period 循环 → 音色循环 (方波/占空比/谐波/不稳定).

**273 / CS-clk 都救不了**: 因为问题不是"同步性", 是 Q0-3 本质上是 4 条独立分频链, 锁存一个时刻的快照只是冻结, 不改变值的分布.

### 3. TC 整形: 必须两级 (对齐 + ÷2)

**发现**: HC161 的 TC (终端计数) 有毛刺. 直接用 → 波形计数器步进错乱.

**v0.4 已验证的做法**: TC 过 HCT74 两级:
- FF1: D=TC, CLK=clk → 对齐去毛刺 (clk 同步采样)
- FF2: D=~Q, CLK=FF1.Q → toggle ÷2 → 洗净相位信息

**关键铁律**: ⚠️ **整形 = 对齐 + ÷2, 缺一不可**. 只对齐一次 (单 FF 或 273) 不够, Q 仍带相位耦合. 必须再 ÷2 分频才纯净.

### 4. 分频链叠加 = 锯齿 (rev.a, 成功!)

**突破**: 既然 Q0-3 是 4 个分频链抽头, 不如**直接用纯净的 tc_hi 分频链**, 4 个抽头二进制加权叠加:

```
tc_hi (HCT74 FF2, 纯净方波)
  ├─────────────→ DB4 (权重1, 最高频)
  ├→ ÷2 ────────→ DB5 (权重2)
  ├→ ÷4 ────────→ DB6 (权重4)
  └→ ÷8 ────────→ DB7 (权重8, 最低频)

CS = clk (同步采样 4 路)
= 锯齿 0→1→2→...→15→0 (16步/周期) ✅
```

**验证通过** (2026-07-14): 干净锯齿波, 高频接 DB4 (低位), 低频接 DB7 (高位) = 正向锯齿.

**5 位升级**: 再加一级 ÷16, 5 抽头进 DB3-7 = 32 步锯齿, 精度翻倍. 明显比 4 位更好.

### 5. XOR 折返做三角 (rev.b, 失败)

**方案**: 分频链 + 1 片 HC86, 用 tc_hi/16 当方向位, 低 4 位 XOR 它:
```
DB4 = tc_hi   ⊕ tc_hi/16
DB5 = tc_hi/2 ⊕ tc_hi/16
DB6 = tc_hi/4 ⊕ tc_hi/16
DB7 = tc_hi/8 ⊕ tc_hi/16
```
数学上 = 三角 0→15→0 (32步, 满幅).

**现象**: **不是标准三角**. 全频段 (低频到高频) 都带锯齿波泛音. 过下拉电容能滤出"类三角", 但不纯正.

**根因**: 三角波对**相位精度**要求极高.
- 折返点 (峰值 15→15→14) 附近, 方向位翻转, 4 位同时取反
- 分频链各位到达 XOR 的延迟不同 (tc_hi 0 级, /8 有 3 级, /16 有 4 级), 累计 30-80ns
- 取反瞬间各位先后翻转 → 折返毛刺 → 锯齿泛音
- 12-bit period 整数 reload 无小数精度, 折返点漂移

**锯齿没事, 三角有事的原因**:
- 锯齿: 递增台阶, 各位变化是"累加", 某位延迟几 ns 形状基本不变
- 三角: 折返动作要求 4 位严格同步翻转, 任何延迟都产生毛刺, 毛刺频谱 = 锯齿泛音

**对比 NES** (STC32G144K246/nes.c): NES 三角用 **NCO (16-bit 相位累加器) + 查表**, freq 连续可变 (亚步进精度), 三角表是静态的 (无折返动作). 74 硬件的计数器 reload + 分频链方案达不到这个精度.

### 6. XOR 折返的"调音色"价值

**发现**: rev.b 的 XOR 折返虽然做不出标准三角, 但**能调波形音色**.
- 方向位 tc_hi/16 = 0 时: 锯齿
- 方向位 tc_hi/16 = 1 时: 反锯齿
- 两种模式按 tc_hi/16 频率交替 = "类三角" (带锯齿泛音)
- 实际效果: 在**锯齿波和方波之间**调音色 (折返频率 = tc_hi/16, 高频时听感接近方波)

**结论**: XOR 折返方案可保留作为"音色调制", 但不能当真正的三角波用.

## 二、⭐ 核心铁律 (全部上板验证)

### 1. 高频不用 CD 系列
CD4029/CD40161 在 4MHz 卡 fmax 边缘, Q2-4 噪音.
→ **高频 (≥1MHz) 只用 HC/HCT**. CD 系列留给低频/模拟开关.

### 2. ~~TC 必须经 HCT74 两级整形 (对齐 + ÷2)~~ (已推翻, 见第 6 条)
- ~~FF1: clk 同步采样 TC → 对齐去毛刺~~
- ~~FF2: toggle ÷2 → 洗净相位信息~~
- ⚠️ **rev.e2/e3 推翻**: TC 直连下一级 HC161 的 CEP/CET, 直连 ROM 地址, 都不需要整形.
  "整形"只针对 rev.a 的 Q→DAC 方案 (TC 分频生成纯净方波叠加锯齿), 不泛化到所有 TC 用途.
- v0.4 方波通道的 TC→HCT74 整形是历史方案 (生成分频链抽头), 新设计 (ROM 查表) 不需要.

### 3. 计数器 Q 不能直接 DAC
HC161/CD4029 的 Q0-3 是**分频链抽头** (Q0=clk/2, Q1=clk/4...), 不是"同步计数值".
- 直 DAC (透明) = 白噪 (4 路异步叠加)
- CS=clk (同步) = 音色循环 (相位关系随 period 循环)
- 273 救不了 (只对齐一次)
→ **Q 不能直 DAC, 但 TC / Q 可以直连任何 TTL (级联/ROM 地址/计数器)**.

### 4. 分频链叠加 = 锯齿 (可行, rev.a 验证)
tc_hi (纯净) → ÷2 → ÷4 → ÷8, 4 抽头进 DB4-7 (二进制加权).
- **高频接 DAC 低位** (DB4=tc_hi), **低频接高位** (DB7=tc_hi/8) = 正向锯齿
- 反过来接 = 反锯齿
- ⚠️ **CS=clk 只对分频链叠加必需** (4 路方波异步叠加会白噪, 必须同步采样).
  ROM 查表架构不需要 CS=clk, 见第 7 条.
- 5 位 (÷16) 比 4 位精度高一倍, 明显更好

### 5. 分频链 + XOR 做不出标准三角 (精度不够)
- 三角波对相位精度要求极高 (折返点 4 位同步翻转)
- 分频链各位延迟不同 → 折返毛刺 → 锯齿泛音
- 12-bit period 整数 reload 无小数 → 折返点漂移
- NES 用 NCO + 查表解决, 74 计数器 reload 做不到
→ **三角波要改 NCO/查表架构 (fsm_player 路线)**.

### 6. TC 直连任何 TTL, 不需整形 (2026-07-15 推翻第 2 条)
- HC161 级联 HC161: TC 直连下一级 CEP/CET, 不需 HCT74.
- HC161 的 TC 直连 ROM 地址线: 不需特殊处理.
- HC161 的 TC 直连地址计数器使能: 不需整形.
- "TC 必须整形" (第 2 条) **只针对 rev.a 的 Q→DAC 方案** (TC 分频生成纯净方波), 不泛化.

### 7. ROM 查表 DAC: CS 接 GND 比 CS=clk 更干净 (2026-07-16 rev.e3 验证)
- **CS=clk 不适合 PCM/ROM 查表模式**: 4MHz clk 通过 DAC 数字输入端耦合到模拟输出 (时钟串扰),
  且 clk 上升沿正好采到 ROM 地址翻转的过渡态 (地址刚变, 输出还在过渡) → 毛刺被冻结进 DAC.
  PCM 采样是连续变化的模拟信号, clk 同步采样反而把每个地址翻转的过渡毛刺都冻进去, 听感更脏.
- **CS=GND (透明模式) 适合 PCM/ROM 查表**: DAC 持续跟踪 ROM 输出, 过渡期平滑过渡
  (乘法 DAC 输出连续), 毛刺被平均而不是冻结; 没有 clk 信号进 DAC 数字端 → 无时钟串扰路径.
- ⚠️ **CS=clk 只对分频链叠加 (rev.a 波形模式) 必需** (4 路方波异步叠加会白噪, 必须 clk 同步采样).
  ROM 查表 (rev.d 波形 / rev.e3 PCM) 输出是完整稳态字节, 不存在异步叠加, 透明模式更好.
- rev.e3 实测: CS=GND 后总线噪声明显降低.
- **结论: clk 适合波形分频链, 不适合 PCM. PCM/ROM 查表一律 CS=GND.**

## 十一、DSP vs MCU 的架构反思 (2026-07-14 总结)

### 复古音源芯片的"秘密"

一周实验下来, 彻底看清了所有复古音源芯片的本质 — **全是同一套**: 计数器/累加器 → 查表 → DAC. 包装不同, 内核一样.

| 芯片 | 宣传 | 实际原理 | rev.e 对应 |
|------|------|---------|-----------|
| NES APU | "5 通道" | 计数器查表 + 方波/三角 | sel=0x01 tri ✓ |
| SID 6581 | "3 通道 + 滤波器" | 计数器查表 + RC 滤波 | rev.c RC 滤波 ✓ |
| AY-3-8910 | "3 通道 + 噪声" | 计数器分频 + LFSR | reg2 噪音 ✓ |
| YM2608 | "FM + PCM + 鼓" | 相位累加 + 查表 + 采样 | rev.e PCM ✓ |
| GameBoy DMG | "4 通道" | 计数器 + WaveRAM (32点查表) | sel=0x0D pacman ✓ |
| SN76489 | "4 通道" | 分频计数器 + LFSR | reg0-1 方波 ✓ |

**核心算法 = 查表, 一行 `output = table[address++]`**. 70 年代就这么干, 现在还这么干. 变化只在计数器位数 / 表内容 / DAC 精度 / 加不加包络 / 加不加滤波. 原理层面确实是"雕虫小技".

### 为什么 rev.e 是 DSP 级 (超过单片机)

STC32G12K128 播放 wavetable 是**中断驱动**, 倍频播放会卡 (ISR 算力瓶颈):
```
每个采样周期 → 定时器中断 → ISR 里查表 + 包络 + 混音 → 写 DAC
步进快过 ISR 处理极限 → 卡顿/爆音
```

rev.e 是**纯硬件并行**, 全单周期:
```
地址生成:  HC161 CEP/CET (组合逻辑) → clk 沿 Q+1 (单周期)
ROM 查表: 地址线 → ROM 译码器 (组合逻辑) → 55ns 出数据 (不等 clk)
273 锁存: ROM 数据 → clk 沿锁存 (单周期)
TLC7524:  DB × vol → R-2R 电阻网络 → 电流 (零时间, 模拟)
```

**全流水线并行**: 每 clk 周期同时走地址 / 查表 / 锁存 / 输出, 无等待.

| | STC32G (软件) | rev.e (硬件) |
|---|---|---|
| 查表 | CPU 取指令 + 读 RAM (10+ 周期) | ROM 组合逻辑 **55ns** |
| 步进 | ISR 里 phase += step (5+ 周期) | HC161 并行自跑 |
| 包络 | CPU 乘法 (20+ 周期) | TLC7524 模拟乘法 **零时间** |
| 混音 | CPU 加法 (串行) | 运放求和 **零时间** |
| 最高采样率 | ~20-40kHz (ISR 极限) | **4MHz** (clk 极限) |

软件是顺序执行 (时间分片), 硬件是空间展开 (物理并行). 同一个算法, 硬件把"步骤"变成"导线", O(n) 时间 → O(1) 时间. 这就是 DSP 芯片比 MCU 快的本质.

**周期除法也是硬件**: `addr / 256` 取高位 = A8-15 这根线; `addr % 256` 取低位 = A0-7 这根线. 没有除法指令, 就是物理接线, 零周期.

### 成本反思: 工业流片 vs 创客积木

芯片公司 (SID/YM2608) 模式 — **重资产, 高风险**:
- 流片 (N 掩模): 几百万到千万人民币
- 设计团队: 十几个工程师 × 一两年工资
- EDA 工具授权: 百万级/年
- Tape-out 失败重来的风险
- 量产才能摊薄

本项目模式 — **零成本, 零风险**:
- **芯片: ¥0**. 库存 300+ 种型号, 5 年从闲鱼超低价收 + 圈子里群友免费接盘. 一颗几分钱都不要, 根本不是"买"的.
- **设计: ¥0** (GLM 一星期搞定原理)
- **试错: ¥0** (面包板拔线重插, 不重新流片)
- **PCB: 嘉立创免费打样**
- **面包板: ¥2**

**库存即"流片资产"**: 随时可重组搭出任何电路, 而且完全免费. 比流片强 — 流片出来是固定的, 库存是可重组的. 74 系列是通用逻辑 (乐高积木), 一片 HC161 今天做 period 计数, 明天拔下来做地址计数, 投资一次无限复用. 专用音源芯片 (SID/YM2608) 用完了就废了, 只能干音源这一件事.

### 本质总结

**工业时代 (芯片公司)**: 重资产 + 长周期 + 高风险 + 规模摊薄. 花的是钱, 钱是有限的.
**创客时代 (本项目)**: 轻资产 + 快迭代 + 零风险 + 知识复用. 花的是脑子和时间, 越攒越多.

复古音源公司花几百万流片做的事, 本项目用 5 年攒的元件库存 + 面包板 + GLM 一星期就重现了原理. 他们花的是钱, 我们花的是脑子和时间. **钱是有限的 (烧完就没了), 知识是无限的 (越攒越多)**.

这才是真正的"雕虫小技" — 技术民主化, 把原来要几百万的事做到 ¥2 (面包板) 内. 这套玩法在 80 年代不可能 (没面包板生态 / 没便宜通用 74 / 没免费 PCB / 没 AI 辅助), 现在全有了.

## 十二、rev.e3 PCM 上板实验结论 (2026-07-16)

> rev.e3 = PCM 专用最简版 (砍掉 7474/HC08/HC04, 10 片, 无限循环). 上板基本能用, 但暴露了 PCM 回放的硬件上限.

### 1. 8kHz 采样率太低, 量化噪声大

- 8kHz 是 PCM 的下限, 高频成分全丢.
- **2-4 倍速回放 (16-32kHz 等效) 音质明显好转**, 3x 最经典 (TOP 踩镲开).
- 1x (8kHz) 量化噪音太大, **几乎不能用**.
- 结论: PCM 至少要 16kHz 起, 8K 不够.

### 2. 面包板 ROM 高速访问 = 噪声上限

- ROM 快速查表 (4MHz 地址翻转) 时, 面包板寄生电容/电感把地址总线噪声耦合进 DAC 和整个电路.
- 这是 **面包板本身的音质上限**, 不是设计问题.
- 要解决必须 **走线优化 (PCB)**, 面包板到此为止.

### 3. a12 接法 = 节拍模式 (意外发现)

- a12 接 GND: 采样每 8192 步从 0 重播 = **快速连打** (鼓机效果).
- a12 接 U6 (最后一片 HC161) 的不同 Q 输出:
  - Q0 (÷4096): 基本同 a12 toggle, 每段 4096 步
  - Q1 (÷8192): 每 8192 步翻一次, 慢一倍节拍
  - Q2, Q3: 更慢的节拍循环
- **无限循环 + a12 选不同抽头 = 不同节拍的循环采样**, 这是地址计数器的天然副产品.

### 4. PCM 音量远小于波形/方波

- rev.d 波形 (sine/square) 是稳态循环, RMS 接近峰值 (71-100%).
- rev.e3 PCM: 鼓组短采样填充静音 (RMS 3-6%), 乐器动态大 (RMS 15-24%).
- **响度差 3-10 倍**, TLC7524 乘法衰减级联压得更低.
- 可能需要 **PCM 通道配独立运放** 才能和波形通道音量匹配.

### 5. 后续方向 (待定)

PCM 要真正可用, 几条路 (都需要更多硬件):

- **提升采样率**: 16kHz+ → 占更多地址空间 (8K/槽 不够, 要 16K+).
- **换 39SF040** (512K): 更多槽位 / 更高采样率.
- **39040 CPU 路线**: 用查表 CPU 做更复杂的采样调度 (变采样率/包络/多段循环).
- **加运放 + 减法放大器**: PCM 通道独立放大, 同时用减法器减掉 ROM 总线耦合噪声 (面包板噪声问题的硬件解法).
- **暂时放弃全量 PCM**: rev.d 波形 + rev.e2 双模式可能比纯 PCM 更实用, PCM 等上 PCB 再认真做.

### rev.e3 实验价值

rev.e3 证明了: **PCM 回放原理完全可行** (HC161 地址计数 → ROM → DAC, 无限循环), 但 **8kHz + 面包板是音质硬上限**. 后续要突破, 必须上 PCB + 独立运放 + 更高采样率, 不是接线能解决的.
